>
> 그래서 본 강의의 핵심 한 줄은 이렇게 외우시면 됩니다. **MCP 서버 1개 = AI에게 새로운 손발 1쌍.** 본 강의에서는 12쌍을 답니다."

### 🎬 [2:30~4:00] Compare · MCP 없이 vs MCP 있으면 (Google Sheets 예시)

〔슬라이드: 04 좌우 2단 비교 · Google Sheets 사례〕

**🎙️ 강사 발화 (그대로 읽기):**

> "추상적이니까 마케터에게 가장 익숙한 도구로 비교해보겠습니다. **Google Sheets(구글 시트)** 예요. 이번 달 캠페인 데이터를 모아 둔 시트가 있다고 가정합니다.
>
> 사용자의 요청은 단순합니다. '**이번 달 캠페인 시트 분석해서 채널별 ROAS(로아스) 정리해줘.**'
>
> **왼쪽 · MCP 없이 하면.** 지금 대부분의 마케터가 하시는 방식이에요. 시트를 열어 셀 범위를 드래그해서 복사. 챗봇 창에 붙여넣고 '분석해줘' 입력. 받은 결과를 다시 시트로 옮겨서 입력. 매주 반복하면 한 번에 30분이 사라집니다. 그리고 본인이 한 일은 사실상 **복사 붙여넣기 노동**이에요. 마케팅 판단이 아니라.
>
> **오른쪽 · MCP가 있으면.** Google Sheets MCP 서버를 1줄로 설치해 두면 끝. AI 가 본인 시트를 **직접 읽고 계산하고 새 탭에 결과를 정리**합니다. 사용자는 채팅창에 한 줄만 입력. '이번 달 시트 분석해줘.' 30초 안에 새 탭이 생기고 채널별 ROAS 가 정리되어 있어요.
>
> 차이가 무엇이냐 · **'사람이 데이터를 옮긴다' 가 '사람이 결과만 본다' 로 바뀌는 것**입니다. 30분이 30초로 줄어드는 게 핵심이 아니라, 마케터의 시간이 **복붙 노동에서 판단으로 이동**한다는 게 핵심이에요.
>
> 이 구조가 본 강의의 12개 MCP 전부에 똑같이 적용됩니다. **Meta(메타) 광고 매니저·Notion(노션)·Buffer(버퍼)·Discord(디스코드)** 모두 같은 원리예요. 한 번 설치해두면, 그 도구의 데이터를 더 이상 사람이 옮기지 않습니다."

### 🎬 [5:30~7:00] Diagram · MCP 작동 원리 4단계

〔슬라이드: 06 흐름 다이어그램 (사용자 → Claude Code → .mcp.json → MCP 서버 → API → 시스템)〕

**🎙️ 강사 발화 (그대로 읽기):**

> "MCP가 실제로 어떻게 작동하는지를 4단계로 따라가 보겠습니다. 마케팅 시나리오를 그대로 따라갑니다.
>
> 사용자가 입력합니다. **'이번 주 광고 성과 시트에 정리해줘.'**
>
> **1단계 · Claude Code가 받습니다.** 자연어 명령을 해석해서 어떤 손발이 필요한지 판단해요. 광고 데이터 조회 + 시트 작성이 필요하다고 결정.
>
> **2단계 · `.mcp.json` 매니페스트를 확인합니다.** 프로젝트 폴더 안의 `.mcp.json` 파일에는 켜질 MCP 서버 목록이 적혀 있어요. 그 중 Meta Ads MCP 서버와 Google Sheets MCP 서버를 호출.
>
> **3단계 · 각 MCP 서버가 자기 일을 합니다.** Meta Ads MCP는 Meta Graph API(그래프 에이피아이)를 호출해 광고 데이터를 가져오고, Google Sheets MCP는 Sheets API를 호출해 시트에 작성.
>
> **4단계 · Claude가 결과를 정리해 응답합니다.** '완료. weekly-ads(위클리 애즈) 시트에 7행이 추가됐어요.'
>
> 시간으로 비교하면 · MCP 없이는 30분 걸리던 작업이 30초 안에 끝납니다. 그리고 이 흐름의 핵심은 · **사용자는 1줄만 입력하고, 나머지는 모두 자동**이라는 점이에요. **이게 본 강의 Part 2의 목표 상태**입니다."

### 🎬 [7:00~8:00] FAQ · 자주 묻는 오해 3가지

〔슬라이드: 07 3카드 (Q&A 3건)〕

**🎙️ 강사 발화 (그대로 읽기):**

> "MCP를 처음 접하는 분들이 자주 헷갈리는 3가지를 풀고 가겠습니다.
>
> **Q1. MCP는 챗봇의 도구 사용(Tool Use(툴 유즈))과 같은 거 아닌가요?**
>
> 비슷해 보이지만 층이 다릅니다. **Tool Use** 는 AI가 도구를 '어떻게 호출하는가' 의 메커니즘이고, **MCP** 는 그 도구가 '어디에 어떤 모양으로 정의되어 있는가' 의 표준이에요. 쉽게 말해 · **MCP는 도구를 미리 만들어 놓은 사람이 있다**는 것이고, Tool Use는 AI가 그 도구를 골라서 부른다는 것. 둘은 한 세트로 쓰입니다.
>
> **Q2. 그냥 API를 직접 호출하면 되지 않나요?**
>
> 가능합니다. 다만 그러려면 API마다 도구 스키마·인증·에러 처리를 본인이 다 짜야 해요. MCP는 그걸 표준화해서 **'한 번 설치하면 평생 재사용'** 으로 만든 규격입니다. 본 강의 마케터의 시간은 통합 코딩이 아니라 마케팅에 쓰여야 하니까요.
>
> **Q3. MCP 서버는 누가 만드나요? 신뢰할 수 있나요?**
>
> 세 부류가 만듭니다. **서비스 회사 공식** (Stripe(스트라이프)·Notion·GitHub 등이 직접 제공), **Anthropic 공식** (자주 쓰는 도구는 Anthropic이 [Directory(claude.ai/directory)]에 검수해서 게시), **커뮤니티** (오픈소스 개발자가 GitHub에 공개). 본 강의에서는 **공식·검수된 것만** 사용합니다. 정체 불명의 MCP 서버는 절대 끼우지 마세요. 프롬프트 주입 위험이 있습니다."

### 🎬 [8:00~9:00] 진행 방식 · 12개 MCP 영상 → 설치 마라톤 1편

〔슬라이드: 08 2단계 흐름 (이론 → 설치 마라톤) + 하단 "왜 설치를 뒤로?"〕

> "본격 시작 전에 · Part 2 의 진행 방식을 한 번 정리하고 가겠습니다. 구조는 아주 단순해요. **두 단계**입니다.
>
> **STEP 1 · 영상 먼저 쭉 보기 (지금부터).** 12개 MCP 를 5분짜리 이론 클립으로 봅니다. 각 클립에서는 그 MCP 의 의미·결과물·마케터 가치를 설명하고, 시연을 1분 보여드려요. **설치도, 키 발급도, 환경 설정도 일절 하지 마세요.** 화면만 보고 학습하시면 됩니다. 일단 12개를 머리에 다 넣는 게 목표예요.
>
> **STEP 2 · 마지막 클립에서 설치 한 번에 (30~40분).** 이론 클립이 모두 끝나면, 마지막에 30~40분짜리 **'설치 마라톤' 클립 한 편**이 기다리고 있습니다. 그 클립에서 **키 발급 안내부터 설치·검증까지 한 번에** 진행해요. 본인이 할 일은 채팅창에 '**MCP 전체 설치하자**' 한 줄 입력하고, 안내에 따라 키 발급해서 붙여넣기만 하면 됩니다.
>
> **왜 설치를 뒤로 미루느냐.** 12개를 다 보시면 '이거 다 쓰고 싶다' 싶어지세요. 그때 한 번에 설치하면 **동기 부여 + 시간 효율 둘 다** 잡힙니다. 만약 매 클립마다 키 발급하고 설치하면, 그 5분이 매번 끼어들어 흐름이 끊겨요. 그래서 본 강의는 **이론 12편 + 설치 1편** 으로 깔끔하게 분리했습니다.
>
> 다시 한 번 · **지금부터는 그냥 보세요.** 손은 마지막 클립에서 한 번에 움직입니다."
