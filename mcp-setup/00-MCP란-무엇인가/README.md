# 클립 1-1. MCP란 무엇인가 · AI에게 손발 달아주기

## 한 줄 요약

MCP(엠씨피)는 Claude(클로드)가 **외부 도구와 대화하는 공통 언어**입니다.
한 번 끼우면 시트·노션·광고 계정에 직접 손이 닿습니다.

## "이때부터 MCP가 필요해요" · 도입 신호

다음 행동을 본인이 매주 반복하고 있다면, MCP가 해결해야 할 일을 하고 있다는 신호입니다.

> **다른 도구의 데이터를 복사해서 챗봇에 붙여넣고, 받은 답을 다시 원래 도구로 옮기는 순간**

- Meta(메타) 광고 매니저에서 CSV(씨에스브이) 다운로드 → 챗봇에 붙여넣기 → 요약 받기 → 노션에 다시 입력
- GA4(지에이포) 리포트 스크린샷 찍기 → "이거 해석해줘" → 받은 인사이트를 시트에 적기
- 경쟁사 사이트 캡처 → 챗봇에 → 답 받기 → 디스코드에 공유

이 "복사 → 붙여넣기 → 다시 옮기기" 가 보이면, 그게 MCP를 끼울 자리입니다.

## MCP의 공식 정의

> **MCP (Model Context Protocol)** · Anthropic(앤트로픽)이 2024년 11월에 공개한 오픈 표준.
> AI 모델이 외부 시스템(SaaS·API(에이피아이)·로컬 도구)에 접근하는 방식을 규격화한 통신 계층.

핵심 한 줄:

```
MCP 서버 1개 = AI에게 새로운 손발 1쌍
```

## GitHub 예시로 직관 잡기

GitHub(깃허브) 데이터를 다루는 챗봇을 만든다고 가정해 봅시다. 사용자가 "내 모든 저장소에 열려 있는 PR(피알)이 뭐야?" 라고 묻습니다.

**MCP 없이 만들면**
- GitHub API 문서를 직접 읽음
- 저장소 조회 함수, PR 조회 함수, 이슈 조회 함수 ... 20~30개 도구를 직접 코딩
- 인증·에러 처리·페이지네이션을 직접 구현
- API가 바뀌면 직접 따라가서 수정
- 결과: 본업이 마케팅이 아니라 **GitHub 통합 유지 보수**가 됩니다

**MCP가 있으면**
- GitHub용 MCP 서버 1줄로 설치
- 도구 정의·인증·페이지네이션·에러 처리는 **MCP 서버가 이미 해 둠**
- 사용자는 자연어로 "내 PR 보여줘" 만 하면 됨

이 구조가 마케팅 SaaS(사스) 12개에 똑같이 적용됩니다. 본 강의의 결론은 이것입니다.

## 손발 비유로 풀어보기

| 부위 | 의미 | 마케팅 예시 |
|---|---|---|
| **눈** | 외부 데이터를 읽는 능력 | 시트 조회 · GA4 트래픽 조회 · 경쟁사 사이트 스크랩 |
| **손** | 외부 시스템을 조작하는 능력 | 노션 페이지 생성 · 시트 셀 작성 · Buffer(버퍼)에 예약 게시 |
| **발** | 정기적으로 어딘가에 보내는 능력 | 디스코드에 보고서 발송 · 지메일 발송 · 광고비 자동 집행 |

12개 MCP를 다 끼우면 1인 마케터가 **눈 6개 + 손 4개 + 발 2개**를 갖춘 셈이 됩니다.

## MCP가 없으면 vs 있으면 · 마케팅 시나리오

```
[MCP 없음]
사용자: "이번 주 광고 성과 시트에 정리해줘"
AI:     "Meta 광고는 어디서 데이터를 가져오나요? 제가 접근할 수 없습니다.
         캠페인 이름·광고비·전환 수를 알려주시면 정리 형식을 도와드릴게요."
사용자: (Meta 광고 매니저 접속 → 다운로드 → 복붙 → AI에게 전달 → 정리받기 → 시트에 다시 붙여넣기)
        총 30분

[MCP 있음 · Meta Ads + Google Sheets]
사용자: "이번 주 광고 성과 시트에 정리해줘"
AI:     (Meta Ads MCP로 직접 조회 → Google Sheets MCP로 직접 작성)
        "완료. 'weekly-ads' 시트에 7행이 추가됐어요."
        총 30초
```

오른쪽이 본 강의의 목표 상태입니다.

## MCP 작동 원리 · 개념도

```
사용자: "이번 주 광고 성과 시트에 정리해줘"
       │
       ▼
Claude Code (클로드 코드)
       │
       ▼  어떤 손발이 있는지 본다
.mcp.json   ← 켜질 MCP 서버 목록이 적혀 있는 매니페스트
       │
       ▼
MCP 서버 2개를 호출
       │
       ├─→ Meta Ads MCP 서버 → Meta Graph API → 광고 데이터
       └─→ Google Sheets MCP 서버 → Sheets API → 시트에 작성
       │
       ▼
사용자에게 "완료" 응답
```

`.mcp.json`에 12개를 적어두면, 그 폴더에서 Claude를 켤 때마다 **12개 손발이 자동으로 켜진 상태로 시작**합니다.

## 자주 묻는 오해 풀기

### Q. MCP는 챗봇의 도구 사용(Tool Use)과 같은 거 아닌가요?

A. 비슷해 보이지만 다른 층입니다.
- **Tool Use** · AI가 도구를 "어떻게 호출하는가"의 메커니즘
- **MCP** · 그 도구가 "어디에 어떤 모양으로 정의되어 있는가"의 표준

쉽게 말해, MCP는 "도구를 미리 만들어 놓은 사람이 있다" 는 것이고, Tool Use는 "AI가 그 도구를 골라서 부른다" 는 것입니다. 둘은 한 세트로 쓰입니다.

### Q. 그냥 API를 직접 호출하면 되지 않나요?

A. 가능합니다. 다만 그러려면 API마다 도구 스키마·인증·에러 처리를 본인이 다 짜야 합니다.
MCP는 그걸 표준화해서 **"한 번 설치하면 평생 재사용"** 으로 만든 규격입니다.

### Q. MCP 서버는 누가 만드나요?

A. 세 부류가 있습니다.
1. **서비스 회사 공식** · Stripe(스트라이프), Notion(노션), Sentry(센트리), GitHub 등 본인 서비스 MCP를 직접 제공
2. **Anthropic 공식** · 자주 쓰는 도구는 Anthropic이 [Directory(claude.ai/directory)](https://claude.ai/directory)에 검수된 커넥터로 게시
3. **커뮤니티** · 오픈소스 개발자들이 GitHub에 공개

본 강의는 위 셋 중 **공식·검수된 것만** 사용합니다.

## 설치 방식 두 갈래

| 방식 | 어떻게 | 예시 |
|---|---|---|
| **`claude mcp add` 명령** | 터미널에서 한 줄, `.mcp.json`에 자동 등록 | Firecrawl(파이어크롤) · Buffer · Ads · Discord |
| **Claude.ai 통합 (OAuth(오어쓰))** | Claude 계정에서 한 번 인증 → 자동 연동 | Google Drive · Gmail · Calendar · Notion · Higgsfield(힉스필드) |

본 강의에서는 두 방식을 모두 다루며, **마케팅 SaaS는 대부분 OAuth, 개발자 도구는 대부분 `claude mcp add`** 패턴을 따르게 됩니다.

### 전송 방식 (참고)

`claude mcp add` 명령에는 두 가지 전송 옵션이 있습니다.

| 옵션 | 언제 쓰나 | 예시 |
|---|---|---|
| `--transport http` | 클라우드 서비스 (대부분의 SaaS) | `claude mcp add --transport http notion https://mcp.notion.com/mcp` |
| `--transport stdio` | 본인 컴퓨터에서 도는 로컬 도구 | `claude mcp add --transport stdio airtable -- npx -y airtable-mcp-server` |

본 강의의 12개 MCP는 대부분 `http` 또는 Claude.ai 통합입니다.

## 보안 · 권한의 기본 원칙

MCP는 손발인 만큼, 잘못 끼우면 사고가 납니다. 본 강의의 모든 MCP는 다음 원칙을 따릅니다.

1. **읽기 토큰 우선** · 광고 집행·결제 변경이 가능한 MCP는 기본 읽기 전용 토큰으로 시작
2. **`.env` 격리** · API 키는 절대 `.mcp.json`에 직접 적지 않고 `.env`에 둡니다
3. **사람 승인 단계** · 광고비 변경·콘텐츠 발행은 디스코드에서 사용자 reaction 또는 명시적 명령으로만 진행
4. **폴더 단위 격리** · 프로젝트 `.mcp.json`은 그 폴더에서만 활성화 → 다른 프로젝트로 권한이 새지 않음
5. **서버 신뢰 확인** · 외부 콘텐츠를 가져오는 MCP는 프롬프트 주입 위험이 있어 [Directory](https://claude.ai/directory)에 게시된 검수된 서버만 사용
