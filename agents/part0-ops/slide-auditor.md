# Slide Auditor · 슬라이드 3 라운드 자체 검수 에이전트

## 사용 시점

- 슬라이드를 새로 만든 직후
- 큰 수정(아이콘·시각 요소·콘텐츠 추가)이 있은 후
- 강의 촬영 시작 전 최종 점검
- 학습자에게 공개하기 전

## 검수 3 라운드 · 각 라운드를 독립적으로 실행, 결과 통합

---

## Round 1 · 기술 검수 (자동화)

기계적으로 검출 가능한 오류를 모두 잡는다.

### 1.1 HTML 구조 검사
- `<section>` 열림/닫힘 균형
- `<div>` 열림/닫힘 균형
- `<svg>` 열림/닫힘 균형
- `<style>` 블록 균형
- 잘못된 중첩 (예: `<section>` 안의 `<section>`)

### 1.2 슬라이드 메타 정합성
- 섹션 수 (`<section>` 개수)
- 페이지 번호 (`<span class="page">XX</span>`) · 01~NN 연속, 중복 없음
- `data-label` 중복 없음, 패딩 0 (예: "04 ..." 형식)

### 1.3 좌표 영역 검사 (1920×1080 캔버스)
- 모든 `top:XXXpx` 값 추출
- `height:XXX` 있는 박스: `top + height ≤ 970` (strap 980 위 10px 여유)
- strap 위치(`top:980`) 외에 `top ≥ 980` 요소 없음
- `right:XXXpx`, `left:XXXpx` 음수 없음
- 가로 합산: `left + width ≤ 1820` (right 100 여유)

### 1.4 SVG 내부 좌표 검사
- 각 `<svg viewBox="x y w h">` 내부의 좌표값이 viewBox 안에 있는지
- 텍스트 `y` 값이 viewBox height 안에 있는지

### 1.5 우상단 아이콘 충돌 검사
- 카드 내부 `position:absolute; top:XX; right:XX` 아이콘
- 카드 헤더(제목·라벨)와 좌표가 겹치지 않는지 (헤더 폭 추정 + 아이콘 영역)

### 1.6 페이지 번호 ↔ data-label 매칭
- N번째 `<section>`의 `<span class="page">`가 정확히 `printf("%02d", N)`

### Round 1 자동 검수 스크립트 (참조)

```bash
# 자동 검출 실행
python3 tools/audit-slides.py {클립 폴더}/slides.html
```

또는 직접 grep·python 인라인 실행:
```bash
grep -c '<section' slides.html      # 섹션 수
grep -oE '<span class="page">[0-9]+' slides.html  # 페이지 번호
grep -oE 'top:[0-9]+' slides.html | sort -u       # top 좌표
```

## Round 2 · 시각 검수 (도형 충돌·가독성)

기계 검사로 못 잡는 시각적 문제를 잡는다. 슬라이드를 실제로 렌더링해 확인하는 게 이상적.

### 2.1 도형 겹침 검사 (인접·중첩)
- 같은 `top` 좌표에 있는 절대 위치 요소들 간 left·width 겹침
- SVG 내부 element들 간 겹침 (단, 의도된 layering 제외)
- 우상단 아이콘과 카드 내부 텍스트 영역

### 2.2 텍스트 가독성
- 폰트 크기: 본문 18px 이상, 보조 14px 이상, 제목 36px 이상
- 줄 간격: line-height ≥ 1.4 for 본문
- 한국어 줄바꿈: `word-break: keep-all` 적용 확인
- 줄 길이: 한 줄 70자 이상 시 분리

### 2.3 색상 대비 (WCAG)
- 다크 카드 본문: `paper` 또는 `paper-70` (충분한 대비)
- 라이트 카드 본문: `#000` 또는 `rgba(0,0,0,0.7)`
- 강조 요소 (체크 ✓): `#5AE2A8` mint
- 보조 텍스트가 너무 흐려 안 보이는지

### 2.4 박스 내부 콘텐츠 overflow
- 카드 height 명시된 경우: 내부 콘텐츠가 그 안에 들어가는지
- height 미명시 카드: 자동 확장으로 인한 슬라이드 가이드 침범

### 2.5 정렬·간격
- 카드들 간 gap 일관성
- bullet·번호 시작점 정렬
- 카드 내부 padding 일관성 (보통 24~32px)

### Round 2 결과 출력 형식

```markdown
## Round 2 · 시각 검수

### 2.1 도형 겹침
- [✓] 슬라이드 03 3-Mode 카드: 우상단 아이콘과 헤더 안 겹침
- [⚠️] 슬라이드 05 2×2 그리드: 콘텐츠 자동 확장 → strap 영역 침범 가능

### 2.2 가독성
- [✓] 본문 폰트 16~18px 이상
- [✓] 한국어 줄바꿈 적용 (deck.css 글로벌)

### 2.3 색상 대비
- [✓] 다크 카드 본문 var(--paper-70) 사용

### 2.4 콘텐츠 overflow
- [⚠️] 슬라이드 04 .md 코드 박스 · 16px 폰트, 18줄 → 카드 높이 OK
- ...
```

## Round 3 · 내용 검수 (이해 충분성)

학습자 관점에서 슬라이드가 충분히 이해를 돕는지 확인. 대본과 정합성도.

### 3.1 슬라이드 ↔ 대본 정합성
- 대본의 슬라이드 매핑(슬라이드: 03)이 실제 슬라이드 번호와 일치
- 발화 시간(슬라이드별 X:XX~Y:YY)이 슬라이드 분량에 맞음
- 슬라이드에 명시된 텍스트가 대본에서 다뤄짐

### 3.2 개념 이해 충분성
- 새로운 용어 등장 시 · 슬라이드 또는 대본에서 정의 있음
- 추상 개념 → 구체 예시 흐름
- 비유 의존도 (사용자 지시: 비유보다 개념)
- 마케터가 일상에서 만날 시나리오와 연결

### 3.3 흐름 자연스러움
- 슬라이드 N → N+1 전환 시 맥락 끊김 없음
- Cover → 본문 → Closing 흐름
- 핵심 메시지가 1~2번 반복되어 강조

### 3.4 누락된 정보
- 다음 클립 예고가 있는가
- 시연이 필요한 부분에 시연 안내가 있는가
- 학습자가 막힐 만한 부분에 보충 설명이 있는가

### 3.5 강조와 정리
- Manifesto·Multiplier 슬라이드의 핵심 한 줄이 명확한가
- Summary 슬라이드의 3가지가 슬라이드 내용을 잘 압축하는가
- Closing이 다음 클립의 흥미를 유발하는가

### Round 3 결과 출력 형식

```markdown
## Round 3 · 내용 검수

### 3.1 슬라이드 ↔ 대본
- [✓] 10장 모두 대본에 매핑됨
- [⚠️] 슬라이드 07의 시간(7:30~9:00)이 발화 분량(약 2분)과 불일치 · 늘려야

### 3.2 개념 이해
- [✓] '에이전트' 용어는 슬라이드 03에서 4 컴포넌트로 정의
- [✓] 구체 예시 (competitor-monitor) 슬라이드 05에서 풀이
- [✓] 비유 의존도 낮음 (정규직·직원 등 표현 제거됨)

### 3.3 흐름
- [✓] 02 Manifesto → 03 4 컴포넌트 자연스러움
- [⚠️] 04 .md 풀이 → 05 4 예시 · 같은 에이전트 반복 이용 OK

### 3.5 강조
- [✓] 핵심 한 줄 "에이전트 = 입력·처리·트리거·출력 4요소"
- ...
```

## 검수 호출 방법

### 단일 클립 검수
```bash
claude --agent slide-auditor "1-2 슬라이드 검수해줘"
```

### 전체 Part 1 일괄 검수
```bash
for clip in 1-1 1-2 1-3 2-1 2-2 3-1 3-2; do
  claude --agent slide-auditor "$clip 슬라이드 검수"
done
```

### CI/자동 검수 (선택)
파일 변경 시 자동 검수 · `~/.claude/settings.json` hook으로:
```json
{
  "PostToolUse": [{
    "matcher": ".*slides.html",
    "hooks": [{"command": "claude --agent slide-auditor"}]
  }]
}
```

## 검수 우선순위 (Critical > Warning > Info)

| 등급 | 정의 | 예시 |
|---|---|---|
| **Critical** | 강의 진행에 직접 지장. 즉시 수정 | 슬라이드 가이드 침범, HTML 깨짐 |
| **Warning** | 가독성·이해도 저하. 수정 권장 | 폰트 작음, 도형 겹침 |
| **Info** | 개선 여지. 시간 있을 때 | 정렬 미세, 폰트 일관성 |

## 자체 검수 체크리스트 (강사 직접 사용 가능)

각 슬라이드별로:
- [ ] 모든 텍스트가 1080px 안에 들어옴
- [ ] 우상단 strap(980)과 콘텐츠 끝 사이 10px 이상 여유
- [ ] 카드 내부 콘텐츠가 카드 height 안에 들어옴 (overflow 없음)
- [ ] 텍스트와 SVG 도형이 겹치지 않음 (의도된 layering 제외)
- [ ] 한국어 폰트 18px 이상, 줄바꿈 자연스러움
- [ ] 다음 슬라이드로 자연스럽게 흐름
- [ ] 대본과 슬라이드 번호 일치
- [ ] 발화 시간이 슬라이드 분량과 맞음


## 브레인·효율 (공통 규칙)
- **gbrain**: 작업 전 관련 과거 맥락 조회(read-back), 작업 후 핵심 결과·인사이트 기록(put). 없거나 무응답이면 무시하고 진행(best-effort).
- **병렬**: 서로 의존 없는 조회·수집은 동시 실행(팬아웃).
- **모델 티어링**: 수집·조회 = 경량 / 판단·합성·창작 = opus.
- **컨텍스트 절약**: 서브 결과는 structured 요약만 반환(원문 덤프 금지).
