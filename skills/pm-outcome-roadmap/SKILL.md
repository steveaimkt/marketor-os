# 성과 기반 로드맵 (Outcome Roadmap)

기능 목록("A 만들기, B 붙이기")을 **고객·비즈니스 임팩트 진술**("누가 어떤 성과를 얻어서 어떤 결과가 나는가")로 바꾸는 스킬. "무엇을 만드나"가 아니라 "무엇이 바뀌나"로 계획을 재구성한다.

## 언제 호출하나
- 사용자가 "로드맵 짜줘", "분기 계획", "우선순위 정하자", "뭐부터 할까", "이 기능들 정리해줘"라고 할 때
- 기능·아이디어가 나열식으로 쌓였는데 순서·근거가 없을 때 (output 목록 → outcome 로드맵으로 승격)
- TFT·프로젝트 착수 전 "왜 이 순서인가"를 성과·가설·지표로 못박아야 할 때
- 다른 스킬/에이전트가 여러 산출물을 큰 그림 위에 배치해야 할 때 (상위 정렬 문서로 호출)

경계: 이미 확정된 단일 산출물 1건을 만드는 작업(뉴스레터 1통·리포트 1장)은 이 스킬이 아니다. 이 스킬은 **여러 항목의 순서·근거를 성과로 정렬**하는 계획 단계 전용이다.

## 입력

```yaml
horizon:              # 선택 · 기본 Now/Next/Later (또는 분기: Q3/Q4/2027)
  - Now              # 지금~6주 · 착수 확정
  - Next             # 다음 · 준비 중
  - Later            # 이후 · 방향만

items:                # 필수 · 로드맵에 올릴 기능·아이디어·프로젝트 원재료 (나열식 OK)
  - "커뮤니티 론칭"
  - "교육 상품 패키징"
  - "컨설팅 오퍼 구성"
  - ...

north_star:           # 선택 · 없으면 Steve 매출 북극성 자동 적용
  goal: "2026 매출 5억 → 2030 100억"
  metric: "월 반복 매출(MRR) · 유료 전환 인원"

context:              # 선택 · 제약·현재 자원·시장 상황
  - "현 커뮤니티 무료 멤버 167명 (유튜브 유입)"
  - "1인 운영 · 동시 2개 이상 TFT 불가"
```

## 워크플로 (성과 로드맵)

**0. gbrain read-back (착수 전 필수)**
`mcp__gbrain__recall` 또는 `search`로 태그 `roadmap`·해당 도메인 과거 로드맵·확정 우선순위·북극성 정렬을 먼저 읽는다. 이미 정해진 순서·성과 정의를 재발명하지 않는다. (없으면 새로 시작한다고 명시)

**1. 정보 수집**
입력 items와 north_star, context를 정렬한다. 북극성이 비면 Steve 매출 북극성(2026 5억 → 2030 100억)을 기본 정렬축으로 놓는다. 각 item이 어느 매출 궤도(커뮤니티·교육·컨설팅·상품화)에 기여하는지 태깅한다.

**2. "그래서?" 심문 (각 item마다)**
- 어떤 **성과(outcome)**를 노리나? (기능이 아니라 상태 변화)
- 어떤 **고객 문제**를 푸나? (누구의 어떤 불편)
- 어떤 **비즈니스 지표**가 오르나? (북극성과 연결)
- 이게 왜 지금/다음/나중인가? (의존성·자원)

**3. 성과 진술 변환 (템플릿 고정)**
각 item을 이 문장으로 다시 쓴다:
> **"[고객 세그먼트]가 [원하는 성과]를 이루게 해서 [비즈니스 임팩트]가 나게 한다"**

그리고 항목마다 3종을 붙인다:
- **목표 성과(Outcome)**: 검증 가능·측정 가능한 상태 변화
- **가설(Hypothesis)**: "이걸 하면 → 이 성과가 날 것이다" (틀릴 수 있는 문장)
- **성공지표(Metric)**: 성과 달성 여부를 판정할 수 1개 숫자

원칙:
- 하나의 성과를 여러 기능이 달성할 수 있다 → **성과를 우선**하고 기능은 교체 가능한 수단으로 둔다
- 날짜 못박기보다 **유연한 시간지평**(Now/Next/Later)을 쓴다
- 지표 없는 항목은 로드맵에 못 올린다 (측정 불가 = 성과 아님)

**4. 시간지평 정렬 + 의존성**
북극성 기여도·의존성·1인 자원 제약으로 Now/Next/Later에 배치한다. 순서 근거(왜 이게 먼저인가)를 각 지평에 한 줄로 남긴다.

**5. gbrain put + 로컬 저장 (착수 후 필수)**
`mcp__gbrain__put_page`로 확정 로드맵을 태그 `roadmap`으로 적재(제목·시간지평·각 항목 성과/가설/지표·북극성 정렬 요약). 그다음 로컬 파일 저장. 노션 로드맵 DB/페이지 연동은 사용자 승인 후.

## 산출물

로컬: `outputs/{YYYY-MM-DD}/pm-outcome-roadmap/{YYYY-MM-DD}-roadmap.md`
(원하면 노션 로드맵 DB 행 또는 페이지로 미러링 · 발행 전 승인 게이트)

형식:

```markdown
# {제목} 성과 로드맵 · {날짜}
북극성: 2026 매출 5억 → 2030 100억 (정렬축: MRR·유료 전환)

## Now (지금~6주)
### 1) 커뮤니티 론칭  [궤도: 커뮤니티]
- 성과: 무료 멤버 167명이 "매주 오는 이유"를 얻어 주간 활성 멤버로 전환된다
- 가설: 주간 라이브+실습 자료를 붙이면 → 4주 내 WAU 30% 도달
- 지표: 주간 활성 멤버(WAU) ≥ 50명 · 유료 대기 신청 ≥ 20명
- 근거: 유료 궤도(교육·컨설팅)의 리드 풀 · 다른 항목의 선행 조건

## Next (다음)
### 2) 교육 상품 패키징  [궤도: 교육]
- 성과: 커뮤니티 활성 멤버가 첫 유료 교육에 등록해 첫 매출 궤도가 열린다
- 가설: 커뮤니티에서 검증된 주제를 4주 코스로 묶으면 → 활성 멤버의 10%가 등록
- 지표: 첫 기수 유료 등록 ≥ 15명 · 결제 전환율 ≥ 10%
- 근거: 커뮤니티 리드 풀 확보(항목 1) 후 착수 · 1인 자원상 동시 불가

## Later (이후)
### 3) 컨설팅 오퍼 구성  [궤도: 컨설팅]
- 성과: 교육 수료 고객 중 상위 니즈층이 고단가 컨설팅으로 올라 객단가가 뛴다
- 가설: 수료생 대상 1:1 오퍼를 열면 → 수료생의 5%가 고단가 계약
- 지표: 컨설팅 계약 ≥ 3건 · 객단가 ≥ 500만원
- 근거: 교육 수료 모수(항목 2)가 있어야 성립 · 방향만 확정

## 정렬 요약
- 순서 논리: 커뮤니티(리드) → 교육(첫 매출) → 컨설팅(객단가). 각 단계가 다음의 모수를 만든다.
- 폐기·보류: {측정 불가·북극성 미연결로 제외한 항목과 사유}
```
