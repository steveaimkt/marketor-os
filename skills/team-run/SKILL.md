# team-run · 마케팅팀 팀 가동 오케스트레이터

트루먼(메인 세션 = 리더 고정)이 에이전트 팀을 소집·조율·해산하는 실행 스킬.
**정본 규약은 [agents/TEAM-MODE.md](../../agents/TEAM-MODE.md)** · 이 스킬은 그 규약의 실행 절차다.

## 실행 모드: 에이전트 팀 (리더 = 트루먼)

## Phase 0: 사전 점검 + 재실행 분기

1. 팀 기능 확인: 이 세션에서 팀원 스폰이 가능한지 (env 미반영 세션이면 "새 세션에서 다시 불러달라" 안내 후 종료)
2. **세션당 활성 팀 1개** · 이미 팀이 떠 있으면: 기존 팀에 작업 추가 vs 해산 후 재구성 중 사용자 선택
3. 재실행 분기 (하네스 Phase 0 준용):
   - `outputs/{오늘}/team-{편성명}/` 미존재 → 신규 가동 (Phase 1)
   - 존재 + 부분 수정 요청 → 해당 팀원만 재스폰, 기존 산출물 경로를 프롬프트에 포함해 피드백 반영 지시
   - 존재 + 새 입력 → 기존 폴더를 `…-{HHMMSS}/`로 이동 후 신규 가동

## Phase 1: 편성 판별 + 승인 게이트 ⏸

1. 요청을 TEAM-MODE §1 기준으로 판별 · **팀이 과한 업무(단건·순차 체인·동일 파일 편집)면 서브에이전트로 강등 제안**하고 종료
2. 표준 편성 3종 중 선택 (또는 혼합 설계):

| 편성 | 팀원 (기존 agents/ 타입 재사용) | 작업 성격 |
|---|---|---|
| 부서 협업팀 | 해당 부서장 3~5 (research/content/ads/commerce/…-lead) | 부서 산출물 상호 인계 |
| 검수 토론 패널 | perspective-reviewer ×3~5 (관점: 대표·재무·고객·법무·브랜드) | 서로 반박, 합의만 생존 |
| 리서치 스웜 | trend-scanner·competitor-monitor·voc-analyzer | 발견 공유·교차 검증 |

3. **브리핑 ⏸ (의무)**: 편성표(팀원·이름·담당·예상 작업 수) + **토큰 비용 고지**("팀원 수만큼 독립 세션 · 단일 세션 대비 수 배") → 사용자 승인 후에만 진행

## Phase 2: 팀원 스폰 (현행 API · TeamCreate 없음, 자연어 스폰)

각 팀원을 다음 규칙으로 생성:
- **타입 지정**: "Spawn a teammate using the {agent-type} agent type" · tools 허용목록·model 자동 준수
- **이름 규칙**: 부서명 그대로 (`research`·`content`·`ads`) 또는 관점명 (`ceo-lens`·`finance-lens`) · 이후 지목 가능하게
- **스폰 프롬프트에 반드시 포함** (팀원은 리더 대화 기록을 상속하지 않는다):
  ① 담당 범위 + 구체 입력(브랜드=brand/profile.md 참조, 기간·목표수치)
  ② 산출물 경로: `outputs/{날짜}/team-{편성명}/{팀원이름}/` **전용** (공용 파일 수정 금지)
  ③ gbrain 규약: 작업 전 read-back → 작업 후 같은 태그 put
  ④ 중간 발견은 관련 팀원에게 직접 메시지로 공유할 것
- **mutating 작업 팀원**: "Require plan approval before changes" + 승인 기준 명시 (발행·예산 포함 계획은 거부하고 리더에게)

## Phase 3: 작업 분배

- 공유 작업 목록에 팀원당 5~6개 작업 등록 (자체 포함 단위: 산출물 1개 = 작업 1개)
- 의존 관계 설정: 인계가 필요한 작업은 선행 작업 완료 후 요청 가능하게
- 남는 작업은 미할당으로 두어 먼저 끝난 팀원이 자체 요청

## Phase 4: 모니터링 (리더 본업 · 직접 구현 금지)

- 팀원 유휴 알림·메시지를 받아 재지정/보완 지시
- 막힌 작업: 상태 수동 갱신 or 대체 팀원 스폰
- 팀원 계획 승인 요청: TEAM-MODE §3-2 기준으로 판정

## Phase 5: 종합 → 게이트 → 기록 → 해산

1. 전 작업 완료 확인 후 리더가 산출물 종합 (팀원 결과를 파일에서 Read)
2. **게이트는 팀 밖**: 발행물이면 gate-auditor를 **서브에이전트로** 호출 (⛔ 차단 권한 단일화)
3. 결과확인 ⏸: "보완할 점 없으세요?" → 피드백 반영
4. 기록: build-log 1행 + gbrain put (편성명·핵심 인사이트 태그) + 필요 시 Discord/Notion 발송(⏸)
5. 팀원 전원 종료 요청 → 세션 종료 시 자동 정리

## Output Format (Phase 1 브리핑)

```
[📢 팀 소집 브리핑] {업무명} · {편성명}
팀원: {이름=타입} × N · 작업 {M}개 · 산출물 outputs/{날짜}/team-{편성명}/
⚠️ 토큰: 독립 세션 {N}개 (단일 대비 수 배)
진행할까요? ⏸
```

## Anti-Patterns

- TeamCreate/TeamDelete 도구 호출 시도 (v2.1.178+에서 제거됨 · 자연어 스폰이 현행)
- 승인 없이 팀 소집 · 토큰 고지 생략
- gate-auditor를 팀원으로 편입 (차단 권한 분산)
- 리더가 팀 가동 중 직접 구현 시작 (모니터링·종합이 본업)
- 두 팀원에게 같은 파일 쓰기 허용 (덮어쓰기 사고)
- 팀원 스폰 프롬프트에 컨텍스트 생략 ("알아서 해줘" · 팀원은 리더 대화를 모른다)

## 테스트 시나리오

**정상 흐름 · 부서 협업팀:**
1. 입력: "신제품 런칭 준비 팀으로 돌려줘"
2. Phase 0: outputs/{오늘}/team-런칭/ 미존재 → 신규 가동 · Phase 1: 부서 협업팀 판별, 편성표+토큰 고지 브리핑 ⏸ → 승인
3. Phase 2~4: research·content·ads 3명 스폰(각자 전용 폴더·gbrain 규약 포함) → 작업 15개 분배 → 리더 모니터링
4. Phase 5: 산출물 종합 → gate-auditor 서브에이전트 검사 → 결과확인 ⏸ → build-log+gbrain 기록 → 해산
5. 검증 기준: 팀원별 폴더에 산출물 존재 · 공용 파일 미수정 · 브리핑/게이트 2회 ⏸ 발생

**에러 흐름 · 팀 기능 미가용 세션:**
1. 입력: "팀 소집" (env 미반영 세션)
2. Phase 0-1에서 팀원 스폰 불가 감지 → "새 세션에서 다시 불러달라" 안내 후 종료 (폴백: 서브에이전트 순차 실행 제안)
3. 검증 기준: 팀 스폰 시도 없이 안내 메시지로 종료 · 산출물 폴더 미생성

**에러 흐름 · 팀원 작업 실패:**
1. 팀원 1명이 작업 중 막힘(도구 거부·데이터 없음) → 리더가 1회 재지정, 재실패 시 해당 작업 없이 진행
2. 검증 기준: 최종 종합 보고서에 누락 항목과 사유 명시 (조용한 누락 금지)
