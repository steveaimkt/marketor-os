# 마케팅팀 시작하자 · 팀별 사용 가이드

> **무엇을 하나**: 구축된 팀(사람+AI)을 **실제로 어떻게 굴리는지** 안내한다.
> 71명 전체가 아니라 **그 회사에 켜진 팀만** — team.md·roster.md 를 읽어 맞춤 가이드한다.
> 새 지식을 가르치는 게 아니라, "이 말을 치면 이 팀이 이걸 해준다"를 손에 쥐여준다.

## GATE 0 · 구축 확인 (자동)
`brand/org-map.md`·`roster.md`·`team.md` 상태를 읽는다.
- **미작성**이면 → "아직 팀이 구축되지 않았습니다. 먼저 **'마케팅팀 구축하자'**로 조직을 진단하고 팀을 만들어야 합니다." → 회사셋업으로 보냄.
- **작성됨**이면 → 아래 안내 시작.

말문:
> "{회사} 마케팅팀 사용 가이드입니다. 지금 켜진 팀은 {N}개예요.
> 각 팀이 뭘 하고, 어떻게 부르고, 사람과 어떻게 나눠 일하는지 하나씩 보여드릴게요."

## 1 · 팀 지도 (켜진 것만)
roster.md 에서 ✅ 켜진 부서·실무 에이전트만 그룹으로 묶어 1화면:
```
🗺 {회사} 마케팅팀 (사람 {N} + AI {M})
① {부서} — {한 줄 혜택} · 담당: {직원} + {에이전트}
② {부서} — ...
(끈 팀은 표시하지 않음)
```

## 2 · 팀별 사용 카드 (켜진 팀마다)
각 팀에 대해 아래 카드를 만든다 (roster·team.md·에이전트 description 기반):
```
▸ {부서/에이전트 한국어 역할}
  하는 일   : {무엇을 만들어 주나}
  부를 때   : "{자연어 트리거 문장}"  ← 명령어 아님, 말로
  사람 몫   : {직원이 검토·승인·판단하는 것}
  AI 몫     : {에이전트가 자동으로 하는 것}
  첫 실습   : "{지금 바로 해볼 한 문장}"
```
예:
```
▸ 콘텐츠 팀 (콘텐츠담당 + email-newsletter·content-publisher)
  하는 일 : 뉴스레터·SNS 초안 생성 → 예약 발행
  부를 때 : "이번 주 뉴스레터 써줘"
  사람 몫 : 톤 확인·최종 발행 승인(⏸)
  AI 몫   : 자료 수집·초안·예약
  첫 실습 : "지난주 트렌드로 뉴스레터 초안 하나 만들어줘"
```

## 3 · 공통 사용법 (모든 팀 공통)
- **자연어로 부른다** — 명령어 외울 필요 없음. "경쟁사 뭐 하나 봐줘" 처럼.
- **안전장치** — 발행·예산·집행은 반드시 사람 승인(⏸). 그냥 안 나간다.
- **팀 소집(하네스)** — 큰 업무는 "팀으로 돌려줘"로 여러 에이전트 협업.
- **막히면** — "매뉴얼 보여줘" / "뭐 할 수 있어".

## 4 · 첫 주 추천 루틴 (사람+AI)
그 회사 진단(org-map §2 갭)에 맞춰 첫 주에 돌려볼 3가지 제안:
```
□ {갭1 해소} — "{트리거}"  (사람: {몫})
□ {갭2 해소} — "{트리거}"
□ 주간 브리핑 — "오늘 마케팅 브리핑"
```

## 완료
> "이제 팀이 돌아갑니다. 위 카드의 '부를 때' 문장을 그대로 쓰시면 됩니다.
> 팀·인력이 바뀌면 '마케팅팀 구축하자'로 다시 진단·조정할 수 있어요."

> 팀 편성 근거: `brand/roster.md`·`team.md` · 조직 진단: `brand/org-map.md`.
