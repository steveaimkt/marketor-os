# 스티브의 뉴스레터 작성 스킬

> Marketing Trueman Show / Marketor 의 주간 정보형 뉴스레터 패턴.
> "직접 해봤다 → 비용·시간 절감을 숫자로 증명 → 단계별로 전부 공개" 구조가 핵심.

## 언제 호출하나
- 스티브가 새 회차 뉴스레터를 쓸 때 (AI 툴 세팅, 마케팅 자동화, 클로드 코드 활용 등)
- 영상 콘텐츠를 레터로 변환할 때 (영상 = 상세, 레터 = 핵심 + 후킹)
- 사용자가 직접 `claude "Skill steve-newsletter 로 ... 작성"`

## 입력 (parameters)

```yaml
topic:                 # 필수 · 이번 회차 핵심 주장 한 줄
  # 예: "맥미니 안 사도 VPS 한 대로 24시간 클로드 코드 비서를 만들 수 있다"

number_contrast:       # 필수 · 후크의 엔진. 비싼 것 ↔ 싼 것/빠른 것 대조쌍
  before: "맥미니 M4 119만 원 일시불"
  after:  "호스팅어 VPS 월 1.3만 원"
  extra:  "3년 누적 58만 원 (맥미니의 절반 이하)"

body_sections:         # 필수 · ❶❷❸❹ 본문 (보통 4개, 3~5개 권장)
  - "왜 다들 맥미니부터 고민할까 (문제 제기)"
  - "VPS 한 대로 끝내는 구조 + 비용 대조"
  - "5분 설치 + 단계별 봇 연결 (실행 가이드)"
  - "Before / After + 보안 한 줄 (정리·주의)"

authority:             # 필수 · '직접 해봤다' 근거 + 경력 신뢰
  proof: "제가 직접 세팅해봤고 오늘 전부 공개합니다"
  bio:   "대학내일 → 뷰티브랜드 파트장 → 애플코리아"

primary_cta:           # 필수 · 1차 CTA 1개만 (영상/제휴 중 1순위)
  text: "VPS 세팅 영상 보러가기"
  url:  "https://..."
  coupon: "TRUMAN (10% 추가 할인)"   # 선택

secondary_cta:         # 선택 · About 뒤에 시각적으로 분리 (교육 홍보 등)
  text: "마케터를 위한 AI 교육 보러가기"
  url:  "https://..."

target_readers:        # 선택 · 추천 대상 4~5개 리스트
  - "클로드 코드를 24시간 돌리고 싶은데 맥미니가 부담인 분"

date_label: "2026년 5월 기준"   # 필수 · 가격·버전 정보의 기준 시점 (시의성 보호)
```

## 출력 (7블록 구조 · 받아쓰기 좋게 고정)

```yaml
subject:        # ① 후크 제목 · 부정 명령형 + 결과 (2줄 허용)
  # "맥미니 119만 원 안 사도 됩니다 / VPS 한 대로 끝낸 24시간 클로드 코드"

preheader:      # ② 프리헤더 1줄 (60자 이내, 제목과 중복 금지) ★ 자주 빠뜨림
  # "80→119만 원 인상… 월 1.3만 원으로 끝내는 법, 5분 설치부터 공개"

tldr:           # ③ TL;DR 인트로 · 한 문단에 전체 요약 + 권위 + "오늘 전부 공개"

toc:            # ④ 목차 ❶❷❸❹ (body_sections 그대로)

body:           # ⑤ 번호 본문 · 섹션별 소제목 + 본문, 숫자·구체 + 출처 표기

key_summary:    # ⑥ 핵심 요약 · Step 1·2·3 재압축 (스캔 독자용 2차 진입점)

cta_block:      # ⑦ 1차 CTA (강조) → 구분선 → About + 2차 CTA (작게 분리)
```

## 글쓰기 원칙 (스티브 시그니처 · 항상 지킴)

1. **부정 명령형 후크** · "~ 안 사도 됩니다", "~ 하지 마세요". 통념을 먼저 깬다.
2. **두괄식 TL;DR** · 첫 문단에서 결론·비용·약속을 다 끝낸다. ("한 줄로 요약하면…")
3. **번호 본문 ❶❷❸❹** · 본문은 반드시 번호 섹션. 스캔 가능하게.
4. **숫자로 증명** · "더 싸다" → "119만 원 vs 월 1.3만 원, 3년 58만 원". 모든 주장에 숫자.
5. **직접 해봤다 권위** · "제가 직접 세팅해봤고" + 경력 스토리로 신뢰 담보.
6. **CTA는 1차 1개만** · 1차 CTA(영상/제휴)를 강조, 2차(교육)는 About 뒤로 시각 분리. 한 통에 목표 1개.
7. **책임감 한 줄** · 리스크·보안(토큰·allowlist 등)을 솔직히 경고. 신뢰 ↑.
8. **가장 자주 빠뜨리는 단계 강조** · 실행 가이드엔 "⚠️ 여기를 빼먹어 30분 날리는 분 많다"를 1개 박아 실전감.
9. **기준 시점 라벨** · 가격·버전 정보엔 `date_label` 명시. 아카이브/재발행 시 오정보 방지.

## 후크 제목 패턴 (5종 중 택1~2)

| 패턴 | 예시 |
|---|---|
| **부정 명령형** (시그니처) | "맥미니 119만 원 안 사도 됩니다" |
| **비용 대조** | "월 1.3만 원이 119만 원을 이기는 계산법" |
| **반전·통념 깨기** | "노트북 닫으면 끝? 그게 진짜 문제가 아닙니다" |
| **결과 약속** | "휴대폰 하나로 24시간 일하는 비서 만드는 법" |
| **데이터 헤드라인** | "구독자 N명이 따라한 5분 세팅" |

## 검증 (레터 발송 전 자체 점검)

- [ ] 제목이 부정 명령형/대조형 후크인가
- [ ] **프리헤더가 있고, 제목과 중복되지 않는가** (★ 6개 보완점 ①)
- [ ] TL;DR 한 문단에 결론·숫자·약속이 다 들어갔는가 (두괄식)
- [ ] 본문이 번호 섹션(❶❷❸❹)으로 스캔 가능한가
- [ ] 모든 핵심 주장에 숫자가 붙었는가
- [ ] **1차 CTA가 정확히 1개로 강조되고, 2차 CTA는 분리됐는가** (★ ②)
- [ ] 실행 가이드에 "가장 자주 빠뜨리는 단계" 1개를 강조했는가
- [ ] 가격·버전 정보에 기준 시점(date_label)이 명시됐는가 (★ ⑤)
- [ ] 보안/리스크 경고 한 줄이 있는가
- [ ] 추천 대상 리스트가 명확한가
