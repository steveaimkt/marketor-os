# pm-value-proposition · 가치제안

marketing-os는 실행·기억·협업은 강하나 "왜 이 고객이 우리 걸 선택하는가"를 고객 관점에서 한 문장으로 벼리는 힘이 약하다. 이 스킬은 Jobs-to-Be-Done 6단계 템플릿으로 고객(Who·Why)에서 출발해 제품(How)으로 수렴하며, 마케팅·세일즈·온보딩에 바로 쓸 가치제안 문장과 포지셔닝 문장을 낸다. 제품부터 시작하는 접근과 달리 "고객 우선" 순서가 핵심이다.

## 언제 호출하나

- "가치제안 만들어줘"·"밸류 프로포지션 잡자"·"이 상품 한 줄 가치"·"포지셔닝 문장 써줘"·"왜 우리 걸 사야 하지"
- 신규 상품·교육 과정·커뮤니티 티어·컨설팅 패키지의 핵심 메시지를 벼릴 때
- 랜딩·상세페이지·모집 카피의 상위 메시지가 필요할 때 (landing-copy·ad-copy-ab의 입력이 됨)

경계: 사업 전체 구조는 pm-business-model, 초기 사업 가설 압축은 pm-lean-canvas다. 이 스킬은 그 캔버스의 Value Propositions 블록을 단일 세그먼트에 대해 문장 수준까지 깊게 판다. 톤·어휘 규범 자체는 brand-voice 스킬이 정본이며, 이 스킬은 그 톤으로 "무엇을 말할지"를 정한다.

## 입력

착수 전 아래를 확인한다. gbrain·프로필에 있으면 다시 묻지 않는다.

1. 대상 세그먼트 (하나로 좁힌다. 여러 개면 각각 별도 실행)
2. 제품·서비스 (WMBB 교육·컨설팅·커뮤니티·신상품 중 무엇)
3. 알고 있는 고객 페인·현재 쓰는 대안
4. 경쟁·대안 (직접 경쟁 + "아무것도 안 함"도 대안)

착수 시 먼저 `mcp__gbrain__recall` 또는 `search`로 태그 `value-prop`(및 관련 `voc`·`brand`)을 회수해 기존 VoC·페르소나·브랜드 보이스를 읽는다 (read-back).

## 워크플로(6단계 JTBD)

같은 입력은 같은 구조를 낸다. 고객에서 출발해 제품으로, 다시 문장으로 수렴한다.

1. **Who** · 타깃 세그먼트를 특성까지 프로파일한다 (누구를 위한 것인가).
2. **Why** · JTBD 프레이밍으로 핵심 문제와 원하는 결과(job)를 정의한다 (왜 이걸 필요로 하는가).
3. **What Before** · 현재 상태의 마찰·부족한 대안을 매핑한다 (지금은 어떻게 견디고 있나).
4. **How** · 제품 기능을 문제 해결과 경쟁 우위에 연결한다 (우리가 어떻게 푸는가).
5. **What After** · 개선된 미래 상태·새로 열리는 가능성을 그린다 (쓰고 나면 무엇이 달라지나).
6. **Alternatives** · 대안과 비교하고 전환 근거를 정리한다 (왜 대안 대신 우리인가).
7. **가치제안 문장 합성** · 1~2문장으로 벼린다. 마케팅·세일즈·온보딩에 즉시 쓸 수 있어야 한다.
8. **포지셔닝 문장 개발** · 대안 대비 차별을 담은 마케팅용 문장 (예: "X가 필요한 Who에게, 우리는 대안과 달리 Y를 준다").

### ⚠️ 안티패턴 (반드시 피한다)

- **제품부터 시작** · 기능 나열 → 고객 갖다붙이기는 이 프레임의 반대다. Who·Why를 먼저 확정한다.
- **세그먼트 안 좁힘** · "모두를 위한 가치"는 누구도 안 산다. 한 세그먼트로 좁히고, 여럿이면 나눠 실행한다.
- **What After에 기능 나열** · 개선된 미래는 고객이 얻는 결과·감정이지 스펙 목록이 아니다.
- **대안 무시** · "아무것도 안 함"을 포함한 대안 대비 우위가 없으면 전환 근거가 약하다.

### 예시 (Steve 비즈니스 · 발췌)

> 세그먼트: 실행이 막힌 1인 마케터 / 제품: WMBB 교육+커뮤니티

- **Who**: 혼자 마케팅을 도맡지만 도구·동료·피드백이 없어 실행에서 멈추는 1인 마케터
- **Why(job)**: "배운 걸 실제 성과로 연결하고 싶다"
- **What Before**: 강의만 여러 개 듣고 실행 못 함·질문할 곳 없음·혼자라 멈춤
- **How**: 실행 중심 교육 + 24시간 마케팅 OS + 167명 동료 커뮤니티 + 피드백
- **What After**: 배운 다음 날 바로 돌리는 자기 마케팅팀·멈추지 않는 실행 리듬
- **가치제안 문장**: "혼자 배우다 멈추던 마케터가, 배운 다음 날 바로 실행하는 24시간 마케팅팀을 갖게 된다."

## 산출물

1. **로컬 파일** · `outputs/{날짜}/pm-value-proposition/{날짜}-vp-{세그먼트}.md` — 6단계 표 + 가치제안 문장 1~2개 + 포지셔닝 문장 + 대안 비교.
2. **gbrain 적재** · `mcp__gbrain__put_page`로 본문 저장 후 `mcp__gbrain__add_tag` 태그 `value-prop` 부여. 관련 VoC·페르소나 페이지와 `add_link` 연결.
3. **노션** · 요청 시 노션 페이지/DB 행으로 미러링.
4. **검토 요약** · 무엇을·왜·한계(검증 안 된 페인·약한 대안 비교)를 3줄로 붙이고, "이 문장으로 갈까요? 더 날카롭게 할 부분은?" 피드백을 청한다.

대외 공개(랜딩·상세페이지·광고)로 나가는 문구는 승인 게이트(⏸)를 거친다. 산출 후 이 문장을 실제 카피로 확장하는 후속(landing-copy·ad-copy-ab·quality-review-6axis)을 제안한다.
