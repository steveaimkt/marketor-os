# 강의 녹음 시작 스킬

본 스킬은 marketing-os 강의 50개 클립의 녹화 환경을 1줄 명령으로 세팅한다. 사용자가 "녹음하자" 라고 하면 어느 클립인지 식별 → 슬라이드·대본을 한 번에 연다.

## 🎬 스킬 시작 시 메시지

본 스킬이 호출되면 Claude 는 우선 클립을 식별:

### Case A · 발화에 클립 번호 있음 (예: "1-3 녹음하자")

```
🎬 Part {N} 클립 {M} 녹화 환경을 준비합니다.
   - 슬라이드 자동 오픈
   - 대본 메타 확인
   - 촬영 체크리스트 출력
```

→ Step 2 로 즉시 진행.

### Case B · 발화에 클립 번호 없음 (예: "녹음하자")

```
🎬 어느 클립을 녹음할까요?

  형식: "Part-번호 클립-번호" 또는 "{N}-{M}"
  예시:
    - 1-3   (Part 1 클립 1-3 에이전트 이해)
    - 2-3   (Part 2 클립 2-3 Firecrawl MCP)
    - 5-2   (Part 5 클립 5-2 광고 카피 변형)
  
  또는 "다음 클립" 입력 시 직전 녹음 + 1.
```

→ 사용자 응답 받은 후 Step 1 클립 식별.

## Step 1. 클립 번호 식별

### Part 별 클립 번호 체계

| Part | 클립 번호 | 위치 패턴 |
|---|---|---|
| Part 1 입문 | 1-1, 1-2, 1-3, 2-1, 2-2, 3-1, 3-2 | `curriculum/part01-입문/대본/{N-M-slug}.md` + `{N-M-slug}/...` |
| Part 2 MCP | 1-1, 1-2, 1-3, 1-4, 2-1, 2-2, 2-3, 2-4, 3-1, 3-2, 4-1, 4-2 | `curriculum/part02-MCP12개/{NN-slug}/대본/{N-M-slug}.md` |
| Part 3~10 | 1, 2, 3 ... | `curriculum/part{NN}-*/{clip-folder}/대본/...` |

### Part 1 클립 매핑 (slides.html 위치)

```
1-1 → curriculum/part01-입문/대본/1-1-slides/slides.html
1-2 → curriculum/part01-입문/대본/1-2-slides/slides.html
1-3 → curriculum/part01-입문/대본/1-3-slides/slides.html
2-1 → curriculum/part01-입문/대본/2-1-slides/slides.html
2-2 → curriculum/part01-입문/대본/2-2-slides/slides.html
3-1 → curriculum/part01-입문/대본/3-1-slides/slides.html
3-2 → curriculum/part01-입문/대본/3-2-slides/slides.html
```

### Part 1 대본 매핑

```
1-1 → 1-1-AI가-마케터를-대체하는가.md
1-2 → 1-2-답하는AI-vs-일하는AI.md
1-3 → 1-3-에이전트-이해.md
2-1 → 2-1-Claude-Code-설치.md
2-2 → 2-2-마케터용-지시어-패턴.md
3-1 → 3-1-Claude-Code-7대-핵심기능.md

```

| 클립 | 폴더 |
|---|---|

| 1-2 | `01-google-sheets` |

| 4-1 | `10-notion` |
| 4-2 | `11-discord` |

각 클립 폴더 내 `대본/{N-M-slug}.md` + `대본/{N-M-slug}-slides/slides.html` 패턴.

### Part 3~10 클립 매핑

Bash 로 자동 탐색:

```bash
find "marketing-os/curriculum/part{NN}-*/" -name "*-slides" -type d
```

각 클립은 `{clip-folder}/대본/{N-slug}-slides/slides.html` 형태.

### 자동 식별 알고리즘

1. 사용자 발화에서 정규식 `\b(\d)-(\d)\b` 매칭 → `(Part 번호, 클립 번호)` 추출
2. 발화에 "Part N" 명시되어 있으면 Part 번호 우선
3. Part 번호만 있고 클립 번호 없으면 사용자에게 묻기
4. 클립 번호로 폴더·파일 경로 결정 (위 표 참조)

## Step 2. 슬라이드 자동 오픈

클립 경로 결정되면 Bash 로 즉시 실행:

```bash
SLIDES="/Users/steve/Library/Mobile Documents/com~apple~CloudDocs/Desktop/0.마케터를 위한 클로드코드/marketing-os/curriculum/{PART}/대본/{SLIDES_FOLDER}/slides.html"

# 파일 존재 검증
test -f "$SLIDES" || { echo "❌ 슬라이드 없음: $SLIDES"; exit 1; }

# 브라우저 오픈
open "$SLIDES"
echo "✅ 슬라이드 열림"
```

**Part 2+ 의 경우** 클립 폴더 안의 `대본/` 하위:

```bash
SLIDES="/Users/.../curriculum/part02-MCP12개/02-ga4/대본/1-3-slides/slides.html"
```

---

## Step 3. 대본 메타 확인 + IDE 노출

`Read` 도구로 대본 첫 50줄 읽기 → 메타 표(분량·슬라이드 수·시연 유무·핵심 한 줄) 추출 + 슬라이드 구성 표 추출.

대본 파일 경로는 사용자에게 명시:

```
📝 대본 파일: {경로}
   (IDE 에서 열어 두면 강사 모니터에 표시)
```

선택: 시연 가이드 파일도 있으면 같이 안내 (`{clip-folder}/대본/시연/{N-M-시연}.md`).

---

## Step 4. 촬영 체크리스트 + 메타 요약 출력

```
🎬 Part {N} 클립 {M} 「{제목}」 녹화 환경 준비 완료

📊 메타
  - 분량: {NN}분
  - 슬라이드: {N}장
  - 시연: {유/무}
  - 핵심 한 줄: "{대본의 핵심 한 줄}"

📂 파일
  - 슬라이드: {slides.html 절대경로} ← 브라우저 열림
  - 대본: {script.md 절대경로}
  - 시연 가이드 (있으면): {시연 파일 경로}

✅ 촬영 직전 체크리스트
  □ 슬라이드 검수 통과 ({slide-audit 실행 여부})
  □ 시연 환경 점검 (해당 시 · .env / MCP / 데이터)
  □ 녹화 도구 (OBS · 카메라 · 마이크) 준비
  □ 대본 마이크 옆 모니터에 띄움
  □ 슬라이드 전체화면 ⌘+Shift+F

▶️ 준비됐으면 녹화 시작. 끝나면 "다음 클립" 입력하면 자동 진행.
```

---

## Step 5. 직전 클립 기억 + "다음 클립" 지원

각 호출마다 직전 클립 번호를 메모리에 저장:
- 사용자가 "다음 클립 녹음" 입력 시 직전 번호 + 1 로 자동 추정
- Part 경계 (예: 1-3 → 2-1) 자동 처리
- 메모리 저장 형식: `project_last_recorded_clip.md` 또는 conversation context

다음 클립 추정 규칙 (Part 1):

```
1-1 → 1-2 → 1-3 → 2-1 → 2-2 → 3-1 → 3-2 → Part 2 1-1
```

다음 클립 추정 규칙 (Part 2):

```
1-1 → 1-2 → 1-3 → 1-4 → 2-1 → 2-2 → 2-3 → 2-4 → 3-1 → 3-2 → 4-1 → 4-2 → Part 3 첫 클립
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| `open` 명령 실패 | 파일 경로 오타·파일 없음 | `ls` 로 경로 확인. Part 폴더명 · 클립 폴더명 정확히 |
| 슬라이드 열렸지만 대본 자동 미노출 | IDE 가 자동 인식 못 함 | 대본 절대경로 출력 → 사용자가 ⌘+P 로 직접 오픈 |
| 클립 번호 모호 (예: "5-2") | Part 5 클립 2 vs 다른 Part 5-2 | 사용자에게 Part 번호 묻기 |
| 동일 클립 번호 두 Part 에 존재 | Part 1 1-3 + Part 2 1-3 (있을 경우) | 우선순위: 사용자가 직전 작업한 Part → 묻기 |
| 슬라이드가 검수 실패 | `slide-audit` Critical 발견 | 본 스킬은 일단 슬라이드만 열고, 사용자에게 검수 결과 알림 |

## 호출되는 다른 스킬·도구

| 시점 | 도구·스킬 | 역할 |
|---|---|---|
| Step 2 | `open` (macOS Bash) | 브라우저에 slides.html 표시 |
| Step 3 | `Read` (Claude 도구) | 대본 메타·구성 표 추출 |
| Step 4 (선택) | `slide-audit` 스킬 | 검수 미실행 시 자동 권유 |
| Step 5 | 메모리 시스템 | 직전 클립 기억 |

## 강의 연결

- 본 스킬은 강의 녹화 워크플로의 진입점.
- 마케팅 OS 의 50개 클립 모두 동일 패턴 (Part {NN}/{clip-folder}/대본/{slides}) 으로 작성되어 있어 본 스킬이 일관 작동.
- 시연이 있는 클립 (Part 2 MCP 설치 등) 의 경우 본 스킬이 시연 환경 (`.env`·`mcp.json`·서비스 계정) 점검까지 안내 가능 (향후 확장).
