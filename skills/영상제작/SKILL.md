# 영상 제작: Hyperframes 기반 풀 파이프라인

> Claude Code가 자연어 지시를 받아 HTML 컴포지션을 작성하고 mp4 로 출력하는 8단계 파이프라인. 마케터(비개발자) 입장에서 코드 한 줄도 직접 쓰지 않습니다.

---

## 🤔 시작 전: 왜 Hyperframes인가?

영상을 코드로 만드는 도구는 여럿 있습니다. 본 강의가 Hyperframes 를 선택한 이유:

| 평가축 | Manim | Remotion | **Hyperframes** ⭐ |
|---|---|---|---|
| 언어/스택 | Python | React + TypeScript | **HTML + CSS + GSAP** |
| 진입장벽 | 매우 높음 (LaTeX·OpenGL 의존) | 높음 (React 코드 직접 작성) | **낮음** (`npx hyperframes init` 한 줄) |
| 주력 콘텐츠 | 수학 설명 영상 전용 | 범용 영상 | **범용 + AI 친화** |
| Claude Code 통합 | 없음 | README에 언급 없음 (사용자가 따로 연결) | **공식 `.claude-plugin/` + 15개 스킬 내장** |
| AI 작성 흐름 | 사람이 코드 → AI 보조 | 사람이 React 코드 → AI 보조 | **AI 가 코드 → 사람은 지시만** |
| 라이선스 | MIT | 상업용 별도 라이선스 필요 (remotion.pro) | **Apache 2.0 (상업 무제한)** |
| 한글 폰트 지원 | 까다로움 | 가능하지만 설정 복잡 | **쉬움** (Pretendard CDN 한 줄) |
| 활성도 | 87k★ (마지막 릴리스 2024.12) | 47.4k★ (활발) | 19.6k★ (활발, HeyGen 운영) |
| **마케터 적합도** | ❌ 부적합 | △ 가능하나 어려움 | ✅ **최적** |

### 한 줄 결론

> **Remotion: "React 개발자가 코드로 영상 만드는 도구" → AI 가 보조**  
> **Hyperframes: "AI 가 영상 만드는 도구" → 사람은 디렉터**  
>
> 마케터(비개발자) 입장에서는 후자가 압도적으로 유리합니다.

### Hyperframes 의 한계 (정직하게)

이 도구로 못 만드는 것들 (강의에서 약속하지 않습니다):
- **페이스캠** (사람이 말하는 영상) → OBS 등 별도 촬영 필요
- **라이브 스크린 녹화** → Loom · QuickTime 등 별도
- **복잡한 3D 물리 시뮬레이션** (예: 로고 깨짐·폭발 효과) → Higgsfield MCP 또는 Blender 별도 생성 후 합성
- **사람 손 그림체 일러스트레이션** → Adobe Illustrator 또는 AI 이미지 생성기

만들 수 있는 것:
- **데이터 영상** (KPI 카드 · 차트 애니메이션 · 인포그래픽)
- **모션그래픽 오버레이** (자막 · 로우어서드 · 콜아웃)
- **키네틱 타이포그래피** (글자 등장·이동·강조)
- **슬라이드 → 영상 변환** (강의·발표 자료 재활용)
- **다이어그램 애니메이션** (플로우차트 · 게이지 · 다이얼)

## 📚 영상 품질의 80%는 슬라이드 구성에서 나옵니다

> 자동화 도구가 좋아도, 슬라이드 자체가 산만하면 영상도 산만합니다.  
> Hyperframes 는 "어떻게 보일지" 를 해결합니다. 사용자는 "무엇을 보일지" 만 정하면 돼요.

영상용 슬라이드를 잘 만드는 4가지 원칙:

### 원칙 1: 1 슬라이드 = 1 메시지

여러 메시지를 한 슬라이드에 넣으면 영상에선 다 못 읽고 다 못 들립니다.

❌ **나쁜 예** (정보 폭격):
```
이번 주 성과:
- ROAS 3.2x 달성 (전주 대비 +0.8)
- CTR 4.8%로 업종 평균 초과
- CPA 12,500원, 목표 대비 -15%
- 신규 캠페인 3건 런칭
- A/B 테스트 2건 완료
```

✅ **좋은 예** (5개 슬라이드로 분리):
```
슬라이드 1: "이번 주 핵심"
슬라이드 2: "ROAS 3.2x"
슬라이드 3: "↑ 전주 +0.8"
슬라이드 4: "CTR 4.8% / CPA 12,500원"
슬라이드 5: "다음 주 → A/B 결과 분석"
```

### 원칙 2: 시각 위계 1:1:3

한 슬라이드의 정보를 3 계층으로 정리:

| 계층 | 개수 | 예시 (KPI 슬라이드) | 권장 폰트 |
|---|---|---|---|
| **1차 (HERO)** | 1개 | "3.2x" (가장 큰 숫자) | 200~360px |
| **2차 (라벨)** | 1개 | "ROAS" (1차 정보의 맥락) | 40~80px |
| **3차 (보조)** | 최대 3개 | "+0.8 vs 전주" · "5/13~5/19" · "타겟 4.0" | 18~32px |

1차가 2개 이상이면 시청자 시선이 분산돼 메시지가 흐려집니다.

### 원칙 3: 4초 룰

평균 시청자는 한 슬라이드 핵심을 **4초 안에** 파악할 수 있어야 합니다.

| 항목 | 권장 한계 |
|---|---|
| 한 슬라이드 단어 수 | ≤ 25개 |
| 한 슬라이드 숫자 수 | ≤ 3개 |
| 한 슬라이드 색상 수 | ≤ 4개 (배경 포함) |
| 한 슬라이드 폰트 사이즈 종류 | ≤ 4개 |

체크 방법: 슬라이드를 4초만 보여주고 "뭐가 핵심?" 질문에 답할 수 있어야 OK.

### 원칙 4: 리듬

전체 영상에 같은 페이싱이 반복되면 시청자가 지루해합니다. **빠른-빠른-느린-임팩트** 패턴 권장.

예) 53초 영상 (9슬라이드):

```
[빠] Cover    5초  · 인트로
[빠] Manifesto 6초 · 핵심 메시지 (마커 강조)
[빠] Case ① 성공  6초
[느] Before/After  6초 · 데이터 시각화 천천히 차오름
[빠] Case ② 실패  6초
[빠] AI vs Human  6초
[★] Multiplier  7초 ⭐ 영상 최대 모먼트 (광선 효과)
[느] Summary     6초 · 게이지 순차 채움
[빠] Closing      5초 · 페이드아웃
```

**임팩트 슬라이드는 전체의 10~15% 만**. 너무 많으면 임팩트가 사라집니다.

## 종이 vs 영상 슬라이드의 결정적 차이

PowerPoint·PDF·Keynote 식 슬라이드를 그대로 영상으로 변환하면 대부분 실패합니다. 다음 표를 의식하고 슬라이드를 다시 손봐야 합니다:

| 종이/PDF 슬라이드 | 영상 슬라이드 |
|---|---|
| 한 페이지 정보 **많이** | 한 씬 정보 **적게** (4초 룰) |
| 시청자가 **다시 읽을 수 있음** | **한 번 지나가면 끝** (스크롤백 불가능) |
| 글자 18~24px OK | **모바일에서도 읽혀야** (최소 24px, 메인은 60px+) |
| 좌→우 자연스럽게 읽힘 | **위→아래** 또는 **중앙 정렬** 권장 |
| 정적, 시선이 페이지 위 자유 이동 | **등장 애니메이션 필수**, 시선 유도 강제 |
| 발표자가 보충 설명 | **자체 완결**, 자막·음성 없으면 텍스트가 다 |
| 한 페이지 = 한 주제 (광범위) | 한 슬라이드 = 한 메시지 (구체적) |

### 변환 시 체크리스트

기존 슬라이드를 영상화하기 전 다음을 확인:

- [ ] 한 슬라이드의 글자가 25단어 넘는 곳 → **분할** 또는 **축약**
- [ ] 본문 폰트 24px 미만 → **확대**
- [ ] 같은 슬라이드에 1차 정보(큰 숫자/타이틀) 2개 이상 → **분리**
- [ ] 표·차트가 9칸 이상 → **요약**하거나 **여러 슬라이드로 분리**
- [ ] 좌측 정렬 글머리표 5개 이상 → **타일/카드** 레이아웃 권장

## 📦 STEP 1: 환경 점검 (1회만, 이미 설치된 경우 스킵)

```bash
cd "/Users/{사용자}/{...}/marketing-os/hyperframes"
npx hyperframes doctor
```

체크 항목:
| 항목 | 필수 | 미설치 시 안내 |
|---|---|---|
| Node.js ≥ 22 | ✓ | nvm 설치 후 `nvm install 22` |
| FFmpeg / FFprobe | ✓ | 본 가이드 하단 "ffmpeg 한 줄 설치" 명령 안내 |
| Chrome | ✓ | https://google.com/chrome 다운로드 |
| Hyperframes 스킬 | ✓ | `npx skills add heygen-com/hyperframes` (1회) |

**ffmpeg 한 줄 설치 (sudo 불필요):**

```bash
mkdir -p ~/.local/bin && cd ~/.local/bin \
  && curl -L https://evermeet.cx/ffmpeg/getrelease/ffmpeg/zip -o ffmpeg.zip \
  && curl -L https://evermeet.cx/ffmpeg/getrelease/ffprobe/zip -o ffprobe.zip \
  && unzip -o ffmpeg.zip && unzip -o ffprobe.zip \
  && chmod +x ffmpeg ffprobe && rm -f ffmpeg.zip ffprobe.zip \
  && echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc \
  && source ~/.zshrc
```

모든 ✓ 면 Step 2 로.

## 🎯 STEP 2: 영상 기획

### 2.1 목적·플랫폼·길이 결정

Claude 가 다음을 질문:

> **어떤 영상을 만드시나요? 다음 4가지로 좁혀드릴게요.**
>
> **A. 콘텐츠 영상**: 유튜브 본편·강의 영상·인터뷰 (60초~10분 이상)
> **B. 광고 영상**: 인스타·페이스북·유튜브 광고 (15~60초)
> **C. 데이터 리포트 영상**: 주간 KPI·월간 광고 성과 (6~30초)
> **D. 인트로/아웃트로**: 채널 시그니처 (3~8초)
>
> 어느 쪽?

선택 후, **플랫폼**·**길이** 추가 질문:

| 플랫폼 | 권장 비율 | 권장 길이 |
|---|---|---|
| 유튜브 본편 / 광고 | 16:9 (1920×1080) | 60초~10분 |
| 유튜브 쇼츠 | 9:16 (1080×1920) | 15~60초 |
| 인스타 피드 | 1:1 (1080×1080) | 15~60초 |
| 인스타 릴스 / 스토리 | 9:16 (1080×1920) | 15~60초 |
| 틱톡 | 9:16 (1080×1920) | 15~60초 |
| 페이스북 광고 | 1:1 또는 4:5 (1080×1350) | 15~60초 |

### 자동 적용되는 패턴

| 원칙 | 적용 |
|---|---|

**도형/그래픽**

**고급 (별도 스킬 호출)**
- GSAP 타임라인 체이닝 (`/gsap`)

- Anime.js (`/animejs`)
- Lottie 애니메이션 (`/lottie`)

- Web Animations API (`/waapi`)

**미디어**
- 동영상 합성 (muted + separate audio)
- TTS 내레이션 (`/hyperframes-media` 의 Kokoro)
- 자막 / 캡션 (whisper 트랜스크립트 → 단어별 동기화)

## 트러블슈팅 (자주 막히는 곳)

| 증상 | 원인 | 해결 |
|---|---|---|
| `No deterministic font mapping` | Pretendard 가 @font-face 없이 사용됨 | `index.html`에 6 weights woff2 임베드 |
| 한글 박스 깨짐 (Linux/Docker 렌더) | 시스템에 Hangul 폰트 없음 | `fonts/` 폴더에 Pretendard .woff2 배치 + `@font-face` 로 로컬 참조 |
| `overlapping_clips_same_track` | 같은 트랙 인덱스 + 시간 겹침 | 각 clip 에 다른 `data-track-index` 부여 |
| `composition_self_attribute_selector` 경고 156개 | `[data-composition-id="X"]` 선택자 다수 사용 | 비차단 경고. 동일 컴포 두 번 임베드 안 하면 무해. 정리 원하면 `#X` 로 변경 + 래퍼에 `id="X"` 추가 |
| 텍스트 오버플로 (text-shadow 글로우 때문) | 인스펙터가 글로우 외곽까지 측정 | 해당 요소에 `data-layout-allow-overflow`, 또는 호스트 div 에 `data-layout-ignore` |
| 렌더 너무 느림 | 워커 수 부족 | `npx hyperframes render --concurrency 8` |
| Chrome 못 찾음 | 시스템에 Chrome 없음 | Google Chrome 설치 또는 `--browser chromium` |
| 변수 변경했는데 반영 안 됨 | 캐시 | dev 서버 재시작 또는 `--force` 플래그 |
