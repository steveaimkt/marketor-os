# 지표 대시보드 (Metrics Dashboard)

*Lean Analytics*의 지표 위계·평가 기준을 marketing-os용으로 각색한 스킬. 보이는 숫자를 아무거나 나열하는 대시보드 대신, **전략 → 북극성 → 입력지표 → 가드레일**로 이어지는 하나의 논리를 세워 "이 숫자가 바뀌면 우리가 무엇을 다르게 할 것인가"에 답할 수 있게 만든다.

## 언제 호출하나
- 사업·채널·캠페인의 성과를 무엇으로 볼지 지표 체계 자체가 없거나 흩어져 있을 때
- 대시보드에 숫자는 많은데 정작 행동으로 이어지지 않을 때 (허영 지표 솎아내기)
- `pm-okrs`·`pm-outcome-roadmap`의 성과 목표를 상시 추적 지표로 내려야 할 때
- 북극성 지표를 정하고 그것을 움직이는 선행·입력 지표를 역설계해야 할 때
- 사용자가 직접 `Skill pm-metrics-dashboard로 ... 대시보드 설계` 호출

## 입력

```yaml
north_star_candidate:   # 선택 · 없으면 워크플로 1단계에서 도출
  business_goal: "매출 5억(2026) → 100억(2030)"   # Steve 매출 북극성

domains:                # 선택 · 지표를 뽑을 사업 축 (있으면 입력지표 매핑에 사용)
  - "커뮤니티 (무료 리드)"
  - "교육 (첫 매출)"
  - "컨설팅 (객단가)"

data_sources:           # 선택 · 실제 연결 가능한 소스
  - "GA4 / Google Sheets / Notion / Buffer / 결제 데이터"

cadence_focus: "weekly" # 선택 · 이번 설계에서 집중할 리뷰 주기
```

- **좋은 지표 4기준** (Lean Analytics): ① 이해 가능 — 조직 공통 언어가 될 것 ② 비교 가능 — 스냅샷이 아니라 시계열·세그먼트로 볼 것 ③ 비율·비중으로 표현 — 절대값보다 더 많은 것을 드러냄 ④ 행동 유도 — "이 지표가 내 행동을 바꾸지 않으면 나쁜 지표다".

## 워크플로 (위계 설계 → 명세)

같은 사업 목표·같은 소스면 같은 지표 트리가 나오도록 위계와 명세 항목을 고정한다.

1. **북극성 지표 확정 (1개)** · 고객 가치와 매출을 동시에 선행하는 지표 하나만. 여러 개를 북극성으로 두지 않는다. 예: 매출 100억 궤도의 북극성은 절대 매출이 아니라 그것을 선행하는 **"교육·컨설팅 유료 전환 리드 수"** 같은 고객 중심 지표.
2. **입력지표 도출 (3~5개)** · 북극성을 직접 움직이는 운영 레버. 사업 축별로 하나씩 매핑: 커뮤니티 구독·활성 → 교육 전환율·교육 매출 → 컨설팅 수주. 각 입력지표는 담당 팀·에이전트가 실제로 손댈 수 있어야 한다.
3. **가드레일 지표 (헬스·비즈니스)** · 성장을 좇다 무너뜨리면 안 되는 하한선. 이탈률·CS 응답시간·객단가·현금·마진. 목표가 아니라 "이 밑으로 내려가면 경보".
4. **선행/후행 · 허영/행동 분류** · 모든 지표에 태그를 단다. 후행(매출·수료)만 있으면 학습이 느리므로 대응하는 **선행 지표**(신청 클릭·랜딩 전환·상담 예약)를 짝지어 채운다. 허영 지표(누적 조회수 등)는 격리하거나 비율(전환율)로 치환.
5. **지표별 명세** · 각 지표에 정의(분자·분모·기간)·데이터소스·시각화(라인·바·퍼널)·목표값·경보선을 채운다. 계산식이 모호하면 대시보드가 아니다.
6. **리뷰 주기 배치** · 일간=운영 헬스(오류·핵심 플로) · 주간=입력지표·인게이지먼트 추세 · 월간=북극성 진척·OKR 정합 · 분기=전략 재보정·프레임워크 검증. 각 지표를 해당 주기 슬롯에 배정.

## 산출물 (지표 트리 + 명세표)

지표 위계 트리와 명세표가 기본 산출물이다.

```
[북극성] 유료 전환 리드 수 (교육·컨설팅) · 월 · 매출 100억 궤도 선행
│
├─ [입력1·커뮤니티] 무료 구독자 순증          선행 | 목표 +200/월 | 경보 <+50 | 소스 Discord·Notion
├─ [입력2·교육]    무료→교육 전환율            후행 | 목표 8% | 경보 <4% | 소스 결제·시트 | 선행짝: 상세페이지 CTA 클릭률
├─ [입력3·교육]    교육 매출                    후행 | 목표 5천만/월 | 소스 결제
├─ [입력4·컨설팅]  컨설팅 수주 건수            후행 | 목표 3건/분기 | 선행짝: 상담 예약 수
│
└─ 가드레일: 유료 이탈률 <5% · CS 응답 <4h · 객단가 ≥ 유지 · 마진 ≥ 하한

명세 예 (입력2):
  정의: (해당월 교육 결제자) ÷ (해당월 무료 신규 구독자) · 월 윈도
  시각화: 라인(추세) + 목표선 · 경보: 2주 연속 4% 미만 시 조사

리뷰: 주간=입력1·2 추세 / 월간=북극성·입력3·4 / 분기=전략 재보정
분류 정리: 허영 격리 = 누적 유튜브 조회수(→ 조회수 대비 구독 전환율로 치환)
```

- 로컬 저장: `outputs/{날짜}/pm-metrics/{날짜}-dashboard-{북극성슬러그}.md`
- 기억 적재: `gbrain put_page`로 지표 트리 저장 후 `add_tag`로 `discovery`·`metrics` 태그. 다음 세션에서 `search`·`recall`로 read-back해 실제 수치가 쌓이면 목표·경보선을 재보정(매번 새로 만들지 않는다).
- 체이닝: 상위 성과 정의가 없으면 `pm-okrs`·`pm-outcome-roadmap`을 먼저 돌려 북극성 원료로 쓰고, 확정된 입력지표는 각 도메인 리포터(`ga4-analyzer`·`3media-integrated-reporter`·`ltv-analyzer`)에 추적 대상으로 넘긴다.
