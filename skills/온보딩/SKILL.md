# 마케팅 OS 온보딩 (게이트형 첫 완주)

> 이 스킬은 **체험으로 가르친다**. 설명을 쏟아붓지 않는다 · 게이트마다 딱 하나만 묻고, 답을 받으면 다음으로 간다.
> 완주자는 통합 OS의 5가지 핵심 동작을 전부 자기 손으로 한 번씩 해본 상태가 된다:
> **스킬 직접 실행 · 에이전트 위임 · 승인 게이트(⏸) 거절 · 체인 핸드오프 · 자동화 루틴 인지**

## Contract

1. **자동 실행 금지** · 게이트마다 ⏸ 질문 하나, 사용자 답을 받기 전에 다음 게이트로 넘어가지 않는다
2. 한 게이트 = 한 화면 = 한 결정. 선택지는 3개 이하
3. 중도 이탈해도 잃지 않는다 · 진행 상태를 [logs/onboarding-state.md](../../logs/onboarding-state.md)에 기록, 복귀 시 "GATE {n}부터 이어할까요?"
4. 브랜드 미작성이어도 막지 않는다 · 결로아 샘플(msk/data/samples) 폴백 + `[샘플]` 태그로 전 게이트 체험 가능
5. **사용자 대면 문장은 사장의 언어** · 영문 에이전트명·파일명·스킬 ID를 화면에 노출하지 않는다 (부서 직함은 한국어). `[샘플]`·`[추정]`·`⏸` 표기는 첫 사용 때 한 줄로 뜻을 설명한다
6. 튜토리얼이라는 이유로 실전 규칙을 생략하지 않는다 · 브리핑 게이트·품질 게이트·기록 전부 실전과 동일 (그게 교육 효과)
7. MSK 정본 참조만 · 스킬 실행 시 해당 SKILL.md를 Read해서 따르되, msk/ 파일을 복사·수정하지 않는다

## Gates (7관문)

### GATE 0 · 진단 & 경로 선택
[brand/profile.md](../../brand/profile.md) · [brand/my-playbook.md](../../brand/my-playbook.md) · logs/onboarding-state.md 상태를 파일로만 확인하고(재질문 금지) 인사:
```
👋 마케팅 OS 첫 완주 가이드입니다. 7개 관문, 25~30분이면 끝납니다.
이 OS는 팀장(트루먼) 1명 + 부서장 10명 + 실행 직원 37명 + 업무 카드 100장이 한 회사로 움직입니다.
지금 상태: 브랜드 프로필 {✅ 있음 / ⬜ 없음} · 지금까지 산출물 {n}건

어떤 경로로 갈까요?
1️⃣ 정석 완주 · 셋업부터 차례로 (처음이면 추천)
2️⃣ 샘플 급행 · 내 정보 입력 없이 가상 브랜드(결로아)로 전 과정 구경
3️⃣ 급한 업무 먼저 · 튜토리얼 접고 지금 할 일부터 (가이드는 나중에)
```
⏸ 답 대기. 3️⃣이면 트루먼 intake(orchestrator.md §4 자연어 라우팅)로 넘기고 종료 · 상태 기록: "가이드 보류".
중도 복귀 사용자면 이 화면 대신 "GATE {n}부터 이어할까요?"를 먼저 묻는다.

### GATE 1 · 브랜드 셋업 (3분, 또는 확인만)
- **profile.md가 이미 있으면 (확인 모드)**: 핵심 5줄(브랜드명·업종·대표 제품·핵심 고객·금기 표현)을 요약해 보여주고 "이대로 쓸까요, 고칠 항목 있나요?" · 재입력 강요 금지.
- **없으면 (경로 1️⃣)**: 딱 6개만 묻는다 · ①브랜드명 ②업종(규제 카테고리) ③대표 제품 1개 ④핵심 고객 한 줄 ⑤절대 쓰면 안 되는 표현 ⑥진행 중인 프로젝트 한 줄. ①~⑤는 brand/profile.md, ⑥은 brand/my-playbook.md에 기록.
- **경로 2️⃣**: "결로아(가상 화장품 브랜드) 샘플로 진행합니다 · 산출물마다 [샘플] 태그가 붙어요."

기록 후 멘트: "이제 스킬 100장과 직원 37명 전부가 이 정보를 재질문 없이 씁니다."
⏸ "다음 관문에서 첫 업무 카드를 직접 돌려봅니다. 준비되셨어요?"

### GATE 2 · 스킬 직접 체험 ("방법이 적힌 업무 카드")
```
첫 실행으로 뭘 뽑아볼까요? 셋 다 입력 파일 없이 됩니다:
1️⃣ 팔리는 검색어 TOP 20 · "우리 제품, 사람들이 뭐라고 검색하지?"
2️⃣ 광고 카피 10종 · 오디언스별 소재 세트 (규제 검사 장면까지 보여드림)
3️⃣ 한 주 콘텐츠 캘린더 · 채널·주제·퍼널 배치
```
⏸ 선택 → 대응 MSK 정본(1️⃣=001 · 2️⃣=043 · 3️⃣=021)의 SKILL.md를 Read → 담당 부서장 명의 브리핑 1화면(계획·예상 산출물·필요 입력) ⏸ → Contract·Phases대로 실행 → 산출물 전달 + outputs/{날짜}/onboarding/ 저장.
핵심 멘트: "방금 쓴 건 '방법이 적힌 업무 카드' 1장입니다. 이런 카드가 100장 있어요 · 전체 진열대는 msk/카탈로그.html."
⏸ "결과 어떠세요? 다음 관문에서는 같은 일을 '직원'에게 맡겨봅니다."

### GATE 3 · 에이전트 체험 ("도구를 든 직원에게 맡기기") ★통합 OS 고유
GATE 2 산출물과 이어지는 에이전트 1개를 **핸드오프 계약**(Context/Deliverable/Quality/Gate 4블록, _conventions.md §B)으로 소환한다:
- 1️⃣ 검색어를 뽑았다면 → SEO 리서치 담당(seo-keyword-research)에게 우선순위 시트化 위임
- 2️⃣ 카피를 뽑았다면 → 품질 채점 담당(quality-reviewer-6axis)에게 6축 채점 위임
- 3️⃣ 캘린더를 짰다면 → 콘텐츠 기획 담당(content-calendar)에게 3채널 Draft 위임

실행 전 한 줄 예고: "지금부터는 제가 하지 않고 담당 직원을 부릅니다 · 독립 작업 공간에서 자기 도구로 일하고 결과만 가져와요."
결과가 오면 차이를 명시적으로 짚는다: "스킬 = 방법(레시피), 에이전트 = 도구 든 직원. 1회성·가벼운 건 카드로, 정기 리포트·발송·병렬 작업은 직원에게 맡기면 됩니다."
⏸ "다음 관문은 이 OS의 안전장치 · 실제로 '거절'을 한번 해보실 거예요."

### GATE 4 · 안전장치 체험 ("결재권은 항상 사장에게")
방금 산출물로 **발송 직전 장면을 재현**한다: 디스코드 공유 초안(또는 메일 초안)을 만들어 보여주고 정식 승인 게이트를 띄운다:
```
⏸ 이 요약을 디스코드 채널로 발송할까요? (승인 전에는 아무것도 나가지 않습니다)
```
**사용자에게 "아니"라고 거절해보게 안내한다** · 거절 시 초안이 그대로 보류되는 것을 확인시킨다. (진짜 발송을 원하면 승인 후 발송해도 된다 · 어느 쪽이든 교육 목적 달성)
이어서 한 화면으로 3층 안전장치를 요약: ① 발송·발행·집행 = 항상 ⏸ 결재 ② 품질 채점(6축)은 기본값이 반려 · 수치·근거 없으면 통과 못 함 ③ 광고·건강기능식품 등 규제 발행물은 전담 감사관(⛔ 차단권)을 거침.
⏸ "안전장치 확인됐죠? 다음은 실전의 꽃 · 직원들 이어달리기입니다."

### GATE 5 · 체인 체험 ("이어달리기 · 앞 결과가 자동으로 입력")
GATE 2~3 산출물의 chains_to 추천 1개를 실제로 수락하게 한다 (예: 검색어→경쟁사 분석, 카피→심의 체크, 캘린더→블로그 글 1편).
실행 시 **피드포워드**를 명시적으로 보여준다: "방금 만든 {산출물}을 그대로 물려받았습니다 · 다시 설명 안 하셔도 돼요."
이어서 한 줄: "이걸 10단계로 묶은 게 커맨드예요 · '시장리서치 돌려줘'처럼 '~돌려줘'라고 하면 여러 직원이 알아서 이어달립니다."
⏸ "마지막 관문 · 잠든 사이에도 돌아가는 자동화를 보여드립니다."

### GATE 6 · 자동화 안내 & 졸업
클라우드 루틴을 소개한다 (실행이 아니라 인지가 목표):
```
이 OS는 노트북이 꺼져 있어도 클라우드에서 스스로 일합니다:
· 매일 아침 채널 브리핑 (06:00) · 매주 월 콘텐츠 기획 (07:00) · 매주 금 성장 감사 (18:00)
→ 결과는 디스코드로 도착. "지금 성장감사 돌려" 하면 즉시 실행도 됩니다.
폰 디스코드에서 트루먼을 바로 부를 수도 있어요 (디스코드 봇 연결 시).
```
졸업 화면:
```
🎓 완주! 오늘 직접 해본 것:
스킬 실행 ✔ · 에이전트 위임 ✔ · 승인 거절 ✔ · 체인 핸드오프 ✔ · 자동화 인지 ✔ (산출물 {n}건 적립)

이제 3가지만 기억하세요:
· 말한다 · "쿠팡 광고 분석해줘" (100장 중 알아서 찾아갑니다. 구체적일수록 잘 나와요)
· 고른다 · 제가 물어볼 때만 답하세요
· 결재한다 · 발송·발행 직전엔 반드시 ⏸ 허락을 받으러 옵니다

다음 한 걸음 (하나만 고르셔도 됩니다):
1️⃣ 내 업무 진단 (10분) · 100장 중 나에게 필요한 15장 + 주간 루틴을 골라줍니다
2️⃣ "에이전트 시작" · 트루먼 가동, 오늘 할 일 제안부터
3️⃣ 여기서 마침 · 막히면 언제든 "매뉴얼 보여줘"
```
⏸ 답 처리 (1️⃣ = MSK 091 진단 실행 / 2️⃣ = 트루먼 시작 모드 / 3️⃣ = 종료) → onboarding-state에 "완주" 기록.

## 상태 파일 규격 · logs/onboarding-state.md

```markdown
# 온보딩 진행 상태
- 최종 갱신: {YYYY-MM-DD HH:mm}
- 경로: {정석|샘플|보류}
- 완료 게이트: [0, 1, 2 ...]
- GATE 2 선택: {1|2|3} · 산출물: {경로}
- GATE 3 위임: {에이전트명} · 산출물: {경로}
- 상태: {진행중|보류|완주 (일시)}
```
게이트 하나가 끝날 때마다 갱신한다. 완주 후 재호출 시: "이미 완주하셨어요 · 다시 돌까요, 특정 관문만 복습할까요?"

## Anti-Patterns

- ❌ 게이트 두 개를 한 번에 진행하거나, 답 없이 다음 게이트 실행
- ❌ 첫 화면에서 에이전트 51개·스킬 100개·문서 전체를 나열 (선택지는 항상 3개 이하)
- ❌ profile.md에 있는 정보를 다시 묻기 (GATE 1에서도 기존 값은 확인만)
- ❌ 튜토리얼이라는 이유로 브리핑 게이트·품질 게이트·기록 생략 (실전과 동일 규칙)
- ❌ GATE 4에서 승인 없이 실제 발송 (거절 체험이 목적 · 발송은 사용자가 명시적으로 승인한 경우만)
- ❌ 경로 3️⃣(급한 업무)를 붙잡고 가이드 강요
- ❌ msk/ 정본 파일 복사·수정 (참조만)

## 활용

- **처음 사용자의 첫 세션**: 설치 직후 "온보딩 시작" 한 마디로 통합 OS 전체 개념(스킬·에이전트·게이트·체인·루틴)을 30분에 체득
- **팀 도입·강의**: 게이트 단위가 시연·촬영 클립 단위와 일치 · 데모 각본으로 그대로 사용
- **복습**: "GATE 3만 다시" 같은 부분 복습 호출 지원 (상태 파일 기반)
