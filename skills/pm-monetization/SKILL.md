# pm-monetization · 수익화 전략 설계

marketing-os는 콘텐츠·광고·리서치 실행은 강하나, "이 제품·서비스로 돈을 어떤 구조로 벌 것인가"를 여러 모델로 벌려놓고 비교·검증하는 힘이 약하다. 이 스킬은 그 빈틈을 메운다. 하나의 정답을 단정하지 않고 3~5개 모델을 벌린 뒤, 타겟 적합·단위경제·리스크로 걸러 저비용 검증부터 붙인다.

## 언제 호출하나

- "수익화 모델 어떻게 가져가지"·"돈 어떻게 벌지"·"수익구조 설계"·"이거 어떻게 돈으로 만들지"
- "구독 vs 단건"·"프리미엄 붙일까"·"멤버십/이너서클 만들까"·"사용량 과금 가능해?"
- 새 제품·교육 상품·커뮤니티·컨설팅 오퍼의 수익 모델을 처음 정할 때, 또는 기존 모델이 성장 정체일 때
- 경계: 이미 모델이 정해졌고 **가격 숫자·티어 설계**만 필요하면 이 스킬이 아니라 `pm-pricing`이다. 이 스킬은 "어떤 방식으로 돈을 받을 것인가(모델)"를, pm-pricing은 "얼마에, 몇 단계로 받을 것인가(가격)"를 다룬다. 자연스러운 체인: pm-monetization → pm-pricing.

## 입력

착수 전 아래를 확인한다. gbrain·프로필에 있으면 다시 묻지 않는다.

1. 대상 제품/서비스와 핵심 가치 (무엇을·누구의 어떤 문제를 해결하나)
2. 타겟 고객군과 지불 주체 (개인·기업·양면시장)
3. 수익 목표·기간 (예: 2026년 매출 5억 북극성 하위 목표)
4. 제약 (마진 하한·초기 자원·경쟁사 관행·브랜드 포지션)

착수 시 먼저 `mcp__gbrain__recall` 또는 `search`로 태그 `monetization` + 해당 상품명을 회수해 과거에 검토·기각한 모델과 그 이유를 읽는다 (read-back · §A). 같은 모델을 두 번 검토하지 않기 위해.

## 워크플로(수익화 설계)

같은 입력은 같은 구조를 낸다.

1. **모델 3~5개 브레인스토밍** · 아래 7개 코어 모델 풀에서 서로 겹치지 않는 3~5개를 고른다. 대부분 실제 제품은 하이브리드(예: 프리미엄 + 업그레이드, 구독 + 거래수수료)이므로 하이브리드도 하나의 후보로 벌린다.

   | 모델 | 한 줄 정의 | 적합 신호 |
   |---|---|---|
   | 프리미엄(Freemium) | 코어는 무료, 고급 기능만 유료 벽 뒤로 (전환율 통상 1~5%) | 바이럴·네트워크 효과, 낮은 한계비용 |
   | 구독(Subscription) | 반복 과금으로 지속 접근 | 지속적으로 가치가 갱신됨 |
   | 사용량 기반(Usage) | 사용량(호출·저장·거래 건수)에 비례 과금 | 사용량과 원가·가치가 연동 |
   | 좌석/시트(Seat) | 사용자 수당 과금 | 조직 단위 협업·팀 도입 |
   | 단건 구매(One-time) | 1회 선결제로 영구 접근 | 예측 가능·완결형 제품 |
   | 마켓플레이스/거래수수료 | 거래액의 %·정액 수수료 | 양면시장·중개 |
   | 광고/스폰서십 | 광고·브랜드 제휴 수익 | 대규모 무료 트래픽 |

2. **각 모델 7축 평가** · 후보마다: (1) 이름·설명 (2) 작동 방식(돈이 실제로 어떻게 들어오나) (3) 타겟 적합(우리 고객이 이렇게 낼까) (4) 단위경제(1건당 매출·원가·마진, 전환율 가정) (5) 리스크·난점 (6) 경쟁 위치(경쟁사 관행 대비 차별/불리) (7) 검증 실험(아래 4번).
3. **저비용 검증 실험 설계** · 각 모델을 완제품 없이 검증할 최소 실험을 붙인다 (랜딩·가격 페이지 A/B, 창업자 직접 판매 대화, 사전판매·대기자, 소액 파일럿). "완벽한 제품을 기다리지 말고 일찍·자주 검증한다."
4. **우선순위화** · 4기준으로 정렬: ①전략 적합 ②구현 용이성 ③검증 가능성 ④경쟁 우위. 각 모델을 이 4축으로 High/Mid/Low 표기.
5. **1~2개 추천 + 로드맵** · 먼저 테스트할 1~2개를 근거와 함께 지목하고, 각각의 성공 기준(숫자)·다음 액션·기각 조건(kill criterion)을 명시한다.

### ⚠️ 안티패턴 (반드시 피한다)

- **모델 1개만 단정** · 처음부터 "구독으로 간다"고 못 박으면 벌려보는 이득을 버린다. 최소 3개는 벌린다.
- **단위경제 없는 후보** · 마진·전환율 가정 없이 "좋아 보인다"로 통과시키지 않는다. 숫자 가정이 없으면 미완성.
- **검증 실험 누락** · 검증 없이 로드맵에 올리지 않는다. 모든 후보에 완제품 없이 돌릴 저비용 실험이 붙어야 한다.
- **추정을 실측처럼** · 전환율·지불의향은 대개 추정이다. `[추정]` 태그를 달고, 검증 전 확정 매출로 보고하지 않는다 (§G).

### 예시 (Steve 비즈니스 · 커뮤니티 유료 이너서클)

> 대상: 유튜브 유입 무료 커뮤니티(현 167명)를 유료 수익으로 전환

- 후보A · **구독(월 이너서클)**: 작동=월 정액 멤버십 / 적합=지속 인사이트·네트워크 갱신에 맞음 / 단위경제=[추정] 무료 167명 × 전환 5% × 월 5만원 = 월 42만 / 리스크=이탈률 / 검증=대기자 폼 + 유료 의향 설문 30명.
- 후보B · **단건(교육 상품)**: 작동=회차 강의 1회 결제 / 적합=완결형 학습 수요 / 단위경제=1인 30만 × 20명 = 600만/회 / 리스크=반복성 낮음 / 검증=사전판매 랜딩.
- 후보C · **하이브리드(무료 커뮤니티 + 유료 컨설팅 티어)**: 작동=무료로 신뢰 축적 → 상위 컨설팅 업셀 / 적합=퍼널 자연스러움 / 검증=무료 멤버 대상 진단 세션 5건 오퍼.
- 추천: B(단건 사전판매)로 지불의향 먼저 검증 → 성공 시 A(구독)로 반복매출화. 기각 조건: 사전판매 랜딩 전환 <2%면 오퍼 재설계.

## 산출물

- **로컬 (필수)** · `outputs/{날짜}/pm-monetization/{날짜}-{상품명}-monetization.md` — 후보 3~5개 7축 표 + 우선순위 + 추천 1~2개 + 성공/기각 기준
- **gbrain 적재 (필수)** · 작성 후 `mcp__gbrain__put_page`로 채택·기각 모델과 그 근거, 검증 실험·기각 조건을 저장. 태그: `monetization` + 상품명. 나중에 "그 모델 왜 접었지" 회수용.
- **Discord 요약 (필요 시)** · 추천 모델 1~2개 + 이번에 붙일 검증 실험만 짧게 발송.
- ⚠️ 가격표·오퍼 페이지 등 **대외 발행물로 넘어가면 발행 전 승인 게이트**(§D·§E) 통과 — 이 스킬은 내부 전략 산출까지만, 대외 노출은 사용자 ⏸승인 후.

이 스킬은 모델을 **설계·추천**만 한다. 가격 숫자·티어는 `pm-pricing`으로 넘긴다 (핸드오프 §B).
