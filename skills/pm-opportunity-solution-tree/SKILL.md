# 기회-해결 트리 (Opportunity Solution Tree · OST)

Teresa Torres의 discovery 프레임워크를 marketing-os용으로 각색한 스킬. "어떤 기능·캠페인을 만들까"로 곧장 뛰지 않고, **원하는 성과 → 고객의 기회(니즈·페인) → 해결책 → 실험**의 4계층으로 문제 공간을 먼저 펼친 뒤 검증 가능한 단위로 좁힌다.

## 언제 호출하나
- 성과 목표는 있는데 "무엇부터 손대야 할지" 막막할 때 (예: 전환율·리텐션·유료화를 올리고 싶은데 액션이 안 보임)
- VoC·리서치에서 페인 포인트는 모였는데 **해결책으로 정리가 안 될 때** (`voc-analyzer`·`research-lead` 산출물을 입력으로 받아 기회 계층에 배치)
- 아이디어가 너무 많아 우선순위가 필요할 때 (기회별 점수화로 상위 2~3개만 집중)
- 캠페인·기능·콘텐츠를 실행하기 전에 "이게 어떤 성과의 어떤 기회를 푸는가"를 한 장으로 검증하고 싶을 때
- 사용자가 직접 `Skill pm-opportunity-solution-tree로 ... 구조화` 호출

## 입력

```yaml
outcome:              # 필수 · 단 하나 · 측정 가능해야 함
  metric: "커뮤니티 무료→유료 전환율"
  current: "3%"
  target: "8% (분기 내)"

opportunities:        # 선택 · 없으면 워크플로 2단계에서 도출
  source: "voc-analyzer 산출물 / research-lead 브리프 / 인터뷰 메모"
  # 있으면 고객 발화 형태로: "나는 ...를 못 해서 ...가 불편하다"

constraints:          # 선택
  - "실험은 2주 내 저비용으로 검증 가능할 것"
  - "개발 리소스 최소"
```

- **핵심 원칙**: 기회 계층에는 **해결책이 아니라 고객의 니즈·페인**만 들어간다. "이메일 리마인더를 보내자"는 해결책이지 기회가 아니다. 대응하는 기회는 "나는 유료 혜택이 뭔지 몰라서 결제를 미룬다" 같은 고객 관점 문장이어야 한다.

## 워크플로 (OST 4계층)

같은 입력이면 같은 트리 구조가 나오도록 순서를 고정한다.

1. **성과 확정 (1개)** · 측정 가능한 원하는 성과 하나만 트리 최상단에 둔다. 여러 목표를 한 트리에서 풀지 않는다. 지표·현재값·목표값을 명시.
2. **기회 도출 (3~7개)** · 리서치·VoC·인터뷰에서 고객 니즈·페인을 추출해 고객 관점 문장으로 적는다. 해결책이 섞이면 "그건 해결책"이라고 되돌려 기회로 다시 쓴다.
3. **기회 우선순위 (상위 2~3개)** · 기회 점수 = **중요도 × (1 − 만족도)** (각 0~1 정규화). 데이터가 없으면 정성 판단으로 상위 2~3개만 남기고 나머지는 백로그로 뺀다.
4. **해결책 브레인스토밍 (기회당 3개 이상)** · 첫 아이디어에 정박하지 않도록 기회마다 서로 다른 접근 3개 이상을 낸다. (제품·디자인·마케팅 관점 교차)
5. **실험 설계 (해결책당 1~2개)** · 각 해결책의 가장 위험한 가정을 2주 내 저비용으로 검증할 실험을 붙인다. 가정 유형 4종(가치·사용성·수익성·실현가능성) 중 무엇을 검증하는지, 성공 기준 수치를 명시.
6. **트리 시각화 + 되돌림 규칙** · 4계층 전체를 계층 텍스트로 출력한다. 실험이 가정을 기각하면 그 해결책을 폐기하고 같은 기회의 다른 해결책으로, 기회 자체가 틀렸으면 상위 기회 계층으로 되돌아간다(iterative). 트리는 새 근거가 쌓일 때마다 주기적으로 갱신한다.

## 산출물 (트리 텍스트 / 다이어그램)

계층 텍스트가 기본 산출물이며, 시각 다이어그램이 필요하면 `diagram` 스킬에 이 트리를 넘겨 노드-엣지 그래프로 렌더한다.

미니 예시 (성과 = 커뮤니티 유료 전환율 향상):

```
[성과] 커뮤니티 무료→유료 전환율 3% → 8%
│
├─ [기회1] "나는 유료가 뭘 더 주는지 몰라서 결제를 미룬다"  (중요도0.9 × (1−만족0.3)=0.63 · 1순위)
│   ├─ [해결책A] 무료/유료 혜택 비교 1장 온보딩 카드
│   │   └─ [실험] 신규 167명 중 절반에게 카드 노출 → 2주 전환율 diff (성공: +3%p)
│   ├─ [해결책B] 유료 전용 콘텐츠 1건을 무료존에 티저로 공개
│   │   └─ [실험] 티저 CTA 클릭→결제 퍼널 (성공: 클릭자 결제율 10%)
│   └─ [해결책C] 온보딩 3일차 유료 후기 DM 자동 발송
│       └─ [실험] DM 수신군 vs 대조군 결제율 (성공: 유의차)
│
├─ [기회2] "가격이 내 상황에 맞는지 판단이 안 선다"  (0.48 · 2순위)
│   └─ [해결책D] 사용 목적별 3티어 제시 …
│
└─ [기회3] "결제 순간 이탈한다"  (0.30 · 백로그로 이연)
```

- 로컬 저장: `outputs/{날짜}/pm-ost/{날짜}-ost-{성과슬러그}.md`
- 기억 적재: `gbrain put_page`로 트리 저장 후 `add_tag`로 `discovery`·`ost` 태그. 다음 세션에서 `search`·`recall`로 read-back해 이어서 갱신(같은 성과의 트리를 매번 새로 만들지 않는다).
- 체이닝: 기회 입력이 없으면 `voc-analyzer`(리뷰·CS 페인 5개)·`research-lead`(시장 브리프)를 먼저 돌려 기회 계층 원료로 쓴다. 확정된 상위 해결책은 `content-calendar`·`landing-copy` 등 실행 에이전트로 넘긴다.
