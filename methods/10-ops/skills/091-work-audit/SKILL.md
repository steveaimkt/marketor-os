# 091 마케팅 업무 자동화 진단

marketing 100 skills 의 **온보딩 진입점**. "뭐부터 자동화하지?" 에 문답으로 답한다: 업무 인벤토리 → 효과×난이도 매트릭스 → 30일 도입 로드맵 → brand/profile.md 질문서 연결. (에이전트구축 스킬의 진단 매트릭스부 계승)

## Contract
1. 진단은 사용자의 **실제 주간 업무**에서 출발한다 — 일반론 업무 목록을 임의로 가정하지 않는다.
2. 모든 업무는 효과(절감 시간×빈도) × 난이도(정형화·판단 개입) 2축 4분면에 배치되고, 우상단 "지금 바로 ★" 사분면이 1순위가 된다.
3. 추천 1순위는 항상 **한 입 크기**다 — 전 과정 자동화가 아니라 가장 자주·가장 아픈 한 스텝 (범위 게이트).
4. 로드맵의 모든 항목은 marketing 100 skills 스킬 ID로 매핑된다 — "자동화하세요" 가 아니라 "이 스킬을 이 순서로".
5. 진단 종료 시 brand/profile.md 상태를 확인하고, 미완이면 온보딩 질문서로 반드시 연결한다 (OS 전체의 전제 조건).

## Phases
1. **현황 로드** — brand/profile.md 를 읽어 업종·채널 현황·팀 상황을 파악한다 (이미 기록된 항목은 재질문 금지). 미작성 상태면 "진단 먼저, 프로필은 마지막에" 로 안내하고 진행.
2. **업무 인벤토리 (발굴 문답)** — 3개 질문으로 반복 업무를 꺼낸다:
   - "매주 여는 파일/시트는?" · "챗봇에 붙여넣었다가 결과를 다시 옮기는 일은?" · "마감·발행 전에 몰아서 하는 손작업은?"
   - 각 업무를 `{업무명, 빈도, 1회 소요, 사용 도구, 아픈 정도(1~5)}` 로 표에 채운다 (5~12개 목표).
3. **자동화 매트릭스** — 2축 점수화 후 4분면 배치:
   - 효과 = 빈도 × 1회 소요 (주당 절감 가능 시간) / 난이도 = 정형화 정도 + 판단 개입 + 데이터 접근성
   - 4분면: ★지금 바로(고효과·저난이도) / 다음 라운드(고효과·고난이도) / 자투리(저효과·저난이도) / 보류(저효과·고난이도)
   - 상위 업무엔 그릇도 판정: 절차 고정 = 스킬 / 판단·다단계 = 에이전트 체인 / 상시 감시 = 스케줄러 결합.
4. ⏸ **1순위 확정 (범위 게이트)** — ★ 사분면에서 1순위 후보 2~3개를 제시하고 사용자가 선택한다. 선택 업무가 "캠페인 기획부터 발행까지" 같은 전 과정이면 가장 아픈 한 스텝으로 좁힌 뒤 확정.
5. **OS 도입 로드맵 (30일)** — 선택 업무 기준 3단계 로드맵을 스킬 ID로 작성:
   - Week 1 즉시 효과(선택 업무의 해당 스킬) → Week 2~3 확장(인접 체인 스킬 2~3개) → Week 4 상시 가동(100 오케스트레이션 결합)
   - 인벤토리의 나머지 업무도 대응 스킬 ID를 병기해 "다음 라운드 대기열" 로 남긴다.
6. **★ 플레이북 저장 — 진단을 상주 파일로** — 결과를 [brand/my-playbook.md](../../../../brand/my-playbook.md) 에 기록한다:
   - ①역할 프로필 ②내 스킬 셋(매트릭스 선별 — ★사분면 스킬을 우선순위로) ③주간 운영 루틴(요일별) ④다음 라운드 대기열
   - 이 저장이 이 스킬의 존재 이유다 — **진단이 세션과 함께 사라지면 실패.** 이후 오케스트레이터가 매 요청마다 이 파일을 보고 사용자 업무 기준으로 라우팅·추천한다.
7. ⏸ **brand/profile.md 온보딩 연결** — (/마케터OS셋업 경유 시 이 Phase는 커맨드가 대체 — 건너뜀) profile.md 미완 항목을 보여주고 "지금 5분 질문서로 채울까요?" 를 확인받는다. 승인 시 섹션 1~6을 문답으로 채워 저장 — 이후 100개 스킬이 이 파일을 자동 참조한다. 완료 후 092(나만의 스킬 만들기) 또는 100(오케스트레이션)으로 핸드오프.

## Anti-Patterns
- **일반론 진단** — "마케터는 보통 이런 일을 하죠" 로 가상 인벤토리를 채우지 않는다. 사용자의 파일·시트·툴 이름이 나와야 진짜다.
- **전 과정 한 방 자동화 추천** — "콘텐츠 기획~발행 전부 자동화하세요" 는 실패 설계. 한 스텝만 잘라 성공 경험 먼저.
- **점수 근거 숨김** — 효과·난이도 점수를 감으로 매기고 수치 근거(빈도×소요)를 생략하면 매트릭스가 장식이 된다.
- **스킬 매핑 없는 로드맵** — "2주차: 자동화 확대" 같은 문장 금지. 모든 행에 스킬 ID.
- **profile.md 건너뛰기** — 온보딩 연결 없이 진단만 끝내면 이후 모든 스킬이 브랜드를 재질문하게 된다 (규격 위반의 씨앗).
- **플레이북 미저장** — 진단 리포트만 출력하고 my-playbook.md 를 안 쓰면, 다음 세션의 오케스트레이터는 이 사용자를 다시 모른다.

## 활용
- **OS 첫 실행 (온보딩)**: 신규 사용자가 가장 먼저 돌리는 스킬 — 진단 → 1순위 확정 → profile.md 작성까지 한 세션에 끝내면 OS 셋업 완료.
- **분기 업무 리셋**: 분기마다 재진단해 "다음 라운드 대기열" 에서 새 자동화 대상을 승격 — 로드맵이 계속 굴러간다.
- **팀 도입 설득 자료**: 인벤토리의 주당 절감 시간 합계가 곧 도입 ROI 근거 — 리포트 그대로 팀장 보고에 사용.
