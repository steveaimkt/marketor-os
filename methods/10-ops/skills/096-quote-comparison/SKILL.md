# 096 경쟁 입찰·견적 비교기

총액 한 줄로 업체를 고르면 싼 견적이 제일 비싸진다. 견적서 생성 에이전트(안티그래비티)의 항목 테이블 구조 — **항목·내용·수량·단가·공급가액·비고 + VAT 포함/별도 산식** — 를 역방향으로 적용: 만들 때 쓰던 구조로 받은 견적을 해체해 항목 정렬 → 누락·과다 검출 → 협상 포인트를 도출한다.

## Contract
1. **총액만 비교하지 않는다** — 모든 견적을 표준 항목 테이블(항목·내용·수량·단가·공급가액·비고)로 정규화한 뒤 항목 단위로 비교한다.
2. VAT 기준을 통일한 뒤 비교한다 — VAT 포함가는 공급가액 = 총액 ÷ 1.1로 환산, 별도가는 부가세 10% 가산. 통일 전 수치로 우열 판정 금지.
3. 누락 항목은 "저렴함"이 아니라 **"계약 후 추가 청구 후보"**로 경고한다 — 업체별 미포함 항목과 예상 추가 비용 범위를 명시.
4. 협상 포인트는 복붙 가능한 구체 문안으로 낸다 ("A사에 'B사는 수정 3회 포함인데 귀사는 2회입니다. 3회로 맞춰주실 수 있나요?'").
5. 추천은 **가격 × 범위 커버리지 × 리스크** 3축 종합 — 최저가 자동 추천 금지. 근거 없는 시장 단가 언급은 `[추정]` 태그.

## Phases
1. **견적 파싱·정규화** — 각 견적서를 표준 항목 테이블로 옮긴다. VAT 포함/별도를 확인해 공급가액 기준으로 통일. 수량×단가 = 공급가액 검산 — 안 맞으면 ⚠️ (계산 오류 또는 숨은 조정).
2. **항목 정렬(매핑)** — 업체마다 다른 이름의 같은 작업을 한 행으로 정렬 ("디자인 시안비" = "크리에이티브 개발"). 094 브리프가 있으면 브리프의 산출물 명세를 정렬 기준 축으로 사용 — 없으면 정렬 후 ⏸ 매핑 확인을 받는다.
3. **누락·과다 검출** — 3종 스캔:
   - **누락**: 한 업체에만 없는 항목 (계약 후 추가 청구 1순위)
   - **과다**: 동일 항목의 단가가 타사 대비 +50% 이상, 또는 수량 부풀리기
   - **뭉뚱그림**: "일식(一式)", "기타 제반 비용" — 항목 분해 요구 대상
4. **3축 평가** — 업체별 {비교 가능 총액(VAT 통일·범위 보정), 범위 커버리지(브리프 대비 %), 리스크(수정 횟수·납기·원본 귀속·하자보수)}를 표로 판정. 범위 보정 총액 = 총액 + 누락 항목의 타사 평균 단가.
5. ⏸ **협상 포인트·추천 확정** — 업체별 협상 문안(항목 분해 요구·수정 횟수 상향·누락 항목 포함가 재견적)과 종합 추천 + 근거를 제시하고 사용자가 결정한다.
6. **체인 제안** — 견적 간 범위 편차가 커서 비교 자체가 안 되면 원인은 브리프 부재 — 094 외주 브리프 빌더로 돌아가 동일 브리프 재발송 후 재견적을 받는 루프를 제안.

## Anti-Patterns
- **총액 한 줄 비교** — "A사 300, B사 350이니 A사". B사에만 원본 파일 귀속이 포함돼 있다면 판정은 뒤집힌다.
- **VAT 기준 혼재 비교** — 포함가와 별도가를 그대로 나란히 놓으면 10%가 통째로 착시가 된다.
- **최저가 = 최선 판정** — 누락 항목으로 만든 최저가는 계약 후 추가 청구로 회수된다. 범위 보정 총액이 진짜 가격.
- **"일식" 방치** — 분해 안 된 일식 항목은 비교 불능 + 협상 불능. 분해 요구가 협상의 시작이다.
- **시장 단가 창작** — 근거 없이 "통상 이 작업은 100만원대"라고 단정하지 않는다. 근거가 타사 견적뿐이면 그렇게 명시, 그 외는 `[추정]`.

## 활용
- **094→096 발주 루프**: 094에서 만든 브리프로 3사 견적 수취 → 096 비교 → 협상 문안 발송 → 재견적 반영해 최종 계약 — 브리프와 견적이 같은 항목 축을 쓰므로 비교가 10분.
- **단독 견적 검증**: 견적이 1개뿐일 때도 항목 검증 모드로 — 뭉뚱그림 분해 요구 + 수정·귀속 조항 확인만으로 협상 여지가 나온다.
- **연간 대행 계약 갱신**: 기존 대행사 갱신 견적을 신규 2개사와 항목 정렬 비교 — "관성 계약"의 과다 항목이 표에서 드러난다.
