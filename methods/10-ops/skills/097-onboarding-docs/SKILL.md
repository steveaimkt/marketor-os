# 097 팀 온보딩 문서 자동화

"물어보면 알려줄게"는 문서가 아니다. 머릿속 업무를 **업무 목록 → 절차 → 계정·자산 → 비상 연락** 4부 표준으로 문서화한다. 백지에서 시작하지 않는다 — [brand/my-playbook.md](../../../../brand/my-playbook.md)의 주간 루틴·스킬 셋과 [logs/build-log.md](../../../../logs/build-log.md)의 실산출물 이력에서 실업무를 자동 추출하고, 인터뷰는 빈 곳만 채운다.

## Contract
1. 매뉴얼은 **신입이 첫 주에 혼자 실행 가능한 수준**으로 쓴다 — 업무마다 목적·주기·절차 단계·완료 기준·막힐 때 대처가 있어야 완성.
2. 실업무 목록은 my-playbook(§2 스킬 셋·§3 주간 루틴)과 build-log(원장 최근 기록)에서 **먼저 추출**한다 — 문서에 없는 업무만 인터뷰로 보완 (백지 인터뷰 금지).
3. 계정·자산 목록은 위치·권한·관리자만 기록한다 — **비밀번호·API 키 등 민감 정보는 평문 기재 금지**, "패스워드 매니저 {볼트명} 참조" 방식으로 마스킹.
4. 절차에는 명시 단계뿐 아니라 **암묵지**(판단 기준·예외 처리·"이럴 땐 이렇게")를 최소 1개씩 포함한다 — 인수인계 사고의 대부분은 여기서 난다.
5. 모든 문서에 갱신 주기·갱신 담당·최종 갱신일을 명시한다 — 없으면 6개월 뒤 죽은 문서가 된다.

## Phases
1. **실업무 자동 추출** — my-playbook §3 주간 루틴(요일별 업무)과 §2 스킬 셋(업무 매핑), build-log 원장의 최근 4~8주 산출물을 집계해 업무 목록 초안을 만든다: {업무명, 주기, 사용 스킬/툴, 최근 실행 근거}. my-playbook 미작성이면 091 업무 진단 선행을 제안 (또는 전체 인터뷰 모드로 진행).
2. ⏸ **보완 인터뷰** — 초안을 보여주고 빈 곳만 묻는다: 초안에 없는 업무(월간·비정기), 업무별 암묵지("이 리포트에서 제일 자주 틀리는 게 뭐예요?", "이 승인은 누가 안 되면 누구에게?"), 대상 독자(신입 온보딩 vs 인수인계 vs 외주 위임).
3. **절차화** — 업무별 표준 카드 작성: {목적 1줄 / 주기·소요 / 절차 단계(입력→처리→산출→전달) / 사용 스킬·툴 / 완료 기준 / 자주 막히는 곳 + 대처}. 반복 업무 중 스킬 ID로 대체 가능한 단계는 스킬 ID를 병기 — 매뉴얼이 곧 OS 사용법이 된다.
4. **계정·자산 인벤토리** — {툴/서비스, 용도, 계정 위치(마스킹), 권한 레벨, 관리자, 결제 주기·카드}. 결제 계정과 소유권(개인 계정에 물린 회사 자산 ⚠️)을 반드시 점검.
5. **비상 연락·에스컬레이션 + 갱신 규칙** — {상황 → 1차 대응 → 에스컬레이션 대상 → 연락 수단} 표 + 문서별 {갱신 주기, 담당, 최종 갱신일}.
6. ⏸ **최종 검수·인도** — "신입이 이 문서만 보고 월요일 루틴을 돌릴 수 있는가" 기준으로 사용자 검수. 확정 후 093 체인 — 온보딩 첫 주 회의의 태스크 배분(093)에 이 매뉴얼의 업무 ID를 연결.

## Anti-Patterns
- **백지 인터뷰** — my-playbook·build-log에 이미 있는 업무를 처음부터 다시 묻기. 자동 추출이 먼저, 질문은 빈 곳만.
- **절차 없는 업무 나열** — "주간 리포트 작성"이라고만 쓰면 신입은 여전히 물어봐야 한다. 목록은 목차일 뿐, 절차 카드가 본체.
- **비밀번호 평문 기재** — 인수인계 문서가 유출 1순위 문서다. 위치 참조만, 값은 절대 금지.
- **암묵지 누락** — 절차는 맞는데 "이 클라이언트는 금요일 발송 금지" 같은 예외가 빠지면 사고는 그대로 반복된다.
- **일회성 문서** — 갱신 담당 없는 매뉴얼은 작성일부터 부패한다. 갱신 규칙 없으면 미완성 판정.

## 활용
- **신입 온보딩 패키지**: 입사 확정 → 이 스킬로 첫 주 가이드 생성 → 093으로 첫 주 태스크 배분 — "옆에 앉아서 알려주기" 일주일이 문서 한 벌로 압축.
- **퇴사·업무 이관**: 이관자 인터뷰 2회(자동 추출 검수 + 암묵지 보완)로 인수인계 표준본 완성 — 퇴사일에 몰아 쓰는 부실 문서 방지.
- **외주·대행 위임**: 매뉴얼의 절차 카드 일부를 잘라 094 외주 브리프의 산출물 명세로 재사용 — 내부 표준이 곧 외주 요구사항.

> 참고 — 저장·이력 규칙: 매뉴얼은 `outputs/{날짜}/onboarding/` + 팀 공유 공간에 두고, 갱신 때마다 build-log 에 1행 기록하면 문서 이력이 실적 원장과 함께 남는다.
