# 067 월간 마케팅 리포트 자동화

> 전 채널 한 달을 한 장으로. 경영진 요약은 결론 3줄 + 의사결정 요청이 먼저, 상세는 그 뒤에.
> 계승: 패스트캠퍼스 6-4 3media-integrated(매체 취합·동일 잣대 비교·인사이트·재배분 제안 구조를 월간·전 채널로 확장) + marketing-os html-report-template(단일 HTML 표준 디자인).

## Contract
- **경영진 요약이 문서 최상단** — 결론 3줄(성과 판정·핵심 원인·다음 달 방향) + 의사결정 요청 1~3건으로 시작한다. 상세 데이터는 그 뒤. 3줄을 읽으면 회의를 시작할 수 있어야 한다.
- 모든 핵심 지표에 **전월 대비·목표 대비**를 병기한다 (목표는 profile.md §4 — 목표 ROAS·CPA·월 예산. 미작성 시 전월 대비만 + 목표 미설정 고지).
- 채널 간 비교는 동일 잣대(공통 스키마)로, 기여 모델 차이 주의 주석 부착.
- 보고서는 html-report-template 표준 단일 HTML — 첨부만으로 열리고 인쇄된다.
- ⏸ Notion 아카이브 적재·월간 자동 가동(cron)은 사용자 승인 후에만 켠다 (mutating).

## Phases
1. **기준 로드** — [brand/profile.md](../../../../brand/profile.md) §4 목표 수치·손익분기 ROAS·월 예산 자동 참조.
2. **데이터 취합** — 입력 우선순위: ① 선행 산출물(045 주간 리포트·061 판매 분석·030 콘텐츠 회고 등 outputs/ 탐색) ② [data/connections.md](../../../../data/connections.md) 연결 MCP 실조회 ③ **미연결 채널은 CSV 업로드 폴백**. 채널별 소스와 모드를 산출물에 명시, `[실데이터]`/`[샘플]` 태그.
3. **월간 분석** — ① 전사 KPI (매출·마케팅비·ROAS·전환·CPA — 전월/목표 대비) ② 채널별 성과표 + 신호등 ③ 이번 달 이벤트 타임라인(캠페인·프로모션·이슈)과 지표 변동 연결 ④ 인사이트 5줄 (왜 이 채널이 이겼나/졌나).
4. **경영진 요약 작성** — 마지막에 쓰되 맨 앞에 배치: 결론 3줄 → 의사결정 요청(예산 이동·중단/확대 승인 등, 각 근거 1줄) → 다음 달 우선순위 3개.
5. **HTML 생성** — html-report-template 골격 (요약 → KPI 카드 → 채널 비교 → 차트[인라인 SVG] → 상세표 → 다음 달 액션). 저장: `outputs/{날짜}/monthly-report-{연월}.html`.
6. ⏸ **적재·정례화** — Notion 아카이브 1행 승인 후 기록. 매월 자동 생성 스위치는 별도 승인.
7. **체인 제안** — 지표 체계가 흔들리면 066 KPI 트리 설계, 다음 달 예산 확정은 095 마케팅 예산 플래너 (채널 재배분 상세는 046).

## Output Format
```
## {연월} 마케팅 월간 보고 · [실데이터|샘플]

### ★ 경영진 요약 (30초)
1. {성과 판정 — 목표 대비 한 줄}
2. {핵심 원인 — 무엇이 끌고 무엇이 깎았나}
3. {다음 달 방향 — 한 줄}
▶ 의사결정 요청: ① {예산·승인 안건 + 근거 1줄} ② ...

### KPI 요약 (전월 대비 · 목표 대비 병기)
| 지표 | 이번 달 | 전월 | Δ | 목표 | 달성률 |

### 채널별 성과 (동일 잣대 + 🟢🟡🔴)   ### 이벤트 타임라인 × 지표 변동
### 인사이트 5줄   ### 다음 달 액션 (담당·기한 포함 3~5개)

파일: outputs/{날짜}/monthly-report-{연월}.html · Notion {✅ 적재/보류}
다음 액션: → 066 KPI 트리 (지표 체계 재설계) / 095 예산 플래너 (다음 달 배분 확정)
```

## Anti-Patterns
- **요약을 상세 뒤에 배치** — 경영진은 2페이지를 안 넘긴다. 결론 3줄 우선 규칙은 문서 구조 자체다.
- **의사결정 요청 없는 보고** — "잘 봤다"로 끝나는 리포트는 실패. 승인이 필요한 안건을 명시적으로 올린다.
- **목표 없이 전월 대비만** — 전월이 최악이면 +20%도 실패다. profile.md 목표 대비를 항상 병기.
- **채널 담당자 시점 나열** — 채널별 자화자찬 취합은 통합 보고가 아니다. 동일 잣대 비교와 전사 결론으로 재구성.
- **승인 없는 자동 적재** — Notion 기록·매월 자동 가동은 mutating — 반드시 ⏸ 승인 후.

## 활용
- **월간 경영 회의**: 매월 1영업일에 생성 → 요약 1페이지만 인쇄해 회의 시작, 상세는 질문 나올 때 스크롤.
- **주간→월간 파이프라인**: 045 주간 리포트 4장을 자동 입력으로 — 주간 운영과 월간 보고가 이중 작업 없이 연결.
- **투자자·본사 보고**: 경영진 요약 포맷 그대로 외부 보고용으로 — 의사결정 요청 항목만 "요청 사항"으로 치환.
- **분기·연간 확장**: 월간 리포트 3장을 입력으로 분기 보고 — 경영진 요약 3줄 규칙은 동일, 타임라인만 분기 스케일로.

> 부록 — 경영진 요약 작성 규칙 (결론 3줄 우선)
> ① 판정 1줄: 목표 대비 달성/미달 + 핵심 수치 하나만 ② 원인 1줄: 끌어올린 것 하나 + 깎은 것 하나 ③ 방향 1줄: 다음 달 가장 큰 베팅.
> 의사결정 요청은 "…을 승인해 주세요" 형식의 완결 문장으로 — 열린 질문("어떻게 할까요?") 금지.
> 파일명 규칙 monthly-report-{연월}.html 을 바꾸지 않는다 — 069 예측 시뮬레이션·095 예산 플래너가 이 경로를 입력으로 탐색한다.
