# 070 설문·NPS 응답 분석

> NPS 산출 + 주관식 응답 테마 추출 + 감성 분류 → 비추천 고객 후속 플로우까지.
> 계승: 006 소비자 리뷰 마이닝의 분류 체계(5분류·감성 점수·클러스터링·대표 인용·PII 마스킹)를 설문 주관식 응답에 전용. 리뷰가 "자발적 VoC"라면 설문은 "질문에 대한 답" — 문항 맥락을 태그에 보존하는 것이 차이다.

## Contract
- NPS는 고정 정의로 산출한다: 추천(9~10) − 비추천(0~6) 비율, 중립(7~8) 별도 표기. 산식과 그룹 인원을 병기한다.
- 주관식은 테마 클러스터로 묶고, 테마마다 **건수 + 대표 인용 1개 이상**을 붙인다 — 인용 없는 테마는 신뢰 불가로 내보내지 않는다.
- 테마 분석은 추천/중립/비추천 **그룹별로 분리**한다 — "고객들이 배송을 언급"이 아니라 "비추천 그룹의 41%가 배송 언급".
- 응답 30건 미만이면 NPS 점수 해석 한계(±오차 큼)를 판정 요약에 명시하고, 점수보다 테마 중심으로 보고한다.
- 개인 식별 정보는 전 구간 마스킹한다 (이름 → 첫 글자, 연락처 제거) — 006과 동일 원칙.

## Phases
1. **기준 로드** — [brand/profile.md](../../../../brand/profile.md) §2 고객이 사는/떠나는 이유 자동 참조 — 테마 명명 시 기존 언어와 정렬 (재질문 금지).
2. **데이터 정리** — 점수 열·주관식 열 매핑 확인, 무응답·중복 제거, PII 마스킹. 문항이 여럿이면 문항별로 분리 집계.
3. **NPS 산출** — 그룹 분포(추천/중립/비추천) + NPS 점수 + 표본 수 판정. 이전 조사 데이터가 있으면 추이 비교.
4. **주관식 테마 클러스터링** — 006 방식 계승: 응답별 단일 테마 태그 → 의미 군집으로 TOP 5~8 테마 압축 (건수 순 + 그룹 가중치: 비추천 그룹 언급 테마 우선). 각 테마에 감성(-2~+2)·카테고리(제품/가격/배송/CS/경험) 부여.
5. **그룹별 사유 분석** — ① 비추천: 이탈 사유 TOP 3 (개선 시급) ② 중립: "9점이 되려면 뭐가 필요한가" 갭 테마 ③ 추천: 추천 이유 = 소구점 후보 (043·051 카피 재료로 표기).
6. **후속 플로우·체인** — 비추천 응답 중 CS 시급 건(환불·분쟁 소지)은 에스컬레이션 목록으로 분리 ⏸ → 주 체인: 078 VOC 취합·분류 시스템 (테마를 개선 백로그로 연결). 보조: 불만 고객 개별 후속 아웃리치는 079 리뷰·후기 요청 자동화의 후속 플로우 응용, 설문 재설계는 007.

판정 요약: NPS {n} (추천 {n}% − 비추천 {n}%) · 비추천 1위 사유 "{테마}" ({n}건) · 에스컬레이션 {n}건

## Anti-Patterns
- **점수만 보고하는 NPS** — "NPS 32입니다"로 끝나면 아무것도 바뀌지 않는다. 그룹별 사유가 본체다.
- **그룹 뭉뚱그리기** — 추천 고객과 비추천 고객의 주관식을 한 덩어리로 클러스터링하면 개선 시급도가 실종된다.
- **소표본 점수 과신** — 응답 18건으로 "NPS가 지난달보다 12점 올랐다" 단정 금지. 30건 미만은 테마 중심 보고.
- **인용 없는 테마** — "가격 불만이 많음"에 원문 발췌가 없으면 검증 불가. 테마 수 = 인용 수 이상.
- **유도 해석** — 문항이 유도형("어떤 점이 아쉬웠나요?")이었다면 부정 편향을 명시한다. 문항 맥락 없는 해석은 왜곡이다.

## 활용
- **교육·서비스 수료 설문**: 수료 직후 NPS + 주관식을 돌려 비추천 사유를 다음 기수 커리큘럼 백로그(078)로 — 이탈 사유가 개선 티켓이 된다.
- **구매 후 30일 설문**: 중립 그룹의 갭 테마("이것만 있었으면 9점")를 011 제품 개선·016 번들 설계의 입력으로.
- **추천 사유의 카피화**: 추천 그룹 인용문을 043 광고 카피·051 상세페이지의 고객 언어 소스로 재사용 — 설문이 리서치와 카피 재료를 동시에 만든다.
- **분기 NPS 추이 관리**: 같은 문항으로 분기마다 재실행해 점수·테마 이동을 추적 — 테마의 등락이 개선 조치의 효과 검증이 된다.
