# 065 고객 세그먼트 RFM 분석

주문 데이터만 있으면 RFM(최근성·빈도·금액)으로 고객을 6~8개 세그먼트로 나누고, 세그먼트마다 "누구에게 · 무엇을 · 언제" 액션 맵까지 설계한다. 분석은 내부 산출물 — 실제 메시지 발송은 074·077 체인에서.

## Contract
1. 모든 고객은 정확히 1개 세그먼트에 속한다 (중복·누락 0 — 세그먼트 인원 합계 = 전체 고객 수 검증 포함).
2. RFM 기준선은 brand/profile.md 의 AOV·구매주기에서 도출한다 — 임의의 절대값(예: "3개월") 을 업종 불문 적용하지 않는다.
3. 세그먼트마다 액션 맵 1행이 반드시 붙는다 — 이름만 있고 할 일 없는 세그먼트는 만들지 않는다.
4. 개인 식별 정보는 산출물에 노출하지 않는다 — 고객ID는 마스킹, 집계 수치만 사용.
5. 같은 데이터를 다시 넣으면 같은 세그먼트 결과가 나온다 (분위 경계값을 산출물에 명시해 재현 가능).

## Phases
1. **브랜드 기준 로드** — brand/profile.md 에서 AOV·마진율·재구매 관련 수치를 자동 참조한다 (재질문 금지). 구매주기 값이 비어 있으면 데이터에서 중위 재구매 간격을 추정하고 "추정" 을 명시한다.
2. **데이터 검증·정규화** — 필수 3열(고객ID·주문일·주문금액) 확인, 결측·중복 주문 제거, 분석 기준일 확정. 고객별로 R(마지막 구매 경과일) · F(기간 내 구매 횟수) · M(누적 구매액) 집계.
3. **RFM 점수화** — R·F·M 각각 5분위(1~5점) 점수화. 분위 경계값 표를 산출물에 명시 (재현성). 고객 수 200명 미만이면 5분위 대신 3분위로 자동 완화하고 사유를 표기.
4. **세그먼트 정의** — 점수 조합으로 표준 세그먼트 매핑:
   - 챔피언(R5·F4+) / 충성(R4+·F3+) / 잠재 충성(R4+·F1~2) / 신규(R5·F1) / 이탈 위험(R2·F3+) / 휴면(R1~2·F1~2) / 고액 이탈(R1~2·M4+) — 필요 시 병합해 6~8종 유지.
5. **세그먼트별 액션 맵** — 각 세그먼트에 `목표 → 제안 액션 → 오퍼 방향 → 실행 스킬` 1행씩. 예: 이탈 위험 → 재활성 → 리마인드 시퀀스 → 077 / 충성 → 객단가 상승 → 재구매 오퍼 → 074.
6. ⏸ **우선순위 확정** — 세그먼트 규모 × 매출 기여로 "먼저 공략할 2개 세그먼트" 를 제안하고 사용자 선택을 받은 뒤, 해당 체인 스킬(073 고객 여정 맵핑 / 074 이메일 시퀀스 빌더)로 핸드오프한다 (보조: 077 이탈 방지 윈백 캠페인).

## Output Format
```
## 고객 세그먼트 리포트 — 기준일 {날짜} · 고객 {N}명
판정 요약: 최대 세그먼트 {이름}({n}명·{%}) · 매출 기여 1위 {이름} · 권장 공략 {이름}, {이름}

### ① RFM 기준 (재현용)
| 축 | 정의 | 분위 경계값 (1→5) |
|---|---|---|

### ② 세그먼트 정의표
| 세그먼트 | 조건 (R·F·M) | 인원 | 비중 | 매출 기여 | 특징 한 줄 |
|---|---|---|---|---|---|
(합계 검증: {n}명 = 전체 {N}명 ✅)

### ③ 세그먼트별 액션 맵
| 세그먼트 | 목표 | 제안 액션 | 오퍼 방향 | 실행 스킬 |
|---|---|---|---|---|

### ④ 우선 공략 제안 (2개) ⏸
- {세그먼트}: 근거 수치 → 다음 스킬 {073|074}

다음 액션: 선택한 세그먼트로 073 고객 여정 맵핑 또는 074 이메일 시퀀스 빌더를 바로 시작할 수 있습니다 (보조: 077 이탈 방지 윈백 캠페인).
```

## Anti-Patterns
- **세그먼트 남발** — 12개, 20개로 쪼개면 액션이 안 나온다. 6~8개 상한, 인원 3% 미만 세그먼트는 인접 세그먼트에 병합.
- **절대 기준 하드코딩** — "90일 미구매 = 휴면" 을 모든 업종에 적용 금지. 기준은 profile.md 구매주기(또는 데이터 추정치)에서.
- **분류만 하고 끝** — 액션 맵 없는 세그먼트표는 벽지다. 세그먼트 수 = 액션 행 수.
- **개인정보 노출** — 산출물·예시에 실명·연락처·전체 고객ID를 싣지 않는다.
- **표본 부족 무시** — 고객 50명에 5분위를 강행하면 노이즈 세그먼트가 나온다. 규모에 따라 분위를 완화하고 명시.

## 활용
- **CRM 첫 설계**: 스마트스토어 주문 내역 6개월치 CSV 하나로 첫 세그먼트 체계 구축 → 075 카카오 알림톡·074 이메일 시퀀스의 타겟 기준으로 사용.
- **분기 리밸런싱**: 분기마다 재실행해 세그먼트 이동(신규→충성, 충성→이탈 위험)을 추적 — 이동량 자체가 리텐션 KPI가 된다.
- **이탈 조기 대응**: "이탈 위험 + 고액 이탈" 두 세그먼트만 뽑아 077 윈백 시퀀스로 즉시 연결 — 매출 방어 우선순위가 데이터로 정해진다.
- **광고 타겟 소재화**: 챔피언·충성 세그먼트의 공통 속성을 043 메타 광고의 유사 타겟 시드로 활용 — CRM 분석이 신규 획득 효율까지 끌어올린다.
- **VIP 프로그램 설계 입력**: 세그먼트 규모·매출 기여표를 076 멤버십·등급 제도의 등급 경계 근거로 그대로 전달한다.
