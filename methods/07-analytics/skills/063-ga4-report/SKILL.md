# 063 GA4 분석 리포트

> GA4 데이터 → **유입 소스 → 랜딩 → 전환 경로** 3단 해석 + 세팅 오류 검출 + 개선 액션까지.
> 계승: 패스트캠퍼스 실습 6-1/6-2 리포트 골격(KPI 카드 → 도식 → 상세표 → 보완 제안 6건, 모든 수치에 해석 코멘트)을 광고 성과에서 트래픽·전환 해석으로 전용. 읽기 전용 원칙 동일.

## Contract
- 분석은 항상 **유입 소스 → 랜딩 → 전환 경로** 3단 구조를 따른다 — 지표 백화점식 나열 금지.
- 수치 해석 전에 **세팅 신뢰도 검사**를 먼저 통과한다: 핵심 이벤트 미수집 · 중복 계측(전환 2회 발화) · unassigned/(not set) 비율 15% 초과 — 하나라도 걸리면 리포트 첫 줄에 경고.
- 모든 표·도식에는 해석 문장이 1:1로 붙는다 — "세션 12,400"이 아니라 "세션 12,400, 그중 오가닉 -18% (원인 후보: …)".
- 개선 액션은 담당 스킬로 연결한다 — 랜딩 문제 → 048 CRO 진단 · 유입 태깅 문제 → 050 UTM 정리.
- 데이터 출처를 `[실데이터]`/`[샘플]`로 태그하고, GA4를 변경(이벤트 수정·잠재고객 생성)하지 않는다 — 읽기 전용.

## Phases
1. **기준 로드** — [brand/profile.md](../../../../brand/profile.md) §5 운영 채널·§2 핵심 타겟 자동 참조. 핵심 전환 이벤트(구매·리드 제출 등)를 사용자에게 1회 확인 ⏸ (이후 재질문 금지).
2. **데이터 확보** — [data/connections.md](../../../../data/connections.md) 모드 판정: GA4 MCP/API 연결 시 직접 조회, **미연결 시 보고서·탐색 CSV 내보내기 폴백** (필요 최소 3종: 트래픽 획득 · 방문 페이지 · 이벤트/전환).
3. **세팅 신뢰도 검사** — ① 핵심 전환 이벤트가 기간 내 수집되는가 (0건이면 세팅 오류 우선 보고) ② 같은 전환의 중복 발화 징후 (전환수/세션수 비정상 비율) ③ unassigned·(not set)·direct 과다 비율 ④ 자사 트래픽·봇 필터 여부. 심각 오류 시 "수치 해석은 참고용" 경고를 붙이고 진행 여부를 확인한다 ⏸.
4. **1단 유입 소스** — 채널 그룹별 세션·참여율·전환 기여 집계 + 전기 대비 증감. 유입 품질 = 참여 세션율×전환율로 "많이 오는 채널"과 "돈 되는 채널"을 분리한다.
5. **2단 랜딩** — 방문 페이지 TOP 10 × 참여율·전환 시작율. 트래픽은 많은데 참여가 죽는 랜딩(전환 저해 후보)을 특정한다.
6. **3단 전환 경로** — 핵심 이벤트 퍼널(예: 세션→상품 조회→장바구니→구매) 단계별 이탈률. 최대 이탈 구간 1곳을 "개입 지점"으로 지목하고 원인 가설을 붙인다.
7. **개선 액션·체인** — 발견 사항을 액션 6건으로 정리 (High/Mid/Low 우선순위 — 실습 6-1 보완 제안 형식 계승) → 주 체인: 048 랜딩페이지 CRO 진단(랜딩 개입) · 050 어트리뷰션 정리(unassigned 과다·태깅 오류 시).

## Anti-Patterns
- **오염된 세팅 위의 해석** — unassigned 30%인 채로 채널 비교를 단정하는 것. 세팅 검사가 항상 0번이다.
- **지표 백화점** — GA4 기본 보고서 전부를 나열하면 리포트가 아니라 화면 캡처다. 3단 구조 밖 지표는 버린다.
- **세션·사용자·이벤트 혼용** — "방문자 1만"이 세션인지 사용자인지 불명확한 문장 금지. 지표명·정의를 병기한다.
- **마지막 클릭 맹신** — 기본 귀속 모델의 한계를 명시하지 않고 "이 채널이 전환을 만들었다" 단정 금지.
- **액션 없는 발견** — "이탈률이 높습니다"로 끝나면 사용자가 다시 조사해야 한다. 발견마다 가설+담당 스킬.

## 활용
- **월간 트래픽 점검**: 매월 초 28일 리포트로 채널 믹스 변화를 추적 — 067 월간 리포트의 트래픽 섹션 입력으로 재사용.
- **광고 집행 전 정지 작업**: 유료 광고 시작 전 세팅 신뢰도 검사 + 전환 퍼널 기준선 확보 — 광고 후 증분을 잴 수 있게 만든다.
- **콘텐츠 성과 연결**: 랜딩 TOP 10에서 블로그 유입 페이지의 전환 시작율을 확인 → 022 SEO 아티클의 주제 선정 근거로.
- **리뉴얼 전후 비교**: 사이트 개편 전 리포트를 기준선으로 보관 → 개편 후 동일 3단 구조로 재실행해 개선 효과를 구간별로 검증.

> 부록 — CSV 폴백 시 내보내기 3종 (GA4 보고서 메뉴 기준)

| 내보내기 | 위치 | 이 스킬에서의 용도 |
|---|---|---|
| 트래픽 획득 | 보고서 > 획득 | 1단 유입 소스 |
| 방문 페이지 | 보고서 > 참여도 | 2단 랜딩 |
| 이벤트/전환 | 보고서 > 참여도 > 이벤트 | 3단 전환 경로 + 세팅 검사 |
