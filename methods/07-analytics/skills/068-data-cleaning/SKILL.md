# 068 데이터 클리닝 & 전처리

> 지저분한 로우데이터를 표준 4축 체크리스트(결측·중복·형식·이상치)로 진단하고, 원본은 건드리지 않은 채 사본+전수 로그로 정리한다.
> 계승: 안티그래비티 「구글 스프레드 MCP 에이전트」 전처리 파트 — 시트 구조 파악 → 스키마 검증 → 정리 기록 흐름을 표준 파이프라인으로 규격화.

## Contract
- **원본 파일은 절대 덮어쓰지 않는다** — 항상 사본(`{원본명}_cleaned`)에서 작업하고 원본 경로를 로그에 남긴다.
- 모든 변경(제거·치환·병합·형변환)은 클리닝 로그에 1행씩 기록된다 — 로그로 원본을 재구성할 수 있는 수준.
- 표준 4축(결측·중복·형식·이상치)을 빠짐없이 진단하고, 축별 검출 건수를 요약에 명시한다.
- ⏸ 삭제·치환 기준은 실행 전에 사용자 확인을 받는다 — 임의 판단으로 행을 지우지 않는다.
- 같은 입력 + 같은 기준이면 같은 결과가 나온다 (적용 규칙을 로그 상단에 명시 — 재현성).

## Phases
1. **구조 파악** — [data/connections.md](../../../../data/connections.md) 모드 판정: Google Sheets MCP 연결 시 get_spreadsheet_info·read_sheet로 직접 조회, **미연결 시 CSV/XLSX 업로드 폴백**. 열 구성·행 수·데이터 타입 스캔.
2. **4축 진단** — 표준 클리닝 체크리스트:

| 축 | 검사 항목 | 기본 처리 후보 |
|---|---|---|
| 결측 | 빈 셀·NULL·"-"·"없음" 혼용 | 행 제거 / 0 대체 / 전후값 보간 / 유지+표기 |
| 중복 | 완전 중복행 · 키(주문번호 등) 중복 | 제거 / 최신 우선 병합 |
| 형식 | 날짜 혼용(YYYYMMDD·YY/M/D), 숫자 내 콤마·원, 채널명 표기 흔들림(쿠팡/coupang), 공백·대소문자 | 표준형 통일 (날짜 ISO·숫자형·명칭 사전) |
| 이상치 | 음수 수량·0원 매출·범위 밖 값(IQR 1.5배 기준) | 확인 후 수정 / 제외 / 유지+플래그 |

3. ⏸ **처리 기준 확정** — 축별 검출 결과와 처리 후보를 표로 제시하고 사용자 선택을 받는다 (특히 이상치 — 실제 특수 판매일 수 있음).
4. **클리닝 실행 + 로그** — 승인된 규칙으로 사본에 일괄 적용. 로그 형식: `{행/열} | {원값} → {새값} | {규칙} | {사유}`. 목표 스키마가 있으면 열 이름·순서·타입까지 매핑.
5. **사후 검증** — 처리 전후 행 수·합계(매출 등 핵심 열) 대조표 제시 — 합계가 변했다면 원인(제거된 행)을 명시.
6. **산출·체인** — 정리본 + 로그 + 품질 요약 전달 → 061 판매 데이터 분석 (정리본 즉시 투입). Sheets 모드면 새 탭에 정리본 기록 제안 (원본 탭 불변).

## Output Format
```
## 데이터 클리닝 결과 — {파일명} · [실데이터|샘플]
품질 요약: {n}행 × {n}열 → 정리 후 {n}행 · 결측 {n} / 중복 {n} / 형식 {n} / 이상치 {n} 처리

### ① 적용 규칙 (재현용) — 축별 선택된 처리 기준
### ② 처리 전후 대조
| 항목 | 처리 전 | 처리 후 | 차이 | 원인 |
| 행 수 | | | | |
| 매출 합계 | | | | |

### ③ 클리닝 로그 (전수) — {행/열 | 원값 → 새값 | 규칙 | 사유}
### ④ 잔여 이슈 — 자동 처리 불가·사용자 판단 보류 항목 {n}건

파일: {원본명}_cleaned.{ext} + cleaning-log.md (원본 불변)
다음 액션: → 061 판매 데이터 분석 (정리본으로 즉시 분석 시작)
```

## Anti-Patterns
- **원본 위에서 작업** — 되돌릴 수 없는 정리는 사고다. 사본 생성이 항상 1단계.
- **무단 행 삭제** — "이상해 보여서 지웠다"는 금지. 이상치는 특수 프로모션·대량 구매일 수 있다 — ⏸ 확인 후 처리.
- **로그 없는 정리** — "정리했습니다"만 남기면 분석 결과를 신뢰할 수 없다. 변경 전수 기록이 산출물의 절반.
- **형식 통일 생략** — 채널명 표기 흔들림(쿠팡/Coupang/coupang)을 남기면 이후 집계가 3개 채널로 쪼개진다. 명칭 사전 통일 필수.
- **합계 검증 생략** — 정리 후 매출 합계가 바뀌었는데 침묵하면 이후 모든 보고가 어긋난다. 전후 대조표 필수.

## 활용
- **분석 전 관문**: 061·065·067 등 분석 계열 스킬 앞에 상시 배치 — /월간리포트 커맨드가 068→061→067→066 순서인 이유.
- **여러 소스 병합 준비**: 쿠팡·스마트스토어·자사몰 주문 파일을 공통 스키마로 통일 — 채널 통합 분석의 전제 조건.
- **반복 업무 표준화**: 한 번 확정한 적용 규칙(①)을 저장해두면 매월 같은 파일 형식에 동일 기준 재적용 — 월간 루틴이 5분으로.
- **B2B 셋업 1단계**: 고객사 납품 시 데이터 품질 진단(4축 검출 건수)을 착수 보고서로 — 프로젝트 범위 산정의 근거가 된다.
