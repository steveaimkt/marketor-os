# 062 마케팅 대시보드 빌더

> 멀티채널 데이터 → 단일 파일 HTML 대시보드. KPI 카드 → 추이 → 채널 비교 3층 구조, 폰에서 열리고 인쇄되고 다크모드까지.
> 계승: marketing-os 「html-report-template」 표준 템플릿 **그대로** — 단일 파일·모바일 우선·인쇄 가능·다크모드 자동 + 표준 섹션 골격(header/kpi/highlights/charts/tables/actions/footer).

## Contract
- 대시보드는 **KPI 카드 → 추이 → 채널 비교** 3층 구조를 따른다 (html-report-template 표준 섹션 순서 준수).
- 산출물은 단일 .html 파일 — CSS/JS 인라인, 외부 CDN·폰트·이미지 의존 0건. 첨부만 받아도 오프라인에서 열린다.
- 모든 위젯에 데이터 출처와 갱신 시점을 표기한다 — 출처 없는 숫자는 싣지 않는다.
- 지표 정의(계산식)를 대시보드 하단에 문서화한다 — ROAS·CPA 등은 profile.md §4 기준값(손익분기 ROAS·목표 CPA)과 함께 표기.
- 검증 3종을 통과해야 완성: 파일 < 500KB · 모바일 360px 가로 스크롤 없음 · 다크모드 자동 전환.

## Phases
1. **지표·소스 정의** ⏸ — 사용자와 핵심 지표 4~8개 확정 (미지정 시 profile.md §5 운영 채널 기준 기본 세트 제안: 매출·광고비·ROAS·전환·CPA·트래픽). 지표별 소스 매핑.
2. **데이터 확보** — [data/connections.md](../../../../data/connections.md) 모드 판정: GA4·Coupler.io·Sheets MCP 연결 시 자동 조회, **미연결 시 CSV 업로드 폴백** (CSV → HTML 경로는 연결 없이 완주 가능). `[실데이터]`/`[샘플]` 태그.
3. **정규화** — 채널별 지표를 공통 스키마·동일 기간으로 정렬. 채널 간 정의 차이(전환 기준 등)는 각주로.
4. **HTML 생성** — html-report-template 골격 적용:
   - `header` 제목·기간·생성 타임스탬프 / `kpi` 카드 4~8개 (값+증감 delta) / `highlights` 핵심 3가지 (텍스트)
   - `charts` 인라인 SVG (추이 라인·채널 비교 막대 — 외부 라이브러리 금지) / `tables` 정렬 가능한 상세표 (가벼운 인라인 JS)
   - `actions` 권장 액션 3~5개 체크박스 / `footer` + 지표 정의·출처·갱신 시점 블록
   - 저장: `outputs/{날짜}/dashboard-{날짜}.html`
5. **검증** — 500KB·360px·다크모드 3종 체크, 미통과 시 수정 후 재검증.
6. **갱신 가이드 작성** — "다음 갱신 시: 같은 형식 CSV 교체 → 본 스킬 재실행" 절차 + 소스별 다운로드 경로 1줄씩 → 체인: 067 월간 리포트 (대시보드 지표를 월간 보고 골격으로).

## Output Format
```
## 마케팅 대시보드 — {기간} · [실데이터|샘플]
파일: outputs/{날짜}/dashboard-{날짜}.html ({n}KB · 360px ✅ · 다크모드 ✅)

[대시보드 내부 구조]
① KPI 카드: {지표} {값} ({±delta}) × {n}개
② 이번 기간 핵심 3가지 (텍스트 3줄)
③ 차트: 추이 라인 + 채널 비교 막대 (인라인 SVG)
④ 상세표 (정렬 가능) — 각 표에 출처·갱신 시점 표기
⑤ 다음 액션 체크박스 {n}개
⑥ 지표 정의: ROAS = 매출/광고비 (BEP {n}, profile.md §4) · ...

갱신 가이드: {소스별 교체 절차}
다음 액션: → 067 월간 마케팅 리포트 자동화 (이 지표 세트로 월간 보고 정례화)
```

## Anti-Patterns
- **외부 의존 삽입** — Chart.js CDN·웹폰트 링크는 첨부 열람에서 깨진다. 차트는 인라인 SVG, 스타일은 인라인 CSS.
- **지표 백화점** — 20개 지표는 아무도 안 본다. KPI 카드는 4~8개, 나머지는 상세표로.
- **계산식 없는 숫자** — 채널마다 ROAS 정의가 다르다. 정의 블록 없는 대시보드는 회의에서 싸움만 만든다.
- **데스크톱 전용 폭** — 1인 마케터는 폰에서 본다. 360px 검증 생략 금지.
- **대시보드에 해석 위임** — highlights 3줄 없이 차트만 나열하면 "그래서 뭐?"가 된다. 핵심 3가지는 텍스트로 박는다.

## 활용
- **주간 팀 공유**: 매주 CSV 교체 → 재생성 → 단일 HTML을 메신저 첨부 — 접속 권한·로그인 없이 전원이 같은 화면.
- **광고 리포트 계열 통일**: 045 주간·067 월간 리포트가 같은 템플릿 골격을 쓰므로, 본 스킬로 만든 대시보드와 디자인이 자동 통일된다.
- **고객사 납품**: B2B 컨설팅에서 고객사 데이터로 대시보드 1장 + 갱신 가이드 전달 — 유지보수 없이 셀프 갱신 가능한 산출물.
- **온보딩 첫 산출물**: /마케터OS셋업 직후 profile.md §5 운영 채널만으로 첫 대시보드를 만들어 두면 이후 모든 리포트 스킬의 기준 지표 세트가 된다.

> 부록 — 완성 검증 체크리스트

> - [ ] prefers-color-scheme 다크모드 전환 확인
