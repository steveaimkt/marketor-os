# 061 판매 데이터 분석

> 매출 로우데이터 → 채널·상품·지역 다축 집계 → 수량×단가×믹스 요인 분해 → 이상치 검출과 원인 가설까지.
> 계승: 안티그래비티 「구글 스프레드 MCP 에이전트」 집계 파트 — 채널별/상품별/지역별 매출·이익·수익률 집계 + 교차 분석. 성장 동인·이상치 해석은 본 스킬에서 보강.

## Contract
- 매출 변화는 **수량 × 단가 × 믹스** 요인으로 분해한다 — "매출이 늘었다/줄었다"에서 멈추지 않는다.
- 집계는 매출과 함께 이익·수익률을 병기한다 (원가 열이 있으면 필수, 없으면 profile.md §4 마진율로 추정 + `[추정]` 태그).
- 이상치는 검출 기준(7일 이동평균 대비 2σ 주의·3σ 긴급)을 명시하고, 반드시 원인 가설 1개 이상과 함께 보고한다.
- 모든 차트는 해석 문장과 1:1로 붙는다 — 해석 없는 차트 금지.
- 원본 데이터는 수정하지 않는다 (읽기 전용 분석 — 시트 기록은 사용자 요청 시 별도 단계).

## Phases
1. **기준 로드** — [brand/profile.md](../../../../brand/profile.md) §4 AOV·마진율·§5 채널 현황 자동 참조 (재질문 금지).
2. **데이터 확보** — [data/connections.md](../../../../data/connections.md) 모드 판정: Google Sheets MCP 연결 시 read_sheet/batch_read로 실시간 조회, **미연결 시 CSV/XLSX 업로드 폴백**. `[실데이터]`/`[샘플]` 태그 필수.
3. **정합성 검사** — 필수 열 확인, 기간 누락일·중복 주문 점검. 심하게 지저분하면 068 데이터 클리닝 선행을 제안하고 멈춘다.
4. **다축 집계** — ① 기간 추이(일/주/월) ② 채널별 매출·이익·수익률 ③ 상품별 TOP/BOTTOM + 파레토(상위 20% 상품의 매출 비중) ④ (열 존재 시) 지역·카테고리.
5. **성장 동인 분해** — 전기 대비 Δ매출 = Δ수량 기여 + Δ단가 기여 + Δ믹스 기여(채널·상품 구성 변화). 기여도 순으로 "왜 늘었나/빠졌나" 서술.
6. **이상치 검출·가설** — 이동평균 기반 스파이크/급락 검출 → 각 건에 원인 가설(프로모션·품절·시즌·데이터 오류)과 확인 방법 제시.
7. **리포트·체인** — 핵심 차트(추이·채널 비중·파레토)와 함께 정리 → 067 월간 리포트(전 채널 통합 보고) · 069 예측 시뮬레이션(시나리오 전망). 광고 연관 이상치는 041/042/046으로 분기 제안.

## Output Format
```
## 판매 데이터 분석 — {기간} · [실데이터|샘플]
판정 요약: 매출 {n}원 (전기 {±%}) · 주동인 {수량|단가|믹스} ({기여 %p}) · 이상치 {n}건 (긴급 {n})

### ① 성장 동인 분해
Δ매출 {±n} = 수량 {±n} + 단가 {±n} + 믹스 {±n} → 해석 2~3줄

### ② 채널별 집계 (매출·이익·수익률·전기 대비)   ### ③ 상품 TOP 10 + 파레토 ({상위 n개 = 매출 {n}%})
### ④ 이상치 목록
| 날짜 | 항목 | 값 | 기준(σ) | 원인 가설 | 확인 방법 |

### ⑤ 핵심 차트 3종 + 각 해석 1줄

다음 액션: → 067 월간 마케팅 리포트 (경영 보고 통합) / 069 예측 시뮬레이션 (다음 분기 시나리오)
```

## Anti-Patterns
- **합계만 보고하는 분석** — "총매출 3억"은 분석이 아니다. 요인 분해와 축별 분포까지가 계약.
- **이상치 무가설 나열** — 스파이크 날짜만 뽑으면 사용자가 다시 조사해야 한다. 가설+확인 방법까지.
- **매출≠이익 혼동** — 매출 성장이 저마진 채널 쏠림이면 악화일 수 있다. 수익률 병기 필수.
- **기간 비교 왜곡** — 28일 vs 31일, 프로모션 주간 vs 평시를 그대로 비교 금지. 일평균·동일 조건 보정 명시.
- **지저분한 데이터 강행 돌파** — 결측·중복 위에 쌓은 집계는 전부 오염. 068 클리닝 선행 제안이 먼저다.

## 활용
- **월초 실적 리뷰**: 지난달 주문 데이터로 요인 분해 → "왜 빠졌나"를 수량(트래픽)·단가(할인)·믹스(채널) 중 하나로 특정해 대응 스킬로 분기.
- **채널 전략 판단**: 채널별 수익률 집계로 "매출은 쿠팡, 이익은 자사몰" 같은 구조를 발견 → 046 예산 재배분의 근거로.
- **파레토 SKU 관리**: 상위 20% 상품 집중도가 과도하면 리스크 — 011 신제품 컨셉·016 번들 설계로 포트폴리오 확장 논의.
- **프로모션 사후 평가**: 행사 기간을 이벤트로 태그해 요인 분해 — 행사 매출 증분이 단가 하락(할인)을 상쇄했는지 수치로 판정.
