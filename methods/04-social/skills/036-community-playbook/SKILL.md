# 커뮤니티 운영 플레이북

> Discord/카페/오픈채팅 커뮤니티의 **채널 구조 → 온보딩 → 활성화 룰**을 설계한다.
> 계승: discord-channels-setup의 단계 게이트식 진행 원칙 + bot-build의 채널 라우팅 표(채널명·목적·허용 행동·매핑 담당) 설계 패턴.
> ※ 봇 연동·셋업 자동화는 이 스킬의 범위 밖 — 구조·운영 설계까지만 담당한다.

## Contract
- 채널 구조는 목적별 최소 구성(5~7개)으로 시작하고, 각 채널에 목적·허용 행동·운영 개입 기준이 1행씩 정의된다.
- 채널 추가는 "기존 채널에서 해당 주제 대화가 주 n회 이상 반복될 때"처럼 조건부 규칙으로만 허용한다.
- 온보딩은 가입 후 첫 24시간 경험(환영→자기소개→첫 액션)이 별도 플로우로 설계된다.
- 활성화 룰북에는 운영자 개입 기준(침묵·분쟁·위기)이 수치 조건으로 명시된다.
- 성장 단계(0~100 / 100~500 / 500+)별 구조 확장 로드맵이 포함된다.

## Phases
1. **목적·플랫폼·규모 확정** — 목적(고객 지원/팬덤·소속감/리드 육성) 1개 확정. 플랫폼 선택 기준: Discord(실시간·세분 채널·역할 권한), 네이버 카페(검색 유입·아카이브), 카카오 오픈채팅(진입장벽 최소·단일 방). 미정 시 ⏸ 목적 기준으로 추천 후 확인.
2. **채널 구조 설계** — 최소 구성 원칙으로 라우팅 표 작성. 기본 골격(Discord 기준): 📢 공지(읽기 전용) / 📜 규칙·시작하기 / 👋 자기소개 / 💬 메인 잡담 / 🎯 주제 채널 1~2 / 🙋 질문·피드백 / 🔒 운영진. 채널마다 권한(읽기/쓰기), 금지 행동(홍보·도배), 담당 운영자를 지정.
3. **온보딩 첫 24시간 설계** — ① 가입 직후: 환영 메시지 + 규칙 채널 안내 + 자기소개 템플릿(3문항: 누구/왜 왔나/요즘 고민) ② 자기소개 시 운영자 첫 댓글 응답(24시간 내 보장) + 역할 부여 ③ 첫 액션 유도: 진행 중인 투표·이번 주 질문에 참여 안내. 신규 멤버가 "아무도 나를 못 봤다"로 이탈하는 첫 24시간을 막는 것이 목표.
4. **활성화 룰북 작성** — 주간 리추얼(요일 고정 코너: 월 질문/수 팁/금 자랑), 운영자 개입 기준: ① 메인 채널 48시간 침묵 → 시드 질문 투입 ② 특정 멤버 공격·분쟁 → 1차 경고, DM 중재 ③ 환불·불만 등 위기 키워드 → 032 응대 체인으로 이관 ④ 2주 무게시물 채널 → 통폐합. 멤버 등급(신규→정착→기여자)과 승급 조건 정의.
5. **성장 로드맵 + 032 체인** — 0~100명: 운영자가 콘텐츠의 80% 공급 / 100~500명: 리추얼 정착·기여자 발굴 / 500+: 모더레이터 위임·주제 채널 분화. 일상 응대·위기 감지 룰은 032 댓글·DM 응대 어시스턴트로 핸드오프.

```markdown

## 1. 채널 구조 (시작 구성 {n}개)

| 💬 잡담 | 일상 대화 | 전체 | 외부 홍보 | 48h 침묵 시 시드 질문 |

**채널 추가 규칙**: {조건부 규칙}

- 주간 리추얼: 월 {코너} / 수 {코너} / 금 {코너}
```

## Anti-Patterns
- **개설일 채널 20개** — 빈 채널의 정적이 커뮤니티를 죽인다. 5~7개로 시작하고 조건부로만 추가.
- **온보딩 방치** — 규칙 채널만 던져놓으면 첫 24시간 내 이탈. 자기소개에 대한 운영자 첫 응답을 보장 항목으로.
- **개입 기준 없는 "활성화 노력"** — "열심히 소통한다"는 룰이 아니다. 침묵 48시간·무게시물 2주 같은 수치 조건으로.
- **플랫폼 관성 선택** — 목적 검토 없이 "남들 하니까 디스코드". 검색 아카이브가 핵심이면 카페가 맞다.
- **운영자 1인 무한 공급 고착** — 100명 이후에도 운영자만 말하는 구조면 확장 실패. 리추얼·기여자 위임 설계 필수.

## 활용
- **결로아 예시**: "민감성 피부 루틴 커뮤니티 만들자" → Discord 6채널 구조 + 자기소개 템플릿(피부 타입/고민) + 수요일 성분 Q&A 리추얼 룰북.
- **기존 커뮤니티 리셋**: 죽은 채널 통폐합 기준을 적용해 구조를 재편하고 온보딩만 다시 설계.
- **교육·멤버십 운영**: 수강생 커뮤니티에 등급(수강생→수료생→앰배서더)과 승급 조건을 적용해 이탈 방지.
