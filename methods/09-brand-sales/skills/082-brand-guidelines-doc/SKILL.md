# 082 브랜드 가이드라인 문서화

> 로고·컬러·보이스 사용 규정을 **외주·협업사에 그대로 전달 가능한 원페이지**로 압축하고, [brand/tone.md](../../../../brand/tone.md) 전체(어휘 사전·문체 규칙·채널 변주)를 갱신한다.
> 계승: marketing-os `brand-voice`의 5요소 체계(톤·어휘·금기어·페르소나·필러)와 "차단이 아닌 치환" 설계(금기어에 반드시 대체 표현 쌍) + 패스트캠퍼스 실습 5-1의 규정 파트.
> **081과 경계**: 082 = 사용 규정 (브랜드의 "어떻게" — 로고를 어디에 어떻게, 어떤 단어를 쓰고 안 쓰고) / 081 = 서사·미션 (브랜드의 "왜"). 서사·미션이 아직 없으면 081 선행을 제안한다 — 근거 없는 규정은 안 지켜진다.

## Contract
- 가이드는 **원페이지**로 압축한다 — 30장 브랜드북은 아무도 안 읽는다. 상세는 부록 링크로.
- 모든 금지 규정에는 **올바른 예(대체)가 쌍으로** 붙는다 — "로고 변형 금지"가 아니라 "❌ 비율 변형 → ✅ 제공 원본 3종 중 선택" (brand-voice 치환 설계 계승).
- 어휘 규정은 "쓰는 말 / 안 쓰는 말 / 이유" 3열 표를 유지한다 (tone.md §2 형식) — 이유 없는 금지어 목록은 지켜지지 않는다.
- 원페이지는 자기완결적이어야 한다 — 이 문서만 첨부받은 외주 디자이너가 추가 질문 없이 작업 가능한 수준 (컬러는 HEX 병기, 로고는 파일 경로/링크 명시).
- tone.md 전체 갱신은 diff 제시 → 사용자 승인 후에만 반영한다 ⏸.

## Phases
1. **자산·서사 로드** — [brand/profile.md](../../../../brand/profile.md) §1·§6, [brand/tone.md](../../../../brand/tone.md) 기존 전문, 081 산출물(내러티브·보이스 형용사) 자동 참조. 081 미실행이고 tone.md도 비어 있으면 ⏸ "서사 없이 규정만 만들지, 081을 먼저 할지" 확인.
2. **비주얼 자산 취합** — 로고 파일(원본·흑백·심볼), 컬러(주·보조·배경 HEX), 폰트(제목·본문·대체 폰트)를 받는다. 없는 자산은 `[미확보]`로 표에 남긴다 — 빈칸을 임의 색으로 채우지 않는다.
3. **사용 규정 정리** — 3파트 허용/금지 쌍 작성: ① 로고 (최소 크기·여백·배경 대비·금지 변형 4종: 비율/색/회전/효과) ② 컬러 (조합 규칙·배경별 사용·접근성 대비) ③ 보이스 (tone.md §1~§3 압축 + 금기 표현→대체 표현 쌍, profile.md §6 금기 반영).
4. **원페이지 조립** — 1장 구성: 상단 브랜드 한 줄(081 미션) / 좌: 로고·컬러·폰트 규정 / 우: 보이스·어휘 표 / 하단: 금지 사례 ❌→✅ 갤러리 + 문의처. 외주 전달용 체크리스트(작업물 셀프 검수 7항) 부착.
5. ⏸ **tone.md 전체 갱신 승인** — 어휘 사전·문체 규칙·채널 변주 갱신안을 diff로 제시 → 승인 시 반영. 원페이지와 tone.md가 서로 어긋나지 않는지 교차 검증 (단일 진실 소스는 tone.md).
6. **배포 안내·체인** — 외주·협업 전달 시나리오별 사용법 1줄씩 (디자인 외주 = 원페이지+로고 파일 / 카피 외주 = 원페이지+tone.md) → 체인: 081 브랜드 스토리·미션 빌더 (서사가 비었거나 낡았으면 세트 완성).

## Output Format
```
## 브랜드 가이드라인 원페이지 — {브랜드} · v{n} · {날짜}
판정 요약: 자산 {로고 ✅/컬러 ✅/폰트 [미확보]} · 규정 {n}쌍 (전부 ❌→✅ 병기) · tone.md {갱신 n개 섹션}

### ① 원페이지 본문
{브랜드 한 줄 (081 미션)}
| 로고 | 컬러 | 폰트 |
|---|---|---|
| 원본 3종 · 최소 {n}px · 여백 규칙 | 주 {HEX} 보조 {HEX} · 조합 규칙 | 제목 {폰트} 본문 {폰트} · 대체 {폰트} |

**보이스**: {형용사 3} · 어휘 표 (쓰는 말/안 쓰는 말/이유) {n}행
**금지 사례**: ❌ {사례} → ✅ {올바른 예} (×{n}쌍)

### ② 외주 셀프 검수 체크리스트 (7항)
### ③ tone.md 갱신안 (diff) ⏸

다음 액션: → 081 브랜드 스토리·미션 빌더 (서사·미션이 비어 있다면 — 규정의 "왜"를 채우는 세트)
```

## Anti-Patterns
- **브랜드북 인플레이션** — 규정을 30장으로 늘리는 것. 안 읽히는 완벽함보다 지켜지는 한 장. 원페이지 초과분은 잘라 부록으로.
- **금지만 있는 규정** — "하지 마세요" 목록만 주면 외주는 매번 되묻는다. 모든 ❌에 ✅ 대체를 쌍으로.
- **서사 없는 규정 강행** — 미션·신념 없이 만든 어휘 규정은 근거를 설명 못 해 팀이 안 따른다. 081 선행 확인이 먼저.
- **원페이지·tone.md 이중 진실** — 두 문서가 다른 말을 하면 100개 스킬이 갈라진다. tone.md가 정본, 원페이지는 파생임을 명시.
- **없는 자산 임의 확정** — 보조 컬러가 없는데 어울리는 색을 정해서 채우는 것. [미확보]로 남기고 결정을 요청한다.

## 활용
- **외주 킥오프 표준 첨부**: 094 외주 브리프에 이 원페이지를 기본 첨부 — "톤이 다르다"는 수정 왕복이 규정 위반 지적으로 바뀐다.
- **카피 스킬 정합성 유지**: tone.md 갱신이 반영되면 021~040·043 콘텐츠·카피 스킬 전체가 다음 실행부터 새 규정을 자동 추종한다.
- **협업·입점 심사 대응**: 플랫폼 입점·공동 마케팅 때 "브랜드 가이드 주세요" 요청에 원페이지 1장으로 즉답 — 087 제휴 제안 팩의 부속 문서로도 쓴다.

> 부록 — 외주 셀프 검수 체크리스트 7항 (원페이지 하단 부착용 표준)
> - [ ] 로고: 제공 원본 3종 중 하나를 무변형 사용 (비율·색·회전·효과 ✗)
> - [ ] 로고 최소 크기·여백 규칙 준수
> - [ ] 컬러: 지정 HEX만 사용, 임의 파생색 ✗
> - [ ] 배경-텍스트 대비 규칙(접근성) 확인
> - [ ] 폰트: 제목·본문 지정 폰트 (미보유 시 대체 폰트만)
> - [ ] 카피: "안 쓰는 말" 목록 검색 0건
> - [ ] 금기 표현(profile.md §6: 보장·과장) 0건
