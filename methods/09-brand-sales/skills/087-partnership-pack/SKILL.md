# 087 파트너십·제휴 제안 팩

받는 쪽(084)이 아니라 **우리가 먼저 문을 두드리는** 제안이다. 084의 분석→Theme→구조 파이프라인을 재활용하되, 핵심은 상호 이익 구조 설계 — 상대가 얻는 것을 먼저 계산하고, 수익 구조를 3형(고정/성과/혼합)으로 짜서 협상 여지를 만든다.

## Contract
1. 제안서에서 **상대가 얻는 것이 우리가 얻는 것보다 먼저, 더 구체적으로** 제시된다 (상대 KPI 기준 기대 효과 수치 포함 — 추정이면 [추정] 태그).
2. 수익 구조는 반드시 3안으로 설계된다: A 고정비형 · B 성과 셰어형 · C 혼합형 — 각 안에 정산 기준·산식·리스크 분담·정산 주기가 명시된다.
3. 상호 자산 맵(우리가 주는 것 ↔ 받는 것)이 제안서 본문보다 먼저 확정된다.
4. 상대 브랜드 자산(로고·콘텐츠·고객 데이터) 사용 조건과 종료 조항이 제안서에 포함되고 게이트에서 점검된다.
5. 최종본은 compliance-gate 를 통과한 것만 전달된다 (상대 브랜드 언급·성과 추정치 근거 검사).

## Phases
1. **제휴 대상 분석** — 상대의 비즈니스 모델·주력 채널·최근 행보에서 상대의 KPI(그들이 지금 키우려는 지표)를 추정한다. 공개 정보 기반, 불확실 항목은 [추정] 태그. 우리 쪽 목표(신규 고객 유입/신뢰 이전/콘텐츠 소재/매출 셰어)도 1순위 1개로 확정.
2. **상호 자산 맵** — 2열 표로 교환 구조를 설계: 우리가 주는 것(고객 접점·콘텐츠·제품·데이터·기술) ↔ 상대에게 받는 것. 교환이 한쪽으로 기울면 기울기를 명시하고 보정 수단(수익 셰어율·비용 분담)을 딜 구조로 넘긴다.
3. **딜 구조 3안 설계** — 수익 구조 옵션 3형:
   - **A 고정비형** — 정액 협찬·입점비·라이선스. 예측 가능, 성과 무관. 상대 리스크 최소 → 첫 거래·신뢰 낮을 때.
   - **B 성과 셰어형** — 판매·전환 기준 n% 셰어 (추적 수단: 코드·링크·전용 SKU 명시). 리스크 공유 → 상호 신뢰·측정 인프라 있을 때.
   - **C 혼합형** — 최소 보장 고정 + 초과 성과 셰어. 대부분의 첫 제휴에서 권장 기본안.
   - 각 안에 손익 시뮬레이션 1줄(우리 마진 기준, profile.md 수익 구조 대조)을 붙인다.
4. ⏸ **구조 선택** — 자산 맵 + 3안 비교표를 제시하고 사용자가 주력 안(과 협상 후퇴선)을 선택한다. 선택 전 제안서 작성 금지.
5. **제안서 작성** — 084 SPARK 구조 축약(5부): 상대에게 의미 있는 기회 → 콜라보 컨셉 → 상대가 얻는 것 → 실행 플랜(역할·일정·브랜드 자산 사용 조건·종료 조항) → 제안 조건(주력 안 제시, 나머지는 "협의 가능 옵션"으로 병기). 협상 카드 3줄(양보 가능/불가/교환 조건)을 내부용으로 별첨.
6. ⏸ **compliance-gate 통과** — 상대 브랜드 표기·성과 추정 근거·권리 조항 검사 후 판정 블록을 부착한 최종본만 전달한다.

## Anti-Patterns
- **우리 이득 선행** — "귀사의 채널에 저희 제품을 노출하고 싶습니다"로 시작하는 제안은 읽히지 않는다. 상대 KPI 문장이 첫 문단.
- **단일 딜 구조 통보** — 옵션 없이 조건 1개만 던지면 협상이 수락/거절 이지선다가 된다. 3안이 협상 공간을 만든다.
- **측정 불가 성과 셰어** — 추적 수단(코드·링크·SKU) 없는 B안 제시 금지. 정산 분쟁의 씨앗.
- **협상 카드 동봉 발송** — 내부용 후퇴선을 제안서에 실어 보내는 것. ④는 항상 별도 문서.
- **종료 조항 누락** — 잘될 때만 설계하고 헤어질 조건(기간·해지·자산 회수)이 없으면 게이트 ⚠️ 대상.

## 활용
- **브랜드 콜라보**: 인접 카테고리 브랜드와 공동 기획전 — 자산 맵에 고객 접점 교환, C 혼합형으로 첫 딜 후 B 전환.
- **채널 제휴**: 교육 그룹 ↔ 커뮤니티/뉴스레터 운영자 — 콘텐츠 제공(우리) ↔ 수강생 유입(상대), 성과 셰어는 전용 코드로 추적.
- **협찬 받는 입장 전환**: 광고주가 우리 채널에 협찬을 제안해 오면 이 스킬의 자산 맵·3안을 역방향으로 적용해 조건을 설계하고, 콘텐츠 기획은 088로 넘긴다.
