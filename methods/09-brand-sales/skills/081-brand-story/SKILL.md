# 081 브랜드 스토리·미션 빌더

> 창업 스토리 문답 → **계기→신념→약속** 내러티브 → 브랜드 미션 문장까지. 산출물은 [brand/tone.md](../../../../brand/tone.md) 상단(보이스 형용사·서사 기반)을 갱신하는 입력이 된다 — 이 스킬이 만든 서사가 이후 모든 카피 스킬의 목소리 근거다.
> 계승: marketing-os `brand-voice`(톤·어휘·금기어 5요소 허브 설계 — "기준이 머릿속에만 있어서 톤이 흔들린다, 문서로 외부화하라") + 패스트캠퍼스 실습 5-1 brand-guidelines의 서사·페르소나 추출 파트.
> **082와 경계**: 081 = 서사·미션 (브랜드의 "왜") / 082 = 로고·컬러·보이스 사용 규정 (브랜드의 "어떻게"). 규정 문서화가 필요하면 082로.

## Contract
- 내러티브의 모든 요소는 문답에서 확인된 **실제 사건·사실**에서만 나온다 — 답변에 없는 극적 장면을 지어내지 않는다 (창작이 필요하면 사용자에게 사실 여부를 되묻는다).
- 내러티브는 **계기(사건) → 신념(그래서 믿게 된 것) → 약속(고객에게 하는 것)** 3막 구조를 따르고, 각 막에 근거 문답 번호가 태깅된다.
- 미션 문장은 고객 관점 언어로 쓴다 — 주어가 "우리"여도 술어는 고객이 얻는 결과다 ("업계 선도" 같은 자사 자랑 금지). 본 문장 1 + 길이 변형 2안(한 줄 슬로건용·소개 문단용).
- 서사에서 보이스 형용사 3개 후보를 도출해 tone.md §1 형식으로 제안한다 — 스토리와 톤이 한 몸이 되게.
- 산출물이 tone.md 상단 갱신과 연동됨을 명시하고, 갱신은 사용자 승인 후에만 반영한다 ⏸.

## Phases
1. **기존 자산 로드** — [brand/profile.md](../../../../brand/profile.md) §1 운영자 경력·§3 포지셔닝 자동 참조. [brand/tone.md](../../../../brand/tone.md) 기존 내용 확인 — 이미 서사가 있으면 "신규 작성"이 아니라 "개정"으로 모드 전환.
2. ⏸ **창업 스토리 문답** — 7문항 인터뷰 (한 번에 다 묻지 않고 2~3개씩): ① 이 일을 시작하게 만든 구체적 사건은? ② 그때 무엇이 가장 불만/부당했나? ③ 그만둘 뻔한 순간과 그래도 계속한 이유는? ④ 첫 고객(또는 첫 성공)의 장면은? ⑤ 절대 타협하지 않는 것 1가지는? ⑥ 고객이 우리를 쓰고 나서 달라져야 하는 것은? ⑦ 10년 뒤에도 변하지 않을 것은?
3. **내러티브 구조화** — 답변을 3막에 배치: 계기(①②④) → 신념(③⑤) → 약속(⑥⑦). 막마다 근거 문답 태그 + 구체 장면 1개 포함. 300~500자 본문 + 3줄 요약본 동시 산출.
4. **미션 문장 작성** — "{누구}가 {어떤 상태}가 되도록, 우리는 {무엇}을 한다" 골격으로 본 문장 1 + 변형 2안. 각 안에 신념과의 연결 근거 1줄.
5. **보이스 형용사 도출** — 서사에서 자연스럽게 나오는 태도 3개를 형용사로 (예: 실증적 — "약속 못 드립니다, 보여드립니다"의 태도). 기존 tone.md §1과 충돌하면 차이를 표로 제시.
6. ⏸ **tone.md 갱신 승인** — 상단 갱신안(보이스 3형용사 + 서사 기반 주석)을 diff 형태로 제시 → 승인 시 반영 안내 → 체인: 082 브랜드 가이드라인 문서화 (이 서사를 사용 규정 원페이지로 확장).

## Output Format
```
## 브랜드 스토리·미션 — {브랜드}
판정 요약: 내러티브 3막 (근거 문답 {n}개) · 미션 본문 1 + 변형 2 · 보이스 후보 {형용사 3} · tone.md {신규|개정}

### ① 브랜드 내러티브 (300~500자 + 3줄 요약)
**계기** [문답①②④] {구체 사건 장면} → **신념** [③⑤] {믿게 된 것} → **약속** [⑥⑦] {고객에게 하는 약속}

### ② 미션 문장
| 안 | 문장 | 용도 | 신념 연결 |
|---|---|---|---|
| 본문 | {누구}가 {상태}가 되도록 … | 소개·모집 페이지 | … |
| 변형A | (한 줄 슬로건) | 프로필·배너 | |
| 변형B | (소개 문단) | 회사소개·제안서 | |

### ③ 보이스 형용사 3 후보 (tone.md §1 형식) + 기존과의 차이
### ④ tone.md 상단 갱신안 (diff) ⏸

다음 액션: → 082 브랜드 가이드라인 문서화 (서사·보이스를 사용 규정 원페이지로)
```

## Anti-Patterns
- **감동 창작** — 답변에 없는 "차고에서 밤새며…" 장면을 지어내는 것. 사실이 아닌 서사는 첫 인터뷰에서 무너진다.
- **자사 자랑 미션** — "최고의 기술로 업계를 선도한다"는 미션이 아니라 보도자료다. 고객의 변화가 술어에 와야 한다.
- **어디에나 붙는 문장** — 경쟁사 이름을 넣어도 말이 되는 미션은 실패다. 문답의 고유 사건·신념이 문장에 남아야 한다.
- **스토리와 톤 분리 납품** — 서사 따로, 보이스 따로 만들면 카피 스킬들이 참조할 때 다시 어긋난다. 형용사 도출까지가 계약.
- **tone.md 무단 덮어쓰기** — 기존 검증된 톤 문서를 승인 없이 교체하는 것. 갱신은 diff 제시 → 승인 후.

## 활용
- **모집 페이지·회사소개 개편**: 내러티브 300자를 랜딩 첫 섹션에, 미션 변형B를 회사소개에 — 051 상세페이지·083 보도자료의 "회사 소개" 단락이 전부 이 문서에서 나온다.
- **채용·협업 온보딩**: 신념·약속 파트를 외주·신규 멤버 온보딩 첫 문서로 — "우리가 안 하는 것"의 근거가 서사로 설명된다.
- **콘텐츠 일관성 점검**: 028 링크드인·024 유튜브 스크립트가 이 서사의 신념과 어긋난 주장을 하는지 분기별로 대조한다.
- **피칭·지원서 재료**: 내러티브 3막을 085 세일즈 덱의 도입부, 090 지원서의 기업 소개 항목에 이식 — 서사가 한 번 만들어지면 문서마다 다시 쓰지 않는다.

> 부록 — 문답 → 3막 매핑표 (구조화 단계 참조)

| 막 | 원천 문답 | 채워야 할 것 |
|---|---|---|
| 계기 | ①②④ | 구체 사건 장면 1개 (날짜·장소·인물) |
| 신념 | ③⑤ | 타협하지 않는 것 1문장 |
| 약속 | ⑥⑦ | 고객의 변화가 술어인 문장 |
