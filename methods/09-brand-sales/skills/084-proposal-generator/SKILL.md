# 084 제안서 자동 생성

문의·RFP 한 건을 **수주 논리가 관통하는 맞춤 제안서**로 바꾼다. 검증 파이프라인 v2 계승: RFP 분석 → Win Theme → SPARK-6 목차 → 섹션 초안. "우리 회사 소개서" 재탕이 아니라, 고객의 평가 기준에 역산해 쓴다.

## Contract
1. 제안서는 반드시 **RFP·문의 분석 이후**에 작성된다 — 요구사항·평가 기준·Pain Point 3개 이상을 추출하기 전에는 본문을 쓰지 않는다.
2. Win Theme(수주 전략 메시지) 3개가 먼저 확정되고, 모든 섹션 초안에 어느 Theme을 뒷받침하는지 태깅된다.
3. 목차는 SPARK-6 구조(문제 재정의→기회→해법→실행→증거→투자 대비 효과)를 따르며, 고객 문의 문구를 최소 2회 직접 인용해 맞춤성을 증명한다.
4. 모든 섹션 제목은 Action Title(인사이트+데이터 문장형, 15~35자)로 쓴다 — "회사 소개" 같은 Topic Title 금지.
5. 가격·조건·성과 표현이 포함된 최종본은 compliance-gate 를 통과한 것만 전달된다 (보장성 표현·근거 없는 수치 전수 검사).

## Phases
1. **입력 분석 (RFP/문의)** — RFP 파일 또는 문의 내용에서 추출: 요구 과업 목록 · 평가 기준(배점 있으면 배점순) · 예산·일정 제약 · 행간의 Pain Point 3개 이상. RFP가 없는 구두 문의면 문의 문장을 같은 틀로 구조화하고 부족 항목은 [확인 필요] 태그.
2. **Win Theme 도출** — Pain Point ↔ 우리 강점(brand/profile.md)을 교차해 수주 전략 메시지 3개를 만든다. 각 Theme = "고객의 {문제}에 우리는 {차별 근거}로 답한다" 1문장 + 지지 증거 1개.
3. ⏸ **Win Theme 승인** — 3개 Theme과 분석 요약을 제시하고 사용자가 확정/수정한다. 승인 전 목차·본문 작성 금지 (여기가 제안서의 뼈대다).
4. **SPARK-6 목차 설계** — 6부 구조로 목차를 만들고 각 섹션에 ① Action Title ② 담을 핵심 메시지 1줄 ③ 대응 Win Theme 번호 ④ 근거 자료를 배정한다. 평가 기준 배점이 높은 항목에 분량을 비례 배분.
5. **섹션 초안 작성** — 섹션별로 S-E-P 설득 구조(Story 공감 → Evidence 출처 있는 근거 → Promise 행동 유도)로 초안을 쓴다. 수치는 출처·기준 시점 병기, 없는 수치는 [추정] 태그. 고객 문의 문구 인용 2회 이상 포함.
6. **커버레터 작성** — 제안서 요지 3줄 + 다음 스텝 제안(미팅·데모)을 담은 발송용 커버레터 1통 (제목 2안).
7. ⏸ **compliance-gate 통과** — gates/compliance-gate.md 3 STEP 실행. "100% 보장" "최저가" 등 보장성·최상급 표현 검사 후 판정 블록을 부착한 최종본만 전달한다.

## Anti-Patterns
- **회사 소개서 재탕** — 우리 자랑 순서(연혁→조직→실적)로 쓰지 않는다. 고객 문제 재정의가 항상 첫 섹션.
- **분석 없이 집필** — RFP를 읽는 척만 하고 범용 제안서를 내는 것. 평가 기준·Pain Point 추출물이 없으면 Phase 2로 못 간다.
- **Topic Title** — "사업 이해", "수행 방안" 같은 명사형 제목 금지. 모든 섹션 제목은 데이터·인사이트가 담긴 문장형.
- **Win Theme 미관통** — Theme을 서두에만 쓰고 본문 섹션과 태깅 연결이 없으면 장식이다. 태깅 안 되는 섹션은 삭제 후보.
- **근거 없는 수치 확정** — 시장 수치·성과 수치를 출처 없이 본문에 박으면 게이트 차단 대상. [추정]·[확인 필요] 태그로 구분한다.

## 활용
- **B2B 서비스 수주**: 인바운드 문의 → 당일 Win Theme 승인 → 익일 제안서 발송. 086 미팅 노트가 있으면 Pain Point 추출 입력으로 재사용 (/세일즈파이프라인 체인).
- **공공·대기업 입찰**: RFP 평가 배점표를 그대로 목차 분량 배분에 반영 — 배점 높은 항목이 얇은 제안서를 구조적으로 방지.
- **제휴·지원사업 파생**: 같은 파이프라인이 087(제휴 제안)·090(지원사업 서류)의 뼈대 — 분석 대상만 브리프/공고문으로 바뀐다.
