# 083 보도자료 작성

"우리 소식" 을 "기자의 기사거리" 로 바꾼다. 뉴스 가치 앵글 도출 → 역피라미드 표준 보도자료 → 매체 배포 리스트까지. 자랑문이 아니라 기사 원고에 가까운 상태로 내보낸다.

## Contract
1. 보도자료는 반드시 **뉴스 가치 앵글 선정 이후**에 작성된다 — 앵글 없는 소식 나열문은 만들지 않는다.
2. 본문은 역피라미드 구조를 지킨다 — 리드 문단만 읽어도 기사를 쓸 수 있는 상태 (5W1H 완결).
3. 모든 수치 주장에는 출처·기준 시점이 붙는다. 근거 없는 수치는 본문에서 제외하고 "근거 필요" 로 별도 표기.
4. 인용문(대표·담당자 코멘트)이 최소 1개 포함되고, 발화자의 실명·직함 확인을 사용자에게 받는다.
5. 최종본은 compliance-gate 를 통과한 것만 전달된다 (보도자료는 대외 발행물 — 최상급·보장성 표현 전수 검사).

## Phases
1. **브랜드 컨텍스트 로드** — brand/profile.md 에서 브랜드명·업종·포지셔닝·공식 채널을 자동 참조한다 (회사 소개 재질문 금지 — 보일러플레이트 초안도 여기서 생성).
2. **뉴스 가치 앵글 도출** — 소식 1건을 5개 렌즈로 검사해 앵글 후보 3안 작성:
   - 시의성(지금 왜) · 최초성/희소성(업계 처음인가) · 데이터(숫자가 이야기인가) · 사회적 맥락(트렌드·정책과 연결되나) · 영향 범위(누구의 문제를 푸나)
   - 각 앵글에 가제(헤드라인) + 예상 관심 매체 유형을 붙인다.
3. ⏸ **앵글 선택** — 3안 중 사용자가 1개를 선택 (또는 조합 지시). 선택 전에는 본문을 쓰지 않는다.
4. **표준 보도자료 작성** — 역피라미드로:
   - 헤드라인(수치 포함 권장) → 서브헤드 → 리드(5W1H 한 문단) → 본문 2~3문단(근거 수치·맥락) → 인용문 1~2개 → 보일러플레이트 → 미디어 문의처
   - 사진·자료 제안 2건(캡션 포함) 병기.
5. **배포 리스트 + 피칭 메일** — 업종·앵글 기준으로 매체 카테고리별 배포 리스트(전문지·경제지·지역지·뉴스와이어 등 8~12곳 골격)와 기자 피칭 메일 1통(제목 2안)을 작성한다. 실제 발송은 사용자가 직접.
6. ⏸ **compliance-gate 통과** — gates/compliance-gate.md 3 STEP 실행. "국내 최초" "1위" 등 최상급 표현은 근거 자료 확보 여부를 확인하고, 게이트 결과 블록을 부착한 최종본만 전달한다.

## Anti-Patterns
- **자랑문 작성** — "혁신적인" "업계 최고의" 로 시작하는 형용사 더미는 기자가 첫 줄에서 닫는다. 사실·숫자·맥락으로.
- **앵글 없이 집필** — "출시했다" 는 뉴스가 아니다. 왜 지금, 누구에게 의미 있는지가 정해지기 전엔 본문 금지.
- **리드에 정보 아끼기** — 핵심을 마지막에 두는 기승전결 구조 금지. 역피라미드: 리드에 전부, 아래로 갈수록 상세.
- **가짜 인용문 확정** — 담당자 확인 없는 코멘트를 확정본처럼 내보내지 않는다 (⏸ 실명·직함 확인 필수).
- **근거 없는 최상급** — "국내 최초" "판매 1위" 를 근거 자료 없이 사용하면 게이트 차단 대상이자 표시광고법 리스크.

## 활용
- **신제품 출시**: 출시 정보 → 데이터 앵글("출시 3주 만에 재구매율 n%")로 전환해 전문지 + 뉴스와이어 배포.
- **소기업 첫 PR**: 보도자료 1건을 027 OSMU 멀티채널 변환기로 블로그·링크드인·뉴스레터까지 확장 — 한 소식으로 4개 채널.
- **제휴·투자 발표**: 양사 공동 배포용으로 인용문 2개(양사 대표) 구조 적용, 상대사 검수 버전을 별도 산출.
