# 047 광고 소재 A/B 판독기

> A/B 테스트 결과를 Bayesian 승리 확률로 판독 — "B가 좀 더 잘 나왔다 ≠ B의 승리." 확정승·박빙·표본부족 3판정 + 다음 변형 제안.
> 계승: 패스트캠퍼스 6-6 ab-test-analyzer — Bayesian 3판정 · 게이트 순서(표본 50건 먼저→확률 95% 나중) · 사전 기준 고정 · 승리 패턴 일반화 적재.
> **경계**: 019 A/B 테스트 설계는 테스트 **시작 전**(가설→변수→표본수→판정 기준), 047은 테스트 **종료 후 판독**. 설계서 없이 온 데이터는 설계 검증부터 통과해야 한다.

## Contract
- 판정은 **확정승/박빙/표본부족 3종만** 낸다. 게이트 순서 고정: ① 표본(전환 합계 ≥ 50건) → ② 확률(승리 확률 ≥ 95%) — 표본 미달이면 확률을 계산하지 않는다.
- 판정 지표는 **1개**이며 데이터를 보기 전에 확정한다 (019 설계서가 있으면 그 값 승계) — 데이터를 본 뒤 지표 변경 금지 (사후 조작 방지).
- ROAS/CPA 판정 시 승패와 무관하게 두 안 모두 [brand/profile.md](../../../../brand/profile.md) §4 **손익분기 ROAS** 대비를 병기한다 — 이긴 안도 적자일 수 있다. profile 미작성 시 BEP 1.0 폴백 + `[추정]` 태그.
- 표본부족 판정에는 필요 표본수와 현재 속도 기준 예상 추가 기간을 수치로 제공한다.
- 확정승은 재사용 가능한 승리 패턴 1줄("긴급형 소구 > 혜택형 · CVR +37%")로 일반화하고 다음 변형 3개를 제안한다.

## Phases
1. **설계 검증** — A=챔피언(현행)·B=챌린저(실험) 확인. **단 한 변수만** 다른가(헤드라인만/형식만/LP만) — 2개 이상 다르면 판독 불가로 반려하고 019로 안내. 예산·타깃·기간·게재위치 동일성 확인.
2. ⏸ **판정 기준 고정** — 지표 1개 확정: 구매·매출 목표→ROAS/CPA · 전환·리드→CVR · 유입→CTR. 종료 조건: 최소 표본(전환 합계 50건 기본)·승리 임계(95% 기본). 019 설계서 값이 있으면 그대로, 없으면 사용자 선택("추천대로"=CVR·50건·95%). CTR·CPC 등 나머지는 진단용(secondary)으로만.
3. **데이터 로드** — 안별 노출·클릭·전환·광고비. 기간·모수 정합성 검증 (한쪽만 기간이 길면 반려).
4. **Bayesian 승리 확률 계산** — 안별 전환율의 Beta 사후분포에서 B>A 확률 산출. 신뢰구간·효과 크기(상대 차이 %) 병기.
5. **3판정** —

| 판정 | 조건 | 액션 |
|---|---|---|
| ✅ 확정승 | 표본 ≥ 50건 + 승리 확률 ≥ 95% | 승자 채택 + 승리 패턴 적재 |
| ⚖️ 박빙 | 표본 ≥ 50건 + 확률 < 95% | 이 차원 종료 — 다른 차원(소구·타깃) 테스트 권고 |
| ❌ 표본부족 | 전환 합계 < 50건 | 판정 보류 — 필요 표본수·추가 기간 제시 |

6. **BEP 대조** — 두 안의 ROAS를 profile §4 BEP와 대조 — 승자가 BEP 미달이면 "채택하되 증액 금지" 단서를 판정문에 부착.
7. **다음 변형 제안** — 확정승: 승리 요인 가설 기반 변형 3개(승리 축 강화 1 + 인접 축 2) → 043 메타 광고 소재 카피 팩으로 제작. 박빙: 차원 교체 방향 제시. 판정문에는 항상 "다음 행동"이 동반된다.

## Output Format
```

판정: {✅ 확정승 | ⚖️ 박빙 | ❌ 표본부족} · 승리 확률 {n}% · 효과 크기 {±n}%

### ① 판정 기준 (재현용): 표본 게이트 ≥ 50건 → 확률 게이트 ≥ 95% · 지표 {n} · 기간 {n}
### ② 데이터 요약 (안별 노출·클릭·전환·광고비·{지표} · BEP {n} 대비)
### ③ 판정 근거 — {표본 충족 여부 → 승리 확률·신뢰구간}
### ④ 다음 행동
- 확정승: {B안 채택 · 패턴 "{승리 패턴 1줄}" · 변형 제안 3개}
- 박빙: {이 차원 종료 → {차원} 테스트 권고} / 표본부족: {필요 전환 {n}건 · 약 {n}일 추가}

```

## Anti-Patterns
- **눈대중 승자 선언** — "B가 3건 더 높으니 B 채택"은 우연을 승리로 오판해 잘못된 학습을 누적시킨다. 확률로만 말한다.
- **데이터 본 뒤 지표 갈아타기** — CVR로 지던 안이 CTR로 이기면 CTR로 판정하는 것. 사전 확정 지표 외 판정 금지.
- **표본 부족인데 %가 커서 조기 종료** — "+45%"도 전환 합계 27건이면 침묵이 정답이다. 표본 게이트가 항상 먼저.
- **박빙 차원 재테스트 반복** — 형식이 박빙이면 형식을 또 돌리지 말고 소구·타깃 차원으로 넘어간다.
- **CTR 승자를 매출 승자로 확대 해석** — 클릭을 이기고 BEP를 지는 소재가 흔하다. BEP 대조 없는 채택 권고 금지.

## 활용
- **주간 테스트 판정 루틴**: 매주 화요일 진행 중 테스트를 일괄 판독 — 확정승만 채택·나머지는 게이트 사유와 함께 보류, 회의에서 "몇 % 확률"로 보고.
- **설계→판독→재생성 루프**: 019 설계서로 테스트 시작 → 047 판독 → 승리 패턴을 043에 넘겨 다음 변형 제작 — 테스트가 자산으로 쌓이는 순환.
- **승리 패턴 라이브러리**: 확정승의 일반화 문장을 누적 기록 — 신규 캠페인 카피 작성 시 검증된 소구 축부터 착수.
- **박빙 기록의 재활용**: 박빙 판정도 버리지 않는다 — "형식 차원은 차이 없음"은 그 차원에 예산·시간을 다시 쓰지 않게 하는 음의 자산이다.

> 참고 — 확률 계산 기본형: 안별 전환을 Beta(전환+1, 클릭-전환+1) 사후분포로 두고 몬테카를로 표본에서 P(B>A) 산출. 판정 임계·표본 게이트는 산출물 ①에 기록 (재현성).
