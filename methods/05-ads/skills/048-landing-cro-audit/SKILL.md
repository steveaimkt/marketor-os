# 048 랜딩페이지 CRO 진단

> 051 상세페이지 24섹션 기준을 **역진단 체크리스트로 뒤집어** 기존 페이지의 전환 저해 요소를 4축(첫 화면·신뢰·마찰·CTA)으로 진단 → 임팩트순 처방.
> 051 상세페이지 24섹션 구조의 역적용 — 랜딩 전환 저해 요소를 진단·실행형으로 분석.
> **경계**: 051은 새로 짓기(생성), 048은 있는 페이지 뜯어보기(진단·처방). 결손이 12섹션 이상이면 부분 수선 대신 051 전면 재구축을 권고한다.

## Contract
- 진단은 **4축 × 24섹션 역진단 체크리스트**로 수행하고, 축마다 결손·약화 섹션 번호를 명시한다 — 축·섹션·근거 없는 감상평 금지.
- 모든 지적에는 개선안(수정 카피 초안 또는 구조 이동안)을 병기한다 — 문제 지적만 하고 끝내지 않는다.
- 처방은 임팩트(전환 영향) × 난이도 매트릭스로 정렬해 **TOP 5만** 우선 제시한다 — 20개 나열 금지.
- 전환 데이터가 있으면 병목 구간 실측을 우선하고, 없으면 구조 기반 추정에 `[추정]` 태그를 붙인다.
- 광고 유입 랜딩의 개선 목표 CVR은 [brand/profile.md](../../../../brand/profile.md) §4 **손익분기 ROAS**에서 역산한다 (목표 CVR = CPC ÷ (AOV × 마진율) 기준 명시) — 미작성 시 `[추정]`.

## Phases
1. **입력 수집** — 랜딩 URL 또는 풀페이지 스크린샷, 전환 데이터(GA4·광고 관리자), 유입 광고 소재. profile.md §4 자동 참조.
2. **구조 분해** — 현재 페이지를 051의 24섹션 기준과 대조 → 섹션별 [있음/약함/없음] 결손 맵 작성. 광고 소재가 있으면 소재 약속 vs 첫 화면 메시지 정합(Scent) 검사.
3. **4축 진단** — 축별 핵심 질문으로 저해 요소 검출:

| 축 | 대응 섹션(051) | 핵심 질문 |
|---|---|---|
| ① 첫 화면 | 01 Hook · 02 정의 | 3초 안에 무엇을·누구에게·왜가 읽히는가, 광고 약속과 이어지는가 |
| ② 신뢰 | 03 브랜드 · 17 비교 · 18 안전 · 20 후기 | 수치에 출처·n이 있는가, 후기·인증이 실물인가 |
| ③ 마찰 | 06→07 공감→해결 · 08 개요 · 21 스펙 · 22 FAQ | 설득 리듬이 끊기는 지점·정보 과부하·모바일 가독성 |
| ④ CTA | 24 CTA (+ 04 구성품 · 23 보증) | 위치·빈도·문구가 행동을 지시하는가, 구매 불안(배송·환불)이 CTA 옆에서 해소되는가 |

4. **저해 요소 목록화** — 축·섹션 번호·근거(스크린샷 위치 또는 데이터)·개선안을 1행씩. 전환 데이터가 있으면 스크롤 도달·이탈 구간과 대조.
5. **우선순위 처방** — 임팩트(상/중/하) × 난이도(카피 수정<섹션 추가<구조 개편)로 TOP 5 선정. 첫 화면 축 결함은 무조건 1순위 후보 (이탈 대부분이 첫 화면에서 발생).
6. ⏸ **처방 확정** — TOP 5 승인 후 분기: 개선안 검증은 019 A/B 테스트 설계로, 결손 12섹션+ 전면 재구축은 051로.

## Output Format
```
## 랜딩페이지 CRO 진단 — {페이지명} · [실측|추정]
판정 요약: 24섹션 중 결손 {n} · 약함 {n} · 최대 병목: {축} ({섹션#}) · 목표 CVR {n}% (BEP 역산, profile.md §4)

### ① 24섹션 결손 맵 (있음 {n} / 약함 {n} / 없음 {n} — 섹션 번호 나열)
### ② 4축 진단표
| 축 | 저해 요소 | 근거(위치·데이터) | 개선안 | 임팩트 |
### ③ 우선순위 처방 TOP 5 ⏸ (임팩트×난이도 · 처방별 예상 효과)
### ④ 판정 기준 (재현용): 24섹션 대조 기준 · CVR 역산식 · 데이터 출처

다음 액션: → 019 A/B 테스트 설계 (처방 1번 검증) / 051 상세페이지 24섹션 빌더 (결손 12+ 전면 재구축)
```

## Anti-Patterns
- **감상 진단** — "좀 밋밋해요"는 진단이 아니다. 축·섹션 번호·근거 없는 지적은 산출물 자격 미달.
- **전부 고치기 처방** — 우선순위 없는 20개 목록은 아무것도 안 고쳐진다. TOP 5 원칙.
- **첫 화면 놔두고 하단 손보기** — 방문자 과반이 첫 화면에서 이탈하는데 FAQ부터 다듬는 것. 축① 결함이 항상 선순위.
- **광고 소재 무시** — 소재가 약속한 것과 첫 화면이 다르면(Scent 단절) 페이지 내부를 아무리 고쳐도 이탈은 못 막는다.
- **진단으로 종료** — 개선안은 아직 가설이다. 019 테스트 연결 없이 "고쳤으니 끝"은 CRO가 아니라 리디자인.

## 활용
- **광고 성과 부진 원인 분리**: 041/042 분석에서 CVR 병목 판정이 나오면 본 스킬로 페이지 쪽 원인을 특정 — 소재 문제(043)와 페이지 문제(048)를 분리 대응.
- **리뉴얼 전 근거 확보**: 디자인 리뉴얼 품의 전에 결손 맵 + TOP 5를 첨부 — "감으로 리뉴얼"을 "진단 기반 개선"으로 전환.
- **월간 CRO 루프**: 매월 처방 1개 → 019 테스트 → 승자 반영 → 재진단 — 페이지가 매월 한 칸씩 좋아지는 운영 루틴.
- **경쟁 페이지 벤치마크**: 같은 4축 체크리스트를 경쟁사 랜딩에 적용 — 우리 결손 맵과 겹쳐 보면 "경쟁은 채웠는데 우리만 빈 섹션"이 우선 보강 후보가 된다.

> 참고 — 목표 CVR 역산 기본형: 손익분기 CVR = CPC ÷ (AOV × 마진율). 실제 적용값·가정은 산출물 ④에 기록 (재현성).
