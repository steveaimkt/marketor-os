# 050 어트리뷰션 정리

> 갈라진 UTM을 하나의 명명 체계로 통일(소스/매체/캠페인 규칙 표 + 기존 태그 매핑)하고, 채널 기여를 라스트클릭의 한계까지 포함해 해석하는 기준을 세운다. "측정 체계가 없으면 046 예산 재배분은 소음 위에서 결정된다."

## Contract
- UTM 명명 규칙은 **소문자·하이픈 구분·한글 금지·날짜 yymm** 강제 규칙 표로 제공한다 — 예외 허용 시 분열이 재발한다.
- `utm_medium`은 GA4 기본 채널 그룹에 매핑되는 표준값(cpc·paid_social·email·referral·organic_social·affiliate)만 사용한다 — 임의 창작값("sns"·"광고") 금지.
- 기존 태그 → 신규 체계 **매핑표**를 반드시 동봉한다 (과거 데이터 해석 연속성 확보).
- 기여 해석 가이드에는 라스트클릭 모델의 한계(상단 퍼널 과소평가)와 보정 관점을 명시한다 — 단일 모델 수치를 진실로 단정하는 서술 금지.
- 채널 성과의 흑자/적자 판정 기준은 [brand/profile.md](../../../../brand/profile.md) §4 **손익분기 ROAS** — 미작성 시 `[추정]` 태그.

## Phases
1. **현행 감사** — GA4 유입 소스/매체 내보내기 또는 운영 중 링크 목록 수집 → 중복·변형 표기 검출 (예: `Facebook`/`facebook`/`fb` 3분열, medium 미분류 비율).
2. **표준 명명 체계 설계** — 운영 채널 전체를 규칙 표로:

| 채널 | utm_source | utm_medium | utm_campaign 규칙 | utm_content |
|---|---|---|---|---|
| 메타 유료 | facebook / instagram | paid_social | {yymm}-{목표}-{프로모명} | {소재 식별자} |
| 구글 검색 | google | cpc | {yymm}-{캠페인 유형}-{테마} | {광고그룹} |
| 뉴스레터 | newsletter | email | {yymm}-{호수 또는 시퀀스명} | {버튼 위치} |
| 인플루언서 | {계정명} | influencer | {yymm}-{협업명} | {콘텐츠 유형} |
| 자연 SNS | instagram 등 | organic_social | {yymm}-{시리즈명} | — |

   공통 강제 규칙: 전부 소문자 · 단어 구분 하이픈 · 한글/공백/특수문자 금지 · campaign 선두는 yymm.
3. **기존 태그 매핑표** — 감사에서 나온 변형 표기 전건을 [기존값 → 신규값 → 병합 사유] 3열로 정리. GA4에서 소급 수정은 불가하므로 "이후 데이터부터 통일 + 비교 시 매핑표 참조" 원칙 명시.
4. **링크 빌더 템플릿** — 채널·캠페인 입력 → 완성 URL 자동 조합 스프레드시트 골격 제공 (규칙을 문서가 아니라 도구로 배포).
5. **채널 기여 해석 가이드** — ① 라스트클릭: 마지막 접점 100% — 상단 퍼널(인지 채널) 과소평가·브랜드 검색 과대평가 ② 보정 관점: 데이터 기반 어트리뷰션(GA4 DDA) 병행 열람 · 매체 자체 리포트의 뷰스루 중복 주의 · 브랜드 검색/직접 유입 증가를 상단 채널 후행 지표로 읽기 ③ 판정: 채널별 ROAS를 profile §4 BEP 대비로만 흑자/적자 판정.
6. ⏸ **체계 확정** — 규칙 표·매핑표 승인 후 전 채널 적용 시작 → 체인: 063 GA4 분석 리포트 (통일된 태그 기준 4주 후 첫 정합 리포트).

## Output Format
```
## 어트리뷰션 정리 — {브랜드} · 채널 {n}개
판정 요약: 현행 태그 {n}종 → 표준 {n}종 통합 · 미분류 유입 {n}% 해소 예상 · 판정 기준 BEP ROAS {n} (profile.md §4)

### ① UTM 명명 규칙 표 (채널 × source/medium/campaign/content + 강제 규칙 5줄)
### ② 기존 태그 매핑표 (기존값 → 신규값 → 병합 사유)
### ③ 링크 빌더 템플릿 (스프레드시트 골격 — 입력 3칸 → URL 자동 조합)
### ④ 채널 기여 해석 가이드 1페이지 (라스트클릭 한계 · 보정 관점 · BEP 판정 기준)
### ⑤ 적용 순서 ⏸ (신규 발행 링크부터 → 상시 채널 링크 교체 → 4주 후 063 정합 점검)

다음 액션: → 063 GA4 분석 리포트 (통일 체계 기준 채널 성과 첫 리포트)
```

## Anti-Patterns
- **표기 혼용 방치** — `Facebook`과 `facebook`은 GA4에서 다른 채널이다. 대문자 하나가 데이터를 반으로 가른다.
- **medium 창작값** — "sns"·"광고" 같은 임의값은 GA4 기본 채널 그룹에서 Unassigned로 빠져 리포트가 깨진다.
- **내부 링크에 UTM** — 자사 사이트 내부 이동에 UTM을 붙이면 세션이 리셋되어 원 유입 기여가 지워진다.
- **라스트클릭 수치를 채널 진실로** — 인지 채널을 라스트클릭만 보고 감액하면 하단 퍼널까지 마른다. 보정 관점 병기 필수.
- **규칙만 만들고 배포 생략** — 문서만 있는 규칙은 2주 안에 죽는다. 빌더 템플릿 배포 + 발행 프로세스 편입까지가 이 스킬의 완료선.

## 활용
- **캠페인 시즌 전 정비**: 대형 프로모션 전에 체계를 통일 — 시즌 종료 후 "어느 채널이 팔았나"를 매핑표 없이 바로 판독.
- **046 예산 재배분의 전처리**: 채널 기여 데이터가 갈라져 있으면 재배분 근거가 무너진다 — 050으로 측정 체계부터 세우고 046 실행.
- **대행사·팀 온보딩 표준**: 신규 대행사에 링크 빌더 템플릿을 인수인계 문서로 전달 — 발행 주체가 늘어도 체계가 유지된다.
- **분기 정합 감사**: 분기마다 Phase 1 감사만 재실행 — 신규 변형 표기·미분류 유입률을 점검해 체계 부패를 조기에 잡는다.

> 참고 — campaign 명명 기본형: `{yymm}-{목표}-{식별자}` (예: `2607-conv-summer-sale`). 채널별 실제 규칙·예외는 산출물 ①에 기록 (재현성).
