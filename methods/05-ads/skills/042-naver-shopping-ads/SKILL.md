# 042 네이버 쇼핑검색광고 최적화

> 합산 ROAS 하나에 속지 않는다. 브랜드/일반 키워드를 분리 집계해 착시를 해부하고, 일반 키워드에서 키워드·소재·입찰 3축 진단으로 조정안을 낸다.
> 계승: 패스트캠퍼스 6-3 naver-ads-analyzer — 브랜드/일반 분리 집계(합산 1.54 → 브랜드 7.41 vs 일반 0.71 착시 해부) + 데이터랩 교차검증 사분면.

## Contract
- **합산 ROAS 단독 보고 금지** — 모든 리포트는 합산·브랜드·일반 3개 ROAS를 나란히 제시한다.
- 판정 기준은 [brand/profile.md](../../../../brand/profile.md) §4 **손익분기 ROAS**다. "일반 키워드 ROAS < BEP = 적자" 판정, profile 미작성 시 BEP 1.0 폴백 + `[추정]` 태그.
- 브랜드 키워드 판별 규칙(자사명·제품명 포함)을 산출물에 명시한다 — 같은 입력이면 같은 분리 결과.
- 조정 제안(입찰 하향·소재 교체·LP 개선·축소)마다 대상 키워드·근거 수치·예상 효과를 병기한다.
- 조정 실행은 사용자 승인 후 광고시스템에서 수동으로 한다 (본 스킬은 읽기·분석 전용).

## Phases
1. **기준 로드** — profile.md §4 손익분기 ROAS·목표 ROAS + §1 브랜드명·제품명(브랜드 키워드 판별 시드) 자동 참조.
2. **데이터 확보** — [data/connections.md](../../../../data/connections.md) 모드 판정: 검색광고 REST API(공식 MCP 없음 — API_KEY·SECRET_KEY·CUSTOMER_ID 서명 호출) 연결 시 실데이터, **미연결 시 광고주센터 → 보고서 → 대용량 다운로드 CSV 업로드 폴백**. ⚠️ 캠페인 유형 확인 — 쇼핑검색(SHOPPING)형은 키워드 단위가 아닌 상품 단위라 키워드 분리 대신 상품/광고그룹 단위로 동일 로직 적용.
3. **브랜드/일반 분리 집계** — 자사명·제품명 포함 = 브랜드, 그 외 = 일반. 그룹별 광고비·매출·ROAS 집계 후 합산과 나란히 비교. 착시 해부 코멘트 필수 ("브랜드 ROAS가 높은 건 광고 실력이 아니라 이미 살 사람의 방어 입찰").
4. **일반 키워드 3축 진단** — ① 키워드: BEP 미달 적자 키워드 랭킹(지출 큰 순) ② 소재: CTR 저조 그룹 → 소재 교체 후보 ③ 입찰: CPC 대비 전환가치 역전 키워드 → 입찰 하향 후보.
5. **수요 교차검증 (선택)** — 데이터랩 MCP 연결 시 카테고리 대표 키워드(5자 이하) 13개월 트렌드로 사분면 판정: 🔴 수요↓수익↓=중단 / 🟡 수요↑수익↓=컷 금지·진단(가격·LP·리뷰) / 🟢 수요↑수익↑=확대(시즌 피크 예산 집중). 미연결 시 이 단계는 생략하고 명시.
6. ⏸ **조정안 확정** — 키워드별 조치(하향/교체/개선/축소) 리스트를 승인받고 → 체인: 046 ROAS 진단 & 예산 재배분 (매체 간 이동).

## Anti-Patterns
- **합산 ROAS로 "잘 되고 있다" 보고** — 브랜드 키워드가 평균을 끌어올린 착시. 분리 없는 보고는 규격 위반.
- **브랜드 ROAS를 성과로 자랑** — 자사명 검색은 어차피 살 사람. 광고 실력 지표는 일반 키워드 ROAS다.
- **적자 키워드 일괄 컷** — 수요 상승기(🟡) 키워드를 끄면 시즌 매출을 잃는다. 컷 전에 수요 교차검증.
- **쇼핑검색형에 키워드 분리 강행** — SHOPPING 캠페인엔 입찰 키워드가 없다. 상품 단위로 로직을 바꿔 적용.
- **92일 이전 데이터 API 조회 시도** — /stats는 최근 92일만. 과거 분석은 광고주센터 CSV 다운로드 경로로.

## 활용
- **월간 광고 결산**: 매월 초 지난달 CSV로 분리 리포트 → "브랜드 빼면 얼마?"를 경영 보고 첫 줄로.
- **대행사 성과 검증**: 대행사가 보고하는 합산 ROAS를 본 스킬로 분리 재집계 — 일반 키워드 효율로 재계약 판단.
- **시즌 예산 설계**: 데이터랩 사분면에서 🟢 확대 키워드의 피크 월을 확인해 046 재배분 시나리오의 근거로 전달.
- **연관 키워드 확장 리서치**: 광고 데이터가 없어도 검색광고 API /keywordstool 경로로 월 검색량·연관 키워드 확장은 가능 — 신규 캠페인 설계 입력으로.
- **입찰 전 시뮬레이션**: /estimate 예상 입찰가 조회로 신규 키워드의 진입 CPC를 추정 — BEP ROAS 역산으로 감당 가능한 CPC 상한을 먼저 계산.

> 실연동 메모 (connections.md 🟡): 검색광고 API 키 3종(API_KEY·SECRET_KEY·CUSTOMER_ID)은 manage.searchad.naver.com → 도구 → API 사용 관리에서, 데이터랩 키 2종(CLIENT_ID·CLIENT_SECRET)은 developers.naver.com에서 발급 — 발급처가 서로 다르다.
> /stats 성과 조회는 최근 92일 한정 — 과거 분석은 광고주센터 보고서 CSV 다운로드 경로로.
> 쇼핑검색(SHOPPING)형 계정은 산출물 ②·③의 "키워드" 열이 "상품/광고그룹" 열로 치환된다 — 분리 원리는 동일.
