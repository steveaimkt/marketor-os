# 045 광고 주간 리포트 자동화

> 매체별 데이터를 공통 스키마로 취합 → KPI 카드 + 인라인 SVG 차트 3종 → 전주 대비 해석까지 담긴 단일 HTML 주간 리포트.
> 계승: 패스트캠퍼스 6-1 meta-ads-analyzer(KPI 카드·인라인 SVG 차트 3종·읽기 전용 원칙) + 6-4 3media(매체 취합·동일 잣대 비교 구조).

## Contract
- 매체별 지표를 공통 스키마(지출·노출·클릭·전환·매출 → CTR·CPC·CPA·ROAS·전환율)로 정규화해 동일 잣대로 비교한다.
- 모든 차트의 손익분기선은 [brand/profile.md](../../../../brand/profile.md) §4 **손익분기 ROAS**다 (ROAS 1.0 하드코딩 금지, 미작성 시 1.0 폴백 + `[추정]` 태그).
- 전주 대비(WoW) 증감을 핵심 지표 전부에 병기하고, 변화마다 원인 가설 1줄을 붙인다.
- 리포트는 단일 HTML 파일 — CSS/JS 인라인, 외부 의존 없음, 모바일 360px 무스크롤 (html-report-template 표준).
- 이 스킬은 읽기 전용 — 캠페인 상태·예산을 절대 바꾸지 않는다. ⏸ 주간 자동 가동·발송 스위치는 사용자 승인 후에만 켠다.

## Phases
1. **기준 로드** — profile.md §4 손익분기 ROAS·목표 CPA·월 예산 자동 참조.
2. **모드 판정** — [data/connections.md](../../../../data/connections.md) 연결 보드 확인: meta-ads MCP·검색광고 API·Notion 연결 시 자동 수집, **미연결 매체는 CSV 업로드 폴백** (매체 혼합 허용 — 매체별 소스를 산출물에 명시). `[실데이터]`/`[샘플]` 태그 필수.
3. **취합·정규화** — 매체별 로우데이터 → 공통 스키마 변환. 기간 정합성 검증(전 매체 동일 7일).
4. **주간 분석** — ① 통합 KPI 카드 4개(광고비·매출·ROAS·전환, WoW 증감 포함) ② 매체 비교표 ③ ROAS TOP 3 / BOTTOM 3 (캠페인 레벨) ④ 변화 원인 가설.
5. **HTML 리포트 생성** — html-report-template 표준 골격 + 인라인 SVG 차트 3종:
   ① 매체/캠페인 ROAS 막대 (전주 vs 이번 주 + BEP 세로선) ② 예산 점유율 vs 매출 기여율 (효자 과소투자·부진 과대투자 검출) ③ 캠페인 ROAS 가로 랭킹 (TOP→BOTTOM + BEP 선). 저장: `outputs/{날짜}/weekly-ads-{날짜}.html`.
6. **적재·발송** — Notion 아카이브 1행 (연결 시). ⏸ 외부 채널 발송·매주 자동 가동(cron)은 사용자 승인 후에만.
7. **체인 제안** — BOTTOM 캠페인·매체 불균형 발견 시 → 046 ROAS 진단 & 예산 재배분.

### 차트 3종 (인라인 SVG): ROAS 막대+BEP선 / 예산 vs 매출 기여 / 캠페인 랭킹

## Anti-Patterns
- **매체 간 ROAS 직접 비교 과신** — 매체별 기여 모델이 다르다(뷰스루·클릭 기준). 비교표엔 주의 주석을 반드시 부착.
- **캠페인 목적 무시한 ROAS 재단** — 도달·트래픽 캠페인을 ROAS로 자르면 오판. 목적별 보조 지표(CPM·빈도) 병기.
- **숫자만 있고 해석 없는 리포트** — 증감 화살표 옆에 원인 가설이 없으면 다음 주에 아무것도 안 바뀐다.
- **외부 차트 라이브러리 의존** — CDN 로드는 오프라인·첨부 열람에서 깨진다. 차트는 인라인 SVG로.
- **승인 없는 자동 발송 개시** — mutating 스킬이다. 매주 자동 가동은 반드시 사용자 스위치 승인 후.

## 활용
- **월요일 아침 루틴**: 주말 사이 데이터로 월요일 아침 리포트 → 팀 주간 회의 첫 안건으로 (Before 40분 수작업 → After 1분).
- **다매체 통합 관리**: 메타는 MCP 자동, 네이버는 CSV 폴백 — 혼합 모드로도 한 장에 취합해 매체별 온도차를 한눈에.
- **경영 보고 파이프라인**: 주간 리포트 4장이 그대로 067 월간 리포트의 입력이 된다 — 주간→월간 이중 집계 없이 연결.
- **캠페인 목적별 보조 지표 각주**: 전환 = ROAS·CPA / 트래픽 = CPC·CTR / 도달 = CPM·빈도 — 매체 비교표에 목적 열을 넣어 ROAS 단독 오판을 구조적으로 막는다.
- **회의 문화 교정**: 모든 ROAS 차트에 BEP 세로선(profile.md §4)이 박혀 있어 "몇이면 잘한 거냐"는 질문이 회의에서 사라진다.

> 부록 — 공통 스키마 정규화 기준 (매체별 원천 필드 → 표준 지표)

| 표준 필드 | 산식 | 메타 원천 | 구글 원천 | 네이버 원천 |
|---|---|---|---|---|
| 지출 | — | spend | cost | 광고비 |
| ROAS | 매출/지출 | purchase value | conv. value | 전환매출액 |
| CPA | 지출/전환 | actions 기반 | conversions | ccnt 기반 |
