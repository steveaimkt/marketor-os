# 소싱 상품 평가

> 원가 구조 → 실마진 계산 → 마진·수요·경쟁 100점 스코어링 → GO/NO-GO + 초도 발주량까지.
> 계승: 재고 분석 에이전트의 일평균 판매량(daily_rate)·ROP(리드타임+안전재고 3일분)·ABC 파레토 사고를 소싱 사전 평가용으로 전환.

## Contract
- 마진은 명목 마진이 아닌 실마진으로 계산한다: 판매수수료·입출고/배송비·반품 손실·보관비까지 반영한 공식을 산출물에 공개한다.
- 스코어는 고정 배점(마진 40 + 수요 30 + 경쟁 30 = 100점)으로 매기고, 각 항목의 판단 근거·출처를 병기한다.
- 판정은 GO / 조건부 GO / NO-GO 3단 + 근거로 내고, 조건부 GO는 충족 조건을 수치로 명시한다.
- GO 판정에는 초도 발주량 제안이 반드시 붙는다 (ROP 공식 기반, 과잉 발주 방지).
- 수요 데이터를 조회할 수 없으면 추정 모드로 전환하고 모든 추정치에 [추정] 태그를 단다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) 자동 참조 (주력 채널·기존 카테고리·자금 여력). 브랜드 정보 재질문 금지.
2. **원가 구조 입력** — 필수: 매입원가·목표 판매가·채널 수수료율·MOQ·리드타임. 누락 항목은 진행 전에 질문 (임의 가정 금지).
3. **실마진 계산** — `실마진 = 판매가 − 매입원가 − (판매가×수수료율) − 입출고/배송비 − (판매가×예상반품률×0.7) − 월보관비/예상월판매량`. 실마진율 = 실마진/판매가.
4. **수요 스코어 (30점)** — 소스: 네이버 데이터랩·쇼핑 검색량, 상위 노출 상품 리뷰 증가 속도. 검색량 추세(상승/보합/하락) + 계절성 체크. 미연결 시 [추정] 모드.
5. **경쟁 스코어 (30점)** — 상위 노출 셀러 수, 1페이지 평균 리뷰수(진입 장벽), 최저가 대비 목표가 격차, 브랜드 지배 여부 (1개 브랜드가 상위 절반 이상이면 감점).
6. **마진 스코어 (40점)** — 실마진율 기준: 30%+ → 만점 / 20~30% → 비례 / 20% 미만 → 절반 이하 / 10% 미만 → 0점 처리.
7. **판정 + 초도 발주량** — 총점 70+ & 실마진율 20%+ → GO / 50~69 → 조건부 GO (조건 명시: 예 "판매가 10% 인상 시") / 50 미만 → NO-GO. GO 시 초도 발주 = 예상 daily_rate × (리드타임 + 30일) + 안전재고(daily_rate×3), MOQ와 비교 제시.
8. **산출 + 체인 제안** — 017 가격 전략 설계 (판매가 시나리오) · 010 진입 시장 우선순위 (후보 여러 개 비교 시).

## Anti-Patterns
- **명목 마진으로 GO** — "원가 4천, 판매가 1.3만이니 남는 장사"는 수수료·반품·보관비를 만나면 무너진다. 실마진 공식 생략 금지.
- **검색량 절대값만 보기** — 검색량이 커도 하락 추세·계절 꼭지면 함정이다. 추세·계절성을 반드시 본다.
- **리뷰 장벽 무시** — 1페이지가 리뷰 수천 개면 광고비가 마진을 삼킨다. 경쟁 스코어에 반영한다.
- **MOQ 무시한 GO** — 판정과 초도 발주량·MOQ 격차를 함께 보여주지 않으면 재고 리스크를 숨긴 것이다.
- **NO-GO 회피** — 3단 판정의 절반은 거절이다. 애매하면 조건부 GO의 조건을 수치로 못 박는다.

## 활용
- **소싱 후보 스크리닝**: 후보 5개를 같은 스코어카드로 돌려 총점 순위표를 만들고 상위 2개만 010으로 심화한다.
- **기존 상품 재평가**: 수수료·광고비 변동 후 실마진을 재계산해 단종/유지 판단에 쓴다.
- **공급처 협상**: 조건부 GO의 조건("원가 15% 인하 시 GO")을 협상 카드로 그대로 사용한다.
