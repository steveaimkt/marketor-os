# 제품 스펙시트 → 셀링포인트

> 스펙 항목 분해 → 기업 언어↔고객 언어 1:1 매핑 → 혜택 카피 변환 → 우선순위 TOP 5~7.
> 계승: `상세페이지기획` 6단계 공감 키워드 맵 — "스펙은 그대로 두고 옆에 고객 표현 병기", 불만 표현→장점 카피 변환, 인용 출처 규칙 (예: 21V 4.0Ah → "주말에 나무 10그루 잘라도 남아요").

## Contract
- 모든 셀링포인트는 '스펙(기업 언어) → 고객 언어 → 혜택 카피' 3열 매핑을 거친다 — 스펙 없는 카피, 카피 없는 스펙을 남기지 않는다.
- 스펙 표현은 삭제하지 않고 그대로 두고, 옆에 고객 표현을 병기한다 (스펙은 신뢰 장치, 고객 언어는 공감 장치).
- 리뷰·VoC 자료가 있으면 고객 언어는 실제 고객 표현에서 인용하고 출처(리뷰 번호·파일명)를 단다. 없으면 [작성] 태그로 창작임을 표시한다.
- 스펙이 뒷받침하지 않는 주장은 셀링포인트에 넣지 않고 "제외 목록"에 사유와 함께 기록한다.
- 우선순위는 고정 기준(타깃 페인포인트 관련도 × 경쟁 차별성)으로 정렬한다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) 자동 참조 (타깃·톤·금기 표현). 타깃 미지정 시 profile 기본 타깃 사용을 알린다.
2. **스펙 분해** — 스펙 문서에서 수치·소재·인증·구성품을 항목 단위로 추출. 모호한 스펙(예: "고성능")은 근거 수치를 질문.
3. **고객 언어 매핑** — 항목별 1:1: 기업 언어(스펙 원문) ↔ 고객 언어(사용 장면 표현). 리뷰 자료 있으면 실제 표현 인용 + 출처, 없으면 타깃의 사용 맥락으로 작성 + [작성] 태그.
4. **혜택 카피 변환** — '기능 → 그래서 고객에게 무엇' 구조로 변환. 고객 불만 표현(경쟁품 리뷰 등)은 자사 장점 카피로 뒤집어 별도 표시.
5. **우선순위 정렬** — 각 셀링포인트에 페인포인트 관련도(상3/중2/하1) × 경쟁 차별성(상3/중2/하1) 점수 → TOP 5~7 선정, 나머지는 보조 목록.
6. **검증 + 제외 목록** — 과장·미검증 주장 스크리닝 (수치 근거 없는 최상급 표현 등) → 제외 목록에 사유 기록.
7. **산출 + 체인 제안** — 051 상세페이지 24섹션 빌더 (TOP 셀링포인트 → 섹션 메시지) · 043 메타 광고 소재 카피 팩 (1순위 소구점 → 카피 시드).

### 변환 3열의 역할 (계승 원칙)
| 열 | 역할 | 예시 |
|---|---|---|
| 스펙 (기업 언어) | 신뢰 장치 — 그대로 유지 | 21V 4.0Ah 리튬이온 |
| 고객 언어 | 공감 장치 — 사용 장면 표현 | "주말 내내 써도 배터리가 남아요" |
| 혜택 카피 | 구매 이유 — '그래서 무엇' | 한 번 충전으로 주말 정원 손질 끝 |

### 지원 입력 소스
- 공급처 사양서·제품 스펙표 (PDF·이미지·텍스트)
- 자사/경쟁품 리뷰 덤프 (고객 언어 인용 소스 — 006 산출물 재사용 가능)
- 인증서·시험성적서 (수치 근거 확보용)

## Output Format
```markdown
# 🔧 셀링포인트 변환: {제품명} (타깃: {타깃 한 줄})

| 스펙 (기업 언어) | 고객 언어 | 혜택 카피 | 근거 | 점수 |
|---|---|---|---|---|
| 21V 4.0Ah 리튬이온 | "주말 내내 써도 배터리가 남아요" | 한 번 충전으로 주말 정원 손질 끝 | 리뷰 #15 | 3×3=9 |
| IPX7 방수 | [작성] "물에 빠뜨려도 멀쩡해요" | 설거지하다 떨어뜨려도 걱정 없음 | 스펙표 | 3×2=6 |

## 우선순위 셀링포인트 TOP 5

- "업계 최고 출력" — 비교 데이터 없음 → 수치 확보 전 사용 금지

→ 051 상세페이지 24섹션 빌더 (TOP 5 → 섹션 배치) / 043 메타 광고 소재 카피 팩 (1순위 → 카피 10종)
```

## Anti-Patterns
- **스펙 나열형 번역** — "21V 배터리 탑재"를 "강력한 21V 배터리!"로 바꾸는 건 번역이 아니다. 사용 장면까지 가야 고객 언어다.
- **스펙 삭제** — 고객 언어로 바꾸면서 스펙 원문을 지우면 신뢰 근거가 사라진다. 반드시 병기한다.
- **인용 위조** — 리뷰에 없는 문장을 인용처럼 쓰지 않는다. 창작 표현은 [작성] 태그로 구분한다.
- **전 항목 셀링포인트화** — 스펙 20개 = 소구점 20개가 아니다. TOP 5~7로 자르고 나머지는 보조로 내린다.
- **과장 통과** — "최고·유일·완벽"은 근거 수치가 없으면 제외 목록행이다 (051의 규제 게이트 전에 여기서 거른다).

## 활용
- **신제품 입고 직후**: 공급처 사양서만 있는 상태에서 상세페이지·광고 준비의 첫 단추로 실행한다.
- **전환율 부진 상품 점검**: 현재 상세페이지 문구를 매핑표와 대조해 "기업 언어만 있는 섹션"을 찾아 교체한다.
- **경쟁품 리뷰 역이용**: 경쟁품 불만 리뷰를 입력에 넣어 자사 장점 카피로 뒤집는 4단계를 집중 활용한다.
