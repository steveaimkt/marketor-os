# A/B 테스트 설계

> 가설 형식화 → 변수 1개 고정 → 표본수·기간 계산(기준 전환율·MDE) → 판정 기준 사전 고정 → 실행 준비 체크.
> 경계: **019는 설계(실행 전), 047은 판정(실행 후)** — "결과 나왔는데 어느 쪽이 이겼어?"는 047 광고 소재 A/B 판독기의 일이다. 이 스킬은 데이터가 생기기 전에 끝난다.

## Contract
- 가설은 표준 형식으로만 작성한다: "변수 X를 A→B로 바꾸면, 지표 Y가 {기준값}에서 {목표값}으로 (상대 {Z}%) 변한다" — 형식 미달 가설로 설계 진행 금지.
- 테스트당 변경 변수는 1개 — 두 요소 이상을 동시에 바꾸는 안은 분리하거나 별도 테스트로 순번을 매긴다.
- 표본수는 공식으로 계산해 대입값을 공개한다: **그룹당 n ≈ 16 × p(1−p) ÷ (p×MDE)²** (p=기준 전환율, MDE=상대 최소탐지효과, α=0.05·검정력 80% 근사). 기간 = 총 표본 ÷ 일 트래픽, 최소 7일(요일 효과).
- 판정 기준을 시작 전에 고정해 설계서에 명문화한다: 유의수준·최소 실행 기간·중간 피킹 금지·조기 중단 허용 조건(가드레일 지표 악화 시만).
- 예상 트래픽으로 MDE 달성이 비현실적(기간 8주 초과)이면 그대로 설계하지 않고 대안(MDE 상향·상위 지표 전환·순차 테스트)을 제시한다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) 자동 참조 (채널·기준 지표 이력).
2. **가설 형식화** — 입력 가설을 표준 형식으로 변환. 기준값(현재 전환율 등)이 없으면 데이터 소스를 질문, 없으면 [추정] 태그 + 보수적 가정.
3. **변수·변형안 설계** — 변경 변수 1개 확정(썸네일/헤드라인/가격 표기/CTA 등), A(현행)·B(변형) 정의. 변수 후보가 여럿이면 예상 임팩트 순으로 테스트 로드맵 번호를 매긴다.
4. **표본수·기간 계산** — 공식 대입: 예) p=2%, MDE=상대 20% → n ≈ 16×0.02×0.98÷(0.004)² ≈ 19,600/그룹, 총 39,200. 일 트래픽 2,000이면 약 20일. 8주 초과 시 대안 제시(Contract 5).
5. **판정 기준 사전 고정** — 유의수준 α=0.05 / 최소 기간 7일 & 표본 충족 / 중간 확인은 가드레일(오류·CS 급증)만 / 승리 선언 조건과 무승부 처리(현행 유지)까지 명문화.
6. **실행 준비 체크** — 트래픽 균등 분배 방법(채널 기능/수동 로테이션), 오염 방지(동일 사용자 중복 노출), 측정 이벤트 확인.
7. ⏸ **설계서 승인** — 설계서 1페이지를 보여주고 사용자가 시작일과 함께 확정한다.
8. **산출 + 체인 제안** — 테스트 종료 후 047 광고 소재 A/B 판독기 (결과 데이터 → 통계 판정). 설계서를 047 입력으로 그대로 전달한다.

## Anti-Patterns
- **다변수 동시 변경** — 썸네일과 가격을 같이 바꾸면 이겨도 왜 이겼는지 모른다. 변수 1개 원칙은 타협 불가.
- **표본수 없는 시작** — "일주일 돌려보자"는 설계가 아니다. 표본 미달 테스트는 결과가 나와도 노이즈다.
- **결과 보고 기준 바꾸기** — 돌리는 중에 "1.5%p 차이면 충분하지 않나"라고 기준을 낮추는 것. 판정 기준은 시작 전 고정이 존재 이유다.
- **중간 피킹 중단** — 3일째 B가 앞선다고 종료하면 거짓 승자를 뽑는다. 가드레일 외 조기 중단 금지.
- **설계 없이 047 직행** — 표본·기준 없이 모은 데이터를 047에 넣으면 판정기가 살릴 수 없다. 순서는 019 → 실행 → 047.

## 활용
- **컨셉 검증**: 011의 검증 가설(사전예약 전환)을 그대로 표준 형식으로 변환해 최소 비용 검증 실험을 설계한다.
- **CRO 개선 루프**: 048 랜딩 진단이 찾은 저해 요소를 변수 후보 로드맵으로 만들어 임팩트 순으로 하나씩 테스트한다.
- **트래픽 현실 점검**: 설계만 돌려 "우리 트래픽으론 MDE 20%가 6주"임을 확인하고, 테스트 대신 정성 조사(080 인터뷰)로 전환하는 판단에도 쓴다.
