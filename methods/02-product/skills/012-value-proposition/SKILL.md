# USP·가치제안 정의

> JTBD 정의 → 차별점 경쟁 대조 필터 → 6단 가치제안문(누구에게/어떤 상황에서/왜/무엇을/어떻게/증거) 조립 → 증거 매핑.
> 경계: **012는 전략 문장** — 제품 전체가 누구에게 왜 팔리는지의 한 문장을 만든다. 스펙 항목을 소구 카피로 번역하는 작업은 **015 (스펙 번역)** 의 일 — 스펙표가 입력이고 카피 목록이 출력이면 015로 핸드오프한다.

## Contract
- 가치제안문은 6단(누구에게/어떤 상황에서/왜: 문제/무엇을: 제공물/어떻게: 차별 방식/증거)을 전부 채운 완성문 1개 + 한 줄 축약(원라이너)으로 낸다 — 빈 단 방치 금지.
- 차별점은 경쟁 대조를 통과한 것만 인정한다: 경쟁사도 하는 것은 "기본기"로 분류하고 차별점 표에서 제외, 최종 차별점은 3개 이하.
- 모든 주장에 증거(수치·리뷰·인증·비교 데이터)를 매핑하고, 증거가 없으면 [증거 필요] 태그와 확보 방법을 명시한다 — 태그 없는 무증거 주장 금지.
- JTBD는 기능적 잡 외에 감정적·사회적 잡을 각 1개 이상 검토하고 채택/기각 사유를 남긴다.
- 산출물의 주 추천 체인은 013 네이밍·051 상세페이지 — 가치제안문이 확정되기 전 카피 작업(043 등)으로 건너뛰지 않도록 안내한다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) 자동 참조 (미션·톤·금기 표현). 브랜드 정보 재질문 금지.
2. **입력 정리** — 제품 정보 + 페르소나(008) + 경쟁 비교(002 또는 011의 갭 표). 페르소나 없으면 타깃 한 줄 정의를 질문, 경쟁 자료 없으면 대조를 [추정]으로 표기.
3. **JTBD 정의** — 타깃이 이 제품을 "고용"하는 상황과 해결하려는 잡: 기능적(무엇을 끝내려) / 감정적(어떤 기분이 되려) / 사회적(어떻게 보이려). 잡별 채택/기각 사유 기록.
4. **차별점 필터** — 자사 강점 후보를 나열 → 경쟁 3사 대조: 경쟁도 하면 "기본기", 우리만 하면 "차별점". 차별점 3개 이하로 압축.
5. **6단 조립** — "{누구에게} / {어떤 상황에서} / {왜: 문제} / {무엇을} / {어떻게: 차별 방식} / {증거}" 완성문 + 원라이너 + 헤드라인형 변형 1개.
6. **증거 매핑** — 문장 속 모든 주장 ↔ 증거 표. 증거 없는 주장은 [증거 필요] + 확보 방법(시험성적서·리뷰 수집·비교 실험).
7. **산출 + 체인 제안** — 013 제품명·네이밍 (가치제안문 → 네이밍 방향) · 051 상세페이지 24섹션 빌더 (가치제안문 → 헤드 섹션). 스펙 단위 카피가 필요하면 015.

## Anti-Patterns
- **형용사 가치제안** — "최고 품질의 혁신적인 제품"은 6단 어디에도 못 들어간다. 상황·문제·증거가 없는 문장은 다시 쓴다.
- **차별점 인플레이션** — 강점 7개를 전부 차별점이라 부르는 것. 경쟁 대조를 통과한 3개 이하만 남긴다.
- **015 영역 침범** — 스펙 항목별 소구 카피를 여기서 뽑기 시작하면 전략 문장이 카피 목록에 묻힌다. 스펙 번역은 015로 보낸다.
- **증거 후불제** — "증거는 나중에 채우자"며 [증거 필요] 태그 없이 발행하는 것. 051·043으로 넘어가면 무증거 주장이 카피가 된다.
- **기능적 잡만 보기** — 감정적·사회적 잡 검토 없이 스펙 우위만 쓰면 저가 경쟁 프레임에 갇힌다.

## 활용
- **신제품 런칭 킥오프**: 011 선택 컨셉 → 012 가치제안 → 013 네이밍 → 018 GTM 순서로 /신제품런칭 체인의 2번 타자로 쓴다.
- **기존 제품 리포지셔닝**: 판매 부진 제품의 현 상세페이지를 6단에 역대입해 빈 단(대개 '어떤 상황에서'와 '증거')을 찾아 메꾼다.
- **팀 정렬 문서**: 원라이너를 brand/profile.md에 반영해 043 카피·028 포스트 등 하위 산출물의 기준 문장으로 삼는다.
