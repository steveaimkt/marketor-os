# 출시 체크리스트 & GTM 플랜

> D-day 역산 타임라인(D-30~D+30) → 채널별 태스크 → 간이 RACI 배정 → 출시 체크리스트 → 출시 후 모니터링까지.
> 런칭은 "출시일에 올리는 것"이 아니라 D-30부터 D+30까지 60일의 운영이다 — 출시 전 준비와 출시 후 초기 대응을 한 장에 담는다.

## Contract
- 타임라인은 목표 출시일(D-day) 역산으로 D-30~D+30 구간을 주 단위 블록(D-30/D-21/D-14/D-7/D-3/D-day/D+7/D+14/D+30)으로 짠다 — 날짜 없는 태스크 금지.
- 모든 태스크에 담당(R)·승인(A)·기한·산출물(완료 판정 가능한 형태)을 붙인다. 1인 팀이면 RACI 대신 "본인/외주/자동화" 구분 + 주차별 우선순위로 축약한다.
- 태스크는 가용 채널 목록에 있는 채널만 배치한다 — 미보유 채널 태스크를 넣지 않는다.
- 출시 후 D+1~D+14 모니터링 지표(첫 판매·전환율·리뷰 확보·CS 인입)와 이상 시 대응 액션을 포함한다.
- 선행 의존이 있는 태스크(예: 상세페이지 → 광고 심사)는 의존 관계를 명시한다 — 의존 무시 병렬 배치 금지.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) 자동 참조 (보유 채널·팀 구성·기존 런칭 이력).
2. **입력 확정** — 출시일·채널·팀·완성 자산 현황(012 가치제안 / 013 이름 / 상세페이지 / 재고 입고일). 출시일 미정이면 재고 입고 + 준비 리드타임으로 역산 제안.
3. **태스크 도출** — 채널별 표준 태스크 풀에서 해당 채널만 선별: 스토어(상품 등록·상세페이지·가격/쿠폰 세팅), 광고(소재 제작·심사·캠페인 세팅), 콘텐츠(티저·런칭 콘텐츠·리뷰 시딩), PR(보도자료·미디어 리스트), CS(FAQ·응대 스크립트), 물류(입고·검수). 의존 관계 표기.
4. **D-day 역산 배치** — D-30 기획 확정/발주 확인 → D-21 자산 제작 착수 → D-14 상세페이지·상품 등록 → D-7 광고 심사 접수·티저 → D-3 최종 점검(가격·재고·링크) → D-day 발행·발송 → D+7 초기 지표 점검·리뷰 요청 → D+14 성과 1차 회고 → D+30 정착 판정.
5. **RACI 배정** — 태스크별 R(실행)·A(승인)·C(협의)·I(공유). A는 태스크당 1명 원칙. 1인 팀은 본인/외주/자동화 구분으로 대체.
6. **모니터링 설계** — D+1~D+14 일간 점검 지표와 임계값(예: D+7 누적 리뷰 10개 미만 → 079 리뷰 요청 가동), 이상 시 대응 액션 매핑.
7. ⏸ **플랜 확정** — 타임라인·RACI를 보여주고 사용자가 일정·담당을 조정 확정한다.
8. **산출 + 체인 제안** — 021 콘텐츠 캘린더 빌더 (D-14~D+30 콘텐츠 태스크 상세화) · 083 보도자료 작성 (PR 트랙 실행).

## Anti-Patterns
- **D-day 하루 플랜** — 출시일 태스크만 빼곡하고 D+가 빈 플랜. 런칭의 절반은 출시 후 2주의 리뷰·CS·지표 대응이다.
- **담당 없는 태스크** — "광고 준비"처럼 담당·산출물이 없는 줄은 안 한 일이 된다. R/A와 완료 판정 가능한 산출물 필수.
- **의존 무시 병렬** — 광고 심사는 상세페이지가 있어야 접수된다. 의존 체인을 무시한 일정은 D-7에 한꺼번에 무너진다.
- **전 채널 욕심** — 미운영 채널까지 타임라인에 넣으면 실행률만 떨어진다. 가용 채널만, 나머지는 D+30 이후 확장 항목으로.
- **모니터링 임계값 부재** — "지표를 지켜본다"는 계획이 아니다. 임계값과 미달 시 액션(스킬 체인)까지 박아야 플랜이다.

## 활용
- **/신제품런칭 마무리**: 011→012→017→018 체인의 종착 — 앞 스킬 산출물(가치제안·확정명·확정가)이 그대로 입력이 된다.
- **리런칭·시즌 재출시**: 기존 제품의 시즌 재출시에도 D-30 역산 구조를 재사용 — 057 행사 일정을 D-day로 잡으면 행사 대응 플랜이 된다.
- **1인 마케터 운영**: RACI 축약 모드로 주차별 "이번 주 본인이 할 3가지"만 뽑아 주간 플래닝(098)에 연결한다.
