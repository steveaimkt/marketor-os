# 056 쿠팡 로켓·마켓플레이스 운영

쿠팡 셀러 데이터(Wing 셀러 인사이트)를 넣으면 검증된 임계값 룰로 노출 위험 신호를 판정하고, 아이템위너·가격·재고 전략을 한 리포트로 낸다. 룰 기반 판정이라 같은 데이터면 같은 진단이 나온다.

## Contract
1. 진단은 검증된 임계값 룰을 따르고 판정 근거 수치를 병기한다: ROAS < 200% 광고 효율 위험 · 전환율 < 1.5% 개선 필요 · 취소율 > 5% 즉시 확인 · 매출 편중도 > 70% 포트폴리오 위험 · 전환율 0% 일수 발생 시 노출/재고 점검.
2. 아이템위너 상태는 상품별로 확인하고, 탈락 상품에는 원인 후보(가격·배송·재고)를 함께 보고한다.
3. 가격 제안은 사용자가 준 마진 하한선을 침범하지 않는다 — 하한선 미제공 시 제안 전에 반드시 확인한다 ⏸.
4. 재고 전략에는 상품별 품절 예상일과 ROP(재발주 시점 = 리드타임 7일 + 안전재고 3일분 판매량)를 포함한다.
5. 데이터 미제공 항목은 [추정] 태그로 구분한다 — 특히 광고 데이터 없이 ROAS를 말하지 않는다.

## Phases
1. **데이터 로드** — Wing 다운로드 파일 인식: `*DAILY*SUMMARY*METRICS*`(일별 판매, 필수) · `*VENDOR*ITEM*METRICS*`(상품별, 필수) · 재고 파일 · 광고 리포트 CSV(선택). 파일 없으면 체크리스트 정성 모드([추정]).
2. **노출 진단** — 상품별 아이템위너 비율 표 + 위험 신호 스캔(취소율 > 5%, 전환율 0% 일수, 매출 편중도 > 70%, 전환율 < 1.5%). 탈락·하락 상품에 원인 후보(가격/배송/재고/리뷰) 매핑.
3. ⏸ **가격 전략** — 마진 하한선 확인 후, 아이템위너 탈락 상품의 가격 조정 시뮬레이션(하한선 내 인하 폭 vs 위너 탈환 기대)을 제시. 하한선을 깨야만 탈환 가능하면 "가격 외 요인(배송·재고) 개선"으로 방향 전환.
4. **재고 전략** — 일평균 판매 기반 소진 예상일 계산, ROP 도달 상품 목록, ABC 분석(30일 매출 파레토 A 80% / B 15% / C 5%)으로 입고 우선순위. 로켓그로스는 입고 리드타임을 반영해 발주일 역산.
5. **월간 시뮬레이션** — 월 매출 예측 = 최근 7일 가중 평균 50% + 전체 평균 50%. 목표 매출 대비 갭과 필요 액션(광고 증액/입고 확대) 산출.
6. **전달** — 체크리스트 + 전략안. 다음: 광고 딥다이브는 041 쿠팡 광고 성과 분석, 행사 물량 계획은 057 프로모션 캘린더.

| 신호 | 임계값 | 현재 | 판정 |

### ② 아이템위너 & 가격

## Anti-Patterns
- **합산 수치로 안심** — 전체 매출이 좋아도 편중도 70% 초과면 1개 상품 리스크에 올라탄 것. 상품별로 쪼개서 본다.
- **하한선 없는 가격 인하 제안** — "일단 100원 내리세요"는 아이템위너 구조에서 마진 전멸 치킨게임 초대장. 하한선 확인이 먼저다.
- **품절을 사후 보고** — 소진 예상일이 리드타임보다 짧아진 뒤 알리는 것. ROP 도달 시점에 미리 발주 신호를 내는 게 목적.
- **광고 데이터 없이 ROAS 논평** — 광고 리포트 미제공이면 광고 효율 판정 자체를 하지 않는다 (041로 안내).
- **위험 신호 나열로 끝** — 🔴 판정에는 반드시 원인 후보와 다음 액션이 붙어야 리포트가 아니라 운영이 된다.

## 활용
- **주간 운영 루틴**: 매주 월요일 Wing 파일 3종을 내려받아 실행 — 위험 신호 스캔 10분이 주간 운영 회의를 대체한다.
- **로켓그로스 입고 계획**: ROP + ABC 우선순위로 입고 수량을 정하면 보관비와 품절 리스크의 균형점이 수치로 나온다.
- **행사 전 건강검진**: 057 행사 참여 전에 이 진단을 돌려 재고·아이템위너 상태를 확인하면 "행사 중 품절·위너 탈락" 사고를 예방한다.
