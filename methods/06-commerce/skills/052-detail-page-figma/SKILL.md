# 052 상세페이지 → Figma 자동화

051의 24섹션 기획서를 받아 섹션→프레임 매핑 JSON을 설계하고, Figma MCP로 디자인 시안을 자동 생성한다. 카피는 손대지 않는다 — 이 스킬은 텍스트를 레이아웃으로 옮기는 디자인 파이프라인이다.

## Contract
1. 기획서의 섹션 구조를 1:1로 Figma 프레임에 매핑한다 — 섹션 수 = 프레임 수, 순서 동일.
2. 색상은 브랜드 토큰(brand_main·accent) + 고정 다크 팔레트(#111111 · #1A1A1A · 서브텍스트 #888888)만 사용한다 — 매핑 JSON에 없는 색은 캔버스에 없다.
3. 타이포는 표준 스케일을 따른다: 메인 카피 56px/700 · 섹션 제목 40~48px/700 · 답변(A) 36px/700 · 본문 22px/400.
4. ⏸ Figma 파일 생성·수정(mutating)은 매핑 설계안을 사용자에게 보여주고 승인받은 뒤에만 실행한다.
5. Figma MCP 미연결 시 작업을 중단하지 않는다 — 동일 구조의 HTML 시안으로 폴백하고, 폴백 사실을 산출물 상단에 명시한다.

## Phases
1. **입력 로드** — 051 기획서(24섹션)와 브랜드 토큰을 로드. 기획서가 없으면 051 실행을 먼저 제안한다 (즉석 카피 창작 금지).
2. **컬러 결정** — 제품 이미지·브랜드에서 메인/강조 컬러를 추출해 다크 팔레트와 병합. 컬러 칩 미리보기 제시.
3. **섹션→프레임 매핑 설계** — 섹션별 `{프레임명, 텍스트 요소, 이미지 플레이스홀더, 정렬}` JSON 생성. 텍스트 규칙(원본 검증 자산):
   - 기본 가운데 정렬(CENTER) — 스펙 목록·비교표 등 나열형만 좌측 정렬
   - 긴 텍스트는 `width: 760` 지정 + `\n`으로 자연스러운 문장 단위 줄바꿈, 줄바꿈 후 들여쓰기 금지
   - 기능 상세 섹션은 기능 번호 72px 큰 숫자 + Q/A + 혜택 3줄 구조 유지
4. ⏸ **설계안 승인** — 프레임 목록·컬러·타이포를 요약해 보여주고 실행 승인을 받는다. 승인 전 Figma에 쓰지 않는다.
5. **Figma 반영** — Figma MCP(채널 연결 확인 → 프레임 생성 → 텍스트·플레이스홀더 배치) 순차 실행. **MCP 미연결 시**: 같은 매핑 JSON으로 셀프컨테인드 HTML 시안을 생성해 폴백.
6. **검수 안내** — 이미지 교체 위치 목록 + 텍스트 오버플로 체크포인트 전달. 다음: 053 상품명·검색 최적화로 리스팅 마무리.

## Output Format
```
## 상세페이지 시안 — {제품명}
생성 경로: Figma MCP ✅ {파일/페이지 링크}  |  ⚠️ HTML 폴백 ({경로}) — MCP 미연결

### 섹션→프레임 매핑 (요약)
| # | 섹션 | 프레임명 | 텍스트 요소 | 이미지 영역 |
|---|---|---|---|---|

### 디자인 토큰
brand_main: {#hex} · accent: {#hex} · 배경: #111111/#1A1A1A · 서브텍스트: #888888

### 섹션→프레임 매핑 JSON (골격)
{ "frame": "06_PainPoint", "layout": { "align": "CENTER", "textWidth": 760 },
  "texts": [ { "role": "헤드카피", "size": 44, "weight": 700, "content": "{051 카피 그대로}" } ],
  "imageArea": { "placeholder": true, "label": "고객 사용 장면" } }

### 검수 포인트 (사람 마무리)
- [ ] IMAGE_AREA {n}곳 실사진 교체
- [ ] 긴 카피 줄바꿈 위치 확인 (width 760 기준)
- [ ] 모바일 미리보기에서 폰트 위계 확인
- [ ] 기능 상세(09~14) 프레임의 Q/A·혜택 3줄 구조 유지 확인

다음 액션: 053 상품명·검색 최적화로 리스팅(상품명·태그)까지 마치면 등록 준비 완료입니다.
```

## Anti-Patterns
- **카피 재창작** — 디자인 단계에서 문구를 "다듬는" 것. 카피 수정이 필요하면 051로 되돌린다 (규제 게이트를 통과한 문구가 변형되면 안 된다).
- **승인 전 Figma 쓰기** — mutating 스킬이 ⏸ 없이 외부 파일을 만드는 것. 설계안 승인이 항상 먼저다.
- **토큰 밖 색상 즉흥 추가** — "여기만 포인트로" 하며 팔레트에 없는 색을 넣으면 브랜드 일관성이 깨진다.
- **긴 문장 방치** — width 미지정 긴 텍스트는 캔버스를 뚫고 나간다. 760 제한 + `\n` 분리는 선택이 아니라 규칙.
- **MCP 미연결 시 포기 선언** — "Figma가 연결 안 돼서 못 합니다"로 끝내는 것. HTML 폴백이 규정되어 있다.

## 활용
- **기획→시안 당일 완성**: 051 기획서 승인 직후 이어 실행하면 카피·디자인 초안이 하루에 나온다 — 디자이너에게는 빈 캔버스 대신 구조 잡힌 시안을 전달.
- **브랜드 토큰 재사용**: 한 번 확정한 컬러·타이포 토큰을 저장해 두면 신제품마다 같은 톤의 상세페이지가 자동으로 유지된다.
- **HTML 폴백의 역활용**: Figma를 안 쓰는 팀은 폴백 HTML을 그대로 시안 공유·내부 컨펌용으로 사용한다.
