# 059 크라우드펀딩 페이지 빌더

> 와디즈형 5막 서사(문제→여정→제품→증거→리워드) 스토리 페이지 + 가격 사다리·한정 구조 리워드 설계 + 심사 대비 증빙 체크.
> **경계**: 051 상세페이지 24섹션은 정보·설득 중심의 **판매** 문법, 059는 메이커의 여정을 파는 **서사** 문법이다 — 051에서 이식하는 것은 증거·FAQ·스펙 파트뿐, 구조는 이식하지 않는다.

## Contract
- 스토리는 **문제→여정→제품→증거→리워드 5막 서사**를 따른다 — 막을 뺄 때는 사유를 명시하며, 여정 막(메이커 서사) 생략은 불가.
- 리워드는 가격 사다리(슈퍼얼리버드→얼리버드→기본→세트) + 한정 수량 구조로 설계하고, 단계마다 할인율과 마진을 병기한다 — 최저가 리워드도 [brand/profile.md](../../../../brand/profile.md) §4 **마진율 기준 손익분기**를 사수한다 (미작성 시 `[추정]`).
- 목표 금액은 감이 아니라 역산한다: 목표 = Σ(리워드 단계별 단가 × 예상 수량), 산식을 산출물에 명시.
- 증거 막의 수치·인증·비교 실험에는 출처와 증빙 문서명을 병기한다 — 증빙 불가 수치는 카피에서 제외 (와디즈 심사 기준).
- 전 카피는 compliance-gate 3 STEP 통과 후에만 전달되며 치환 로그가 부착된다.

## Phases
1. **입력 정리** — 제품 스펙·메이커 스토리 소재(개발 계기·실패담·제작 과정)·목표 금액·원가·리워드 구성 요소를 표로 정규화. profile.md 업종으로 규제 사전 결정.
2. **5막 서사 설계** — 표준 구조 (순서 고정):

| 막 | 목적 | 핵심 질문 | 분량 가이드 |
|---|---|---|---|
| 1 문제 | 공감 후킹 | "이런 불편, 있으시죠?" | 전체의 ~15% |
| 2 여정 | 메이커 신뢰 | "왜 우리가 이걸 만들었나" | ~20% |
| 3 제품 | 해법 시연 | "그래서 무엇이 어떻게 다른가" | ~30% |
| 4 증거 | 의심 해소 | "믿을 근거는 무엇인가" | ~20% |
| 5 리워드 | 참여 전환 | "지금 참여하면 무엇을 얼마에" | ~15% |

3. **리워드 사다리 설계** — 단계별 역할·할인율·수량·마진 표:

| 단계 | 할인율 | 수량 | 역할 |
|---|---|---|---|
| 슈퍼얼리버드 | 30~40% | 극소량 한정 | 오픈 직후 달성률 부스터 (오픈 알림 전환용) |
| 얼리버드 | 20~30% | 한정 | 메인 볼륨 |
| 기본 | 10~15% | 무제한 | 기준가 앵커 (정가 대비 명분) |
| 세트/번들 | 개당가 절감 | 선택 | 객단가 상승 |

   목표 금액 역산 + 슈퍼얼리버드 마진 검증(BEP 미달 단계 검출 시 경고).
4. **막별 카피 작성** — 여정 막은 메이커 1인칭 서사(실패·시행착오 포함), 증거 막은 시험성적서·인증·비교 실험을 증빙 문서명과 함께, 리워드 막은 사다리 표 + 마감 구조(수량·기간) 명시.
5. **심사 대비 증빙 체크** — "최초·최고·유일" 표현별 증빙 유무, 기능성 표현의 시험 근거, 배송 일정의 실현 가능성 → 체크리스트로 검출.
6. ⏸ **규제 게이트** — gates/compliance-gate.md 전수 스캔 → 금기어 치환 + 치환 로그. ⛔ 판정 시 강행 불가. 통과 후 전달 → 체인: 펀딩 종료 후 자사몰·스토어 전환 시 051 상세페이지 24섹션 빌더로 재구성.

## Output Format
```
## 크라우드펀딩 페이지 기획서 — {제품명} · 목표 {n}원
타깃: {페르소나 1줄} · 플랫폼: {와디즈|텀블벅} · 목표 역산: Σ(단계 단가×예상 수량) = {n}원

### ① 5막 스토리 페이지 (막·헤드카피·본문 요지·소재 근거 — 5막 전체)
### ② 리워드 사다리 설계표 (단계·구성·정가 대비 할인·수량·마진율·BEP 판정)
### ③ 심사 대비 증빙 체크리스트 (표현 → 증빙 문서명 → 유무 ☐)
### 규제 치환 로그 (검출어·위치·대체어·근거 조항)

─── 🛡 컴플라이언스 게이트 ─── 판정: ✅ / ⚠️ ({n}건 치환) / ⛔ ───

다음 액션: 펀딩 종료 후 → 051 상세페이지 24섹션 빌더 (스토어 전환용 판매 문법 재구성)
```

## Anti-Patterns
- **판매 상세페이지 복붙** — 스펙 나열은 펀딩 문법 위반이다. 펀딩 참여의 구매 이유는 제품보다 메이커의 여정이다.
- **슈퍼얼리버드 물량 과다** — 최저가에 물량을 풀면 이후 단계 전환이 실종되고 마진이 무너진다. 극소량 한정이 원칙.
- **증빙 없는 "세계 최초·1위"** — 심사 반려 1순위. 증빙 문서명을 못 대는 표현은 쓰지 않는다.
- **목표 금액 감으로 설정** — 리워드×수량 역산 없는 목표는 달성률 연출도 물류 계획도 불가능하게 만든다.
- **여정 막 생략** — 제품 소개만 남기면 브랜드 없는 신생 메이커가 신뢰를 얻을 유일한 자산(서사)을 버리는 것.

## 활용
- **신제품 시장 검증**: 양산 전 5막 페이지로 펀딩 오픈 — 달성률·리워드별 선택 비율이 곧 수요·가격 수용도 데이터.
- **펀딩→스토어 전환 루프**: 펀딩 종료 후 증거 막(달성률·서포터 후기)을 051의 후기·신뢰 섹션 소재로 이식 — 서사 자산을 판매 자산으로 환류.
- **리워드 사다리 재활용**: 사다리 구조를 016 번들·프로모션 설계에 넘겨 정식 판매의 런칭 프로모션 가격 구조로 전용.
