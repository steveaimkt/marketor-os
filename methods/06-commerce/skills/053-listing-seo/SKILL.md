# 053 상품명·검색 최적화

검색에 걸리는 상품명은 감각이 아니라 조합 공식이다. 플랫폼별 규칙에 맞춰 브랜드→모델→상품유형→속성 순서로 상품명을 조합하고, 태그·카테고리를 상품명과 중복 없이 세팅한다. 상세페이지 본문은 051의 몫 — 이 스킬은 리스팅(제목·태그·카테고리)만 다룬다.

## Contract
1. 상품명은 플랫폼별 글자수·금지 규칙을 준수한다 — 초과 시 자동 축약안을 함께 제시한다.
2. 조합은 **브랜드 → 시리즈/모델 → 상품유형 → 핵심 속성(용량·수량·색상)** 순서 공식을 따르고, 동일 키워드는 상품명 안에서 1회만 쓴다.
3. 키워드 우선순위는 검색량 × 적합도 × 경쟁강도 점수로 정렬한다 — 실데이터(유입 검색어 리포트·데이터랩) 미제공 시 [추정] 태그를 붙인다.
4. 태그는 상품명에 이미 들어간 키워드와 중복시키지 않는다 (중복 태그 = 노출 기회 낭비).
5. "최저가"·"1위"·"정품보장" 등 최상급·보장성 표현은 compliance-gate에서 차단·치환 후 전달한다.

## Phases
1. **입력 확인** — 상품 정보(브랜드·모델·유형·속성)와 판매 플랫폼을 확정. brand/profile.md의 업종·카테고리 참조.
2. **키워드 후보 수집** — 우선순위 소스: ① 실제 유입 검색어 데이터(쿠팡 키워드 리포트 `pa_daily_keyword`, 스토어 유입 검색어) ② 네이버 데이터랩·검색광고 키워드 도구 ③ 미제공 시 카테고리 연관어 [추정] 모드.
3. **우선순위 점수화** — 키워드별 검색량(상/중/하) × 적합도(내 상품과의 일치) × 경쟁강도(역가중) 표 작성. 대표 키워드 1개 + 보조 키워드 4~8개 선정.
4. **상품명 조합 생성** — 플랫폼별 규칙 적용, 각 3안(검색형/속성형/절충형):
   | 플랫폼 | 조합 공식 | 규칙 |
   |---|---|---|
   | 쿠팡 | 브랜드+제품명(모델)+상품유형+속성(수량·색상) | 수식어·중복 키워드 금지, 간결 우선 |
   | 스마트스토어 | 브랜드+대표 키워드+속성+수량 | 50자 이내 권장, 동일 단어 반복·특수문자 금지 |
   | 자사몰·기타 | 브랜드+베네핏 소구+상품유형 | 검색보다 클릭 소구 허용 |
5. **태그·카테고리 세팅** — 태그는 보조 키워드에서 상품명 미사용분으로 채우고, 카테고리는 대표 키워드의 상위 노출 상품들이 속한 카테고리와 일치시킨다.
6. ⏸ **게이트 검사 → 전달** — 금지·과장 표현 스캔 후 전달. 다음: 051 상세페이지 24섹션 빌더로 본문 제작.

## Anti-Patterns
- **키워드 욕심 채우기** — 걸리는 대로 키워드를 이어 붙인 60자 상품명은 가독성과 클릭률을 함께 죽인다. 공식 순서 + 간결이 원칙.
- **상품명·태그 중복** — 상품명에 쓴 키워드를 태그에 또 쓰는 것. 태그는 추가 노출면이지 복사면이 아니다.
- **검색량만 보고 선택** — 검색량 큰 키워드가 내 상품과 적합하지 않으면 유입은 있어도 전환이 없다. 3요소 곱으로 판단.
- **카테고리 감으로 지정** — 상위 노출 상품들의 실제 카테고리를 확인하지 않고 지정하면 검색 매칭에서 밀린다.
- **데이터 없이 확정 선언** — 유입 검색어 데이터 없이 만든 우선순위를 실측처럼 말하는 것. [추정]은 [추정]으로.

## 활용
- **신규 등록 세트**: 053(상품명·태그) → 051(본문) → 052(시안) 순서로 돌리면 상품 등록 풀세트가 나온다 (/스토어셋업 커맨드의 첫 단추).
- **기존 상품 리네이밍**: 유입 검색어 리포트를 넣으면 "실제로 들어오는 검색어"와 현재 상품명의 갭이 보인다 — 갭이 큰 상품부터 교체.
- **시즌 키워드 스왑**: 시즌마다 보조 키워드·태그만 교체해 같은 상품의 검색 접점을 갱신한다 (057 행사 캘린더와 연동).
