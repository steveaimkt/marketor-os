# 051 상세페이지 24섹션 빌더

제품 정보 하나로 검증된 24섹션 표준 구조의 기획서 + 섹션별 카피를 생성한다. 페인 섹션은 고객의 언어(VoC 원문)를 인용하고, 규제 금기어는 게이트에서 자동 치환된다. "잘 쓰는 것보다 먼저, 법에 걸리지 않게. 그 다음 고객의 언어로."

## Contract
1. 산출물은 항상 아래 24섹션 표준 구조를 따른다 — 섹션을 뺄 때는 사유를 해당 행에 명시한다.
2. 페인포인트 섹션(06)은 VoC·리뷰의 고객 문장을 1개 이상 그대로 인용한다. VoC 미제공 시 [추정] 태그로 구분한다.
3. 임상·실험·수치 인용에는 출처와 표본수(n)를 반드시 병기한다 — 병기 불가한 수치는 카피에서 제외한다.
4. 경쟁사 직접 비교("타사 대비 30%")는 일반 비교 표현("비교 시 30%")으로 우회한다.
5. 전 카피는 compliance-gate 3 STEP을 통과한 뒤에만 전달되며, 치환 로그가 항상 부착된다.

## Phases
1. **입력 정리** — 제품 스펙(성분·기능·가격)·페르소나·VoC를 표로 정규화. brand/profile.md 업종 값으로 규제 사전(화장품법·건기식 등)을 결정한다.
2. **핵심 기능 6개 선정** — 셀링포인트 중 구매 결정력 순으로 6개를 골라 09~14 상세 섹션에 배정 (6개 미만이면 있는 만큼 배정하고 축소 사실 명시).
3. **24섹션 배치** — 표준 구조 (검증 자산 — 순서 고정):

| # | 섹션 | 목적 | # | 섹션 | 목적 |
|---|---|---|---|---|---|
| 01 | Hook 후킹 | 강렬한 메인 카피 | 13 | 기능 5 상세 | Q&A 상세 |
| 02 | 이게 뭔가요? | 제품 한마디 정의 | 14 | 기능 6 상세 | Q&A 상세 |
| 03 | 브랜드 의미 | 브랜드명·철학 | 15 | 사용 꿀팁 | 활용 노하우 |
| 04 | 세트 구성품 | 구매 시 받는 것 | 16 | 핵심 차별화 | 우리만의 강점 |
| 05 | 핵심 기능 중요성 | 차별점이 왜 중요한가 | 17 | 경쟁사 비교 | 비교표 (일반 표현) |
| 06 | 페인포인트 공감 | VoC 고객 문장 인용 | 18 | 안전/신뢰 | 인증·품질 보증 |
| 07 | 해결책 제시 | 우리 제품의 해법 | 19 | 추천 대상 | 적합 고객 |
| 08 | 핵심 기능 개요 | 6가지 한눈에 | 20 | 고객 후기 | 실사용 후기 |
| 09 | 기능 1 상세 | Q&A 상세 | 21 | 제품 스펙 | 상세 사양 |
| 10 | 기능 2 상세 | Q&A 상세 | 22 | FAQ | 구매 전 궁금증 5+ |
| 11 | 기능 3 상세 | Q&A 상세 | 23 | 보증/정책 | A/S·환불 정책 |
| 12 | 기능 4 상세 | Q&A 상세 | 24 | CTA | 최종 구매 전환 |

4. **섹션별 카피 작성** — 기능 상세(09~14)는 Q&A 형식 고정: "Q. 왜 [기능명]인가요?" → "A. 핵심 혜택" + 구체 혜택 3가지 + 스펙 수치 강조(선택). 페인(06)은 VoC 원문 인용, 후기(20)는 실자료만 사용.
5. ⏸ **규제 게이트** — gates/compliance-gate.md 전수 스캔 → 금기어 자동 치환 + 치환 로그(검출어·위치·대체어·근거 조항). ⛔ 판정 시 생성 강행 불가 — 위반 목록을 보고하고 수정 방향을 제시한다.
6. **전달** — 기획서 + 치환 로그. 다음: 052(Figma 시안) 또는 049(광고 정책·심의 체커).

## Output Format
```
## 상세페이지 기획서 — {제품명} · 24섹션
타깃: {페르소나 1줄} · 톤: {brand/tone 요약} · 규제 사전: {업종}

| # | 섹션 | 헤드카피 | 본문 요지 | 소재 근거 |
|---|---|---|---|---|
| 01 | Hook | {카피} | {요지} | {스펙/VoC/임상 출처} |
| … | (24행 전체) | | | |

### 규제 치환 로그
| 검출어 | 위치(섹션) | 대체어 | 근거 조항 |
|---|---|---|---|

─── 🛡 컴플라이언스 게이트 ───

──────────────────────────

```

## Anti-Patterns
- **섹션 임의 통폐합** — "비슷하니까" 06·07을 합치면 공감→해결의 설득 리듬이 무너진다. 뺄 때는 사유 명기.
- **추측 페인** — VoC가 있는데 마케터의 상상으로 페인 섹션을 쓰는 것. 고객 문장 인용이 원칙이다.
- **근거 없는 수치** — 출처·n수 없는 "만족도 98%"는 게이트 차단 사유이자 심의 리스크.
- **게이트 후치** — 카피를 먼저 전달하고 나중에 검사하는 것. 게이트는 전달 전 필수 관문이다.
- **CTA 1종** — 24번 CTA는 A/B 테스트용으로 최소 2안 이상 제공한다.

## 활용
- **외주 전 1차 초안 내재화**: 제품 스펙 1장만 정리하면 24섹션 초안이 나온다 — 외주 3~7일 카피 작업을 사내 초안 검토 10분으로 전환.
- **규제 사전 교체**: 치환 사전을 식품·건기식·의료기기 등 본인 업종으로 바꾸면 같은 게이트가 그대로 작동한다.
- **리뉴얼 진단 짝**: 기존 페이지를 24섹션 기준과 대조하면 빠진 섹션이 곧 개선 백로그가 된다 (048 CRO 진단과 세트).
