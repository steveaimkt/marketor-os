# 074 이메일 시퀀스 빌더

> 웰컴·장바구니·재구매 3시나리오를 표준 시점표로 설계하고, 카피 작성 → 게이트 → ⏸승인 → 발송 → 아카이브까지 한 파이프라인으로 처리한다.
> 계승: 패스트캠퍼스 3-1 email-newsletter의 HITL 발송 파이프라인(작성→⏸승인→발송→아카이브) + "초안=기본, 실발송=검증된 정기 발송만" 분기 원칙.

## Contract
1. 모든 시퀀스는 트리거·발송 시점·목표 지표·중단 조건 4가지가 정의된 설계도를 가진다 — 중단 조건 없는 시퀀스는 만들지 않는다 (목표 행동 달성 시 잔여 발송 즉시 중단).
2. 메일 1통은 CTA 1개만 요구한다 — 제목·preheader·본문·CTA가 같은 행동 하나를 가리킨다.
3. 모든 발송 카피는 compliance-gate 전수 스캔을 통과한 뒤에만 승인 단계로 넘어간다.
4. ⏸ 발송은 사용자 승인(y/n) 후에만 실행된다 — 승인 없이 발송된 메일은 0통이어야 한다.
5. 발송 결과는 아카이브(Notion 또는 로컬 로그)에 1행씩 기록된다 — 무엇이 언제 누구에게 나갔는지 추적 불가한 시퀀스 금지.

## Phases
1. **브랜드·세그먼트 로드** — brand/profile.md(톤·구매주기 P·AOV) 자동 참조. 065 세그먼트 리포트가 있으면 대상 세그먼트 확정, 없으면 전체 리스트 기준 + [추정] 명시.
2. **시나리오 선택·표준 로드** — 3시나리오 표준 시점표에서 시작해 브랜드에 맞게 조정:
   | 시나리오 | 트리거 | 발송 시점 | 목적 | 카피 방향 |
   |---|---|---|---|---|
   | 웰컴 | 가입·첫 구독 | 즉시 → D+2 → D+7 | 관계 형성 → 첫 구매 | 환영·브랜드 스토리 → 베스트·후기 → 첫 구매 오퍼 |
   | 장바구니 | 담기 후 미결제 | +1h → +24h → +72h | 결제 완료 | 상품 리마인드 → 사회적 증거·혜택 → 마감·재고 알림 |
   | 재구매 | 구매 후 주기 도래 | P×0.8 → P → P×1.2 | 재구매 | 사용 팁 → 재구매 제안 → 오퍼 (P = 구매주기) |
3. **시퀀스 설계도 작성** — 통별 {발송 시점, 목적, 목표 지표(오픈/클릭/전환), 분기 조건, 중단 조건}. 시나리오당 3통 기본, 5통 상한.
4. **단계별 카피 작성** — 통당 제목 2안 + preheader + 본문 300~500자 + CTA 1개. 브랜드 톤 적용. (단건 콘텐츠형 뉴스레터는 023 뉴스레터 제작 자동화 담당 — 이 스킬은 트리거 기반 자동화 시퀀스 전용.)
5. **compliance-gate 검사** — gates/compliance-gate.md 3 STEP 실행, 게이트 결과 블록 부착. ⛔ 판정 카피는 승인 단계로 못 간다.
6. ⏸ **발송 승인** — 설계도+카피 전문을 보여주고 y/n. n이면 발송 없이 설계도만 인도 (안전 원칙).
7. **발송·아카이브** — 이메일 도구 연결 시 초안 생성(기본: 사람이 최종 클릭), 미연결 시 ESP 복붙 패키지로 인도. 완전 자동 실발송은 검증된 정기 시퀀스에만 (신규·일회성은 초안+사람 확인). 발송/인도 내역을 아카이브에 1행 기록.
8. **체인 제안** — 077 이탈 방지 윈백 캠페인 (재구매 시퀀스에도 반응 없는 고객을 윈백으로 이관).

## Output Format
```

| # | 발송 시점 | 목적 | 목표 지표 | 중단 조건 |
|---|---|---|---|---|
| 1 | 가입 즉시 | 환영·기대 설정 | 오픈 40% | 첫 구매 시 전체 중단 |

─── 🛡 컴플라이언스 게이트 ───

다음 액션: 재구매 시퀀스 무반응 고객은 077 이탈 방지 윈백 캠페인으로 이관하세요.
```

## Anti-Patterns
- **승인 전 발송** — 게이트 통과 ≠ 발송 허가. ⏸ y를 받기 전 초안 발송 버튼도 누르지 않는다.
- **뉴스레터와 혼동** — "이번 주 소식 메일 1통"은 023 담당. 시퀀스는 트리거로 도는 자동화 흐름이다.
- **CTA 뷔페** — 구매+후기+앱설치를 한 통에 넣으면 전환이 죽는다. 1통 1행동.
- **중단 조건 누락** — 이미 결제한 고객에게 "장바구니가 기다려요"가 나가는 순간 브랜드 신뢰가 깎인다.
- **전 세그먼트 동일 오퍼** — 챔피언에게 첫구매 쿠폰, 신규에게 VIP 오퍼를 보내는 낭비 — 065 세그먼트 기준으로 오퍼를 분리한다.

## 활용
- **D2C 첫 자동화 3종 세트**: 웰컴→장바구니→재구매 순으로 1개씩 가동 — 보통 장바구니 시퀀스가 가장 빠른 매출 회수를 보여준다.
- **/리텐션캠페인 실행 계층**: 065(누구)→073(어디서) 뒤 "무엇을 보낼지"를 이 스킬이 실행 — CRM 체인의 발송 엔진.
- **주기 상품 재구매 루프**: 소모품(P=30~60일)은 재구매 시퀀스만으로 리텐션 KPI가 움직인다 — P×0.8 시점 사용 팁 메일이 핵심.
