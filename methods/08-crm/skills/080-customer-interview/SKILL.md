# 080 고객 인터뷰 스크립트 & 요약

> JTBD 질문 금형(상황·동기·대안)으로 유도 없는 인터뷰 스크립트를 설계하고, 수행 후 녹취를 인용 보존 요약 → 패턴 인사이트로 추출해 008 페르소나·012 가치제안의 입력으로 넘긴다.
> 계승: /시장조사 스킬의 "근거 있는 페르소나" 원칙(리뷰·발언 근거 필수, 교과서 페르소나 금지)을 인터뷰 설계·요약에 적용.

## Contract
1. 모든 질문은 과거의 실제 행동을 묻는다 — 미래 의향 질문("~라면 사시겠어요?")은 스크립트에 넣지 않는다.
2. 유도 질문은 자체 검수 단계에서 전량 개방형으로 재작성된다 ("저희 제품이 편하지 않았나요?" → "그때 어떻게 하셨나요?").
3. 스크립트는 5블록(오프닝/상황 재구성/동기·계기/대안 비교/미충족 니즈) 구조와 상황·동기·대안 질문 금형을 지킨다.
4. 요약에서 인용과 해석은 분리된다 — 발언 원문 인용에는 " " 표기, 요약자 해석에는 [해석] 태그.
5. 인사이트는 2인 이상 반복된 발언만 패턴으로 인정하고, 각 패턴에 인용 근거가 붙는다. 1인 발언은 [단독 신호]로 구분 보관.

## Phases
1. **브랜드·목적 로드** — brand/profile.md(제품·타깃·"사는/떠나는 이유" 가설) 자동 참조. 인터뷰 목적을 검증 가설 3개 이내로 압축 (가설 없는 "이것저것 물어보기" 금지).
2. **대상 선정 기준 정의** — 목적별 표본: 신규 구매 동기 = 최근 구매주기 내 첫 구매 고객 / 이탈 사유 = 휴면 전환 고객 / 개선 = 헤비 유저. 권장 5~8명, 리크루팅 안내 문구 포함.
3. **JTBD 스크립트 작성** — 5블록 × 40~60분, 질문 금형 적용:
   - **상황**: "마지막으로 {제품군}을 구매했던 날을 떠올려 주세요. 그날 무슨 일이 있었나요?"
   - **동기**: "무엇이 해결되길 바랐나요? 사기 직전, 결심의 계기가 된 사건은요?"
   - **대안**: "그 전에는 어떻게 해결하셨어요? 비교했던 다른 방법·브랜드는요? 왜 탈락했나요?"
   각 블록에 후속 심화 질문("구체적으로요?" "예를 들면요?") 2개씩 배치.
4. **유도 질문 자체 검수** — 전 질문을 3종 위반 검사: ① 답 정해주기(긍/부정 프레이밍) ② 미래 의향 ③ 복수 질문 한 문장. 위반은 개방형 재작성본과 함께 정오표로 표시.
5. ⏸ **스크립트 확정** — 사용자 승인 후 최종본 인도 (인터뷰 수행은 사람이 — 진행 팁 5줄 동봉: 침묵 견디기·순서 유연·녹음 동의).
6. **녹취 요약** — (수행 후 녹취 입력) 인터뷰이별 5블록 구조로 요약, 핵심 발언은 원문 인용 보존, 해석은 [해석] 분리. 개인 식별 정보 마스킹.
7. **인사이트 추출·체인 인계** — 반복 패턴 3개 이상(각 인용 근거 병기) + [단독 신호] 목록 → 008 타깃 페르소나 빌더 입력 형식({상황, JTBD, 대안, 불만, 인용})으로 정리해 핸드오프 (보조: 012 USP·가치제안 정의).

```

### ② JTBD 스크립트 (5블록 · 40~60분)
| 블록 | 핵심 질문 | 심화 질문 |
|---|---|---|
| 상황 재구성 | 마지막으로 …했던 날, 무슨 일이 있었나요? | 구체적으로요? |
### ③ 유도 질문 검수 정오표: 위반 {n}건 재작성 ✅
### ⏸ 스크립트 확정 → 인터뷰 수행 (사람)

- 상황: "…" (원문 인용) / [해석] …
### ⑤ 인사이트 패턴 (2인 이상 반복만)
| # | 패턴 | 근거 인용 | 시사점 |
|---|---|---|---|

```

## Anti-Patterns
- **미래 의향 질문** — "이런 기능 나오면 쓰시겠어요?"의 예스는 구매가 아니다. 과거 행동만이 증거다.
- **유도 질문 방치** — "많이 편해지셨죠?"는 데이터가 아니라 맞장구를 수집한다. 검수 정오표 없는 스크립트 금지.
- **1인 발언의 패턴화** — 목소리 큰 한 명의 의견이 "고객의 니즈"로 둔갑하는 것. 패턴은 2인 이상, 단독은 단독으로.
- **인용 없는 요약** — 해석만 남은 요약은 요약자의 편견 문서다. 원문 인용을 반드시 보존.
- **가설 없는 인터뷰** — 검증할 것이 없으면 40분 잡담이 된다. 가설 3개 이내로 시작.

## 활용
- **페르소나 검증 루프**: 008 페르소나 초안의 가설을 이 스킬로 검증 인터뷰 → 인사이트로 페르소나를 갱신하는 왕복 루프 — 리뷰 데이터(006)가 부족한 신생 브랜드일수록 유효.
- **이탈 사유 정밀 진단**: 077 윈백의 이탈 가설이 [추정]일 때, 휴면 고객 5명 인터뷰로 진짜 사유를 확보 — 오퍼 방향이 달라진다.
- **상세페이지 카피 소재**: ⑤ 패턴의 원문 인용을 "고객의 언어" 그대로 012 가치제안·상세페이지 페인 섹션에 공급한다.
