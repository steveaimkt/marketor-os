# CS 응대 매뉴얼 빌더

> 문의·리뷰 3분류(악성🔴/문의🟡/칭찬🟢) → 유형별 글자수 제한 스크립트 → 수치화된 에스컬레이션 기준표.
> 계승: 디스코드 CS 응대 에이전트의 3분류 체계·감정점수(1~5)·복합 판정 규칙·유형별 답변 구조(200/300/150자)·금지사항.

## Contract
- 모든 문의·리뷰 유형은 3분류(악성/문의/칭찬) 중 하나에 배정되고, 복합 판정 규칙(악성+문의→악성 우선, 문의+칭찬→문의 우선, 불명확→문의)이 명시된다.
- 유형별 스크립트는 글자수 상한을 지킨다: 악성 200자 / 문의 300자 / 칭찬 150자 이내.
- 악성 스크립트는 사실 기반 응대만 담는다: 보상 약속·고객 주장 부정·변명·감정 대응 문구 금지.
- 에스컬레이션 기준은 수치로 정의된다 (감정점수·금액·반복 횟수) — "심하면 상담원 연결" 같은 모호한 기준 금지.
- 스크립트는 brand/profile.md의 톤(기본 해요체)을 따르고, 경쟁사 언급·내부 프로세스 노출 문구가 없다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) 자동 참조 (톤·금기어·고객센터 정보). 연락처·영업시간 없으면 질문 (플레이스홀더 방치 금지).
2. **문의 유형 인벤토리** — 실제 문의·리뷰 로그에서 유형·빈도 집계. 로그가 없으면 업종 표준 유형으로 구성하고 [추정] 태그 (예: 커머스 = 배송/반품·교환/환불/제품/기타).
3. **3분류 체계 정의** — 유형별 판별 기준·키워드·감정점수(1: 분노 ~ 5: 열렬 추천)·긴급도 표 작성: 악성🔴(욕설·법적 언급·근거 없는 비방, 긴급도 높음) / 문의🟡(질문형·절차 확인, 보통) / 칭찬🟢(만족·재구매 의사, 낮음) + 복합 판정 규칙.
4. **유형별 스크립트 작성** — 구조 고정: 악성 = 공감·사과→해결 방안→고객센터 안내 (200자) / 문의 = 감사→직접 답변→번호 목록 절차→추가 문의 환영 (300자) / 칭찬 = 감사→고객이 칭찬한 포인트 인용→지속 이용 기대 (150자). 상황별 변형(배송·반품·제품·환불) 각 1개 이상.
5. **FAQ 작성** — 빈도 상위 문의 10개를 Q&A로 정리, 각 답변에 근거 정책 문서 참조 표기.
6. **에스컬레이션 기준표 작성** — 즉시 연결(감정점수 1 + 법적 언급·금전 보상 요구·개인정보·동일 고객 3회 이상 반복) / 부분 응대(기준 금액 초과 환불·복합 문의·정책 확인 필요) / 자동 처리(표준 절차) — 금액 기준은 사용자에게 확정받는다 ⏸.
7. **산출 + 체인 제안** — 071 CS 챗봇 구축 (이 매뉴얼을 KB 소스로 투입해 자동응대로 확장).

### 운영 규칙 (매뉴얼에 함께 수록)
- 모든 응대는 로그로 남긴다: 일시 · 유형 · 감정점수 · 사용 스크립트 · 에스컬레이션 여부 (월 1회 빈도 재집계 → 인벤토리 갱신).
- 스크립트 수정은 매뉴얼 버전을 올리고 변경 이력 1줄을 남긴다 (구버전 응대 혼용 방지).
- 답글에는 출처·내부 문서명을 노출하지 않는다 (고객에게 보이는 채널이므로).

## Output Format
```markdown
# 📞 CS 응대 매뉴얼: {브랜드} (v1.0, {작성일})

## 1. 분류 기준
| 유형 | 판별 기준 | 키워드 | 감정점수 | 긴급도 |
|---|---|---|---|---|
| 악성 🔴 | 욕설·법적 언급·비방 | 사기, 최악, 신고 | 1~2 | 높음 |
복합 판정: 악성+문의→악성 / 문의+칭찬→문의 / 불명확→문의

## 2. 유형별 스크립트 (상황별)
### 악성 — 제품 불만 (200자 이내)
"안녕하세요, 고객님. 제품이 기대에 미치지 못해 진심으로 죄송합니다. …"
## 3. FAQ TOP 10
| Q | A (근거) |
## 4. 에스컬레이션 기준표
| 레벨 | 조건 (수치) | 처리 |
|---|---|---|
| 즉시 연결 | 감정점수 1 / 보상 요구 / 반복 3회+ | 스크립트 중단, 상담원 |
| 부분 응대 | 환불 {금액}원 초과 / 복합 문의 | 답변 + 고객센터 권유 |
## 다음 액션
→ 071 CS 챗봇 구축 (이 매뉴얼을 KB로 투입해 자동응대 확장)
```

## Anti-Patterns
- **만능 스크립트 1개** — "죄송합니다. 고객센터로 연락주세요"를 전 유형에 돌려쓰면 매뉴얼이 아니라 회피다. 유형×상황별로 쪼갠다.
- **악성 리뷰에 감정 대응 문구** — 방어·반박 문장을 템플릿에 넣는 순간 사고가 복제된다. 사실+연락처만.
- **보상 약속 내장** — "환불해드리겠습니다"를 스크립트에 박으면 상담원 재량이 사라진다. 고객센터 안내로 대체.
- **글자수 무제한** — 긴 사과문은 플랫폼 답글에서 잘리고 변명처럼 읽힌다. 200/300/150 상한을 지킨다.
- **모호한 에스컬레이션** — "심각하면 보고"는 기준이 아니다. 점수·금액·횟수로 못 박는다.

## 활용
- **플랫폼 리뷰 답글 운영**: 쿠팡·스마트스토어 리뷰에 유형 분류→스크립트 적용→붙여넣기 루틴으로 응대 시간을 건당 1~2분으로 줄인다.
- **외주·신입 인수인계**: 매뉴얼 1부로 CS 담당 교체 시 톤 붕괴를 막는다 (에스컬레이션 기준표가 권한 위임 문서 역할).
- **챗봇 이중화**: 071 챗봇의 KB 소스로 넣어 "사람 매뉴얼 = 봇 지식"을 단일 원본으로 유지한다.
