# 073 고객 여정 맵핑

인지→탐색→구매→재구매 4단계에 터치포인트를 배치하고, 단계마다 행동·감정·마찰·개입 스킬 4행을 채운 매트릭스로 "어디서 왜 새는지"를 특정한다. 맵은 내부 산출물 — 개입 실행은 074·048 체인에서.

## Contract
1. 여정 매트릭스는 4단계(인지→탐색→구매→재구매) × 4행(행동·감정·마찰·개입 스킬)을 빠짐없이 채운다 — 근거 없는 칸은 상상으로 채우지 않고 [자료 없음]으로 남긴다.
2. 이탈점은 데이터 수치 또는 VOC 원문 인용으로 특정한다. 데이터 미연결 시 업종 표준으로 추정하되 [추정] 태그를 반드시 단다.
3. 모든 마찰 항목에는 개입 스킬 ID가 1개 이상 붙는다 (예: 결제 이탈 → 048 / 재구매 공백 → 074·077) — 지적만 하고 담당 없는 마찰은 만들지 않는다.
4. 감정은 -2~+2 5점 척도로 통일해 감정 곡선을 그릴 수 있게 한다 — 형용사만 있는 감정 행 금지.
5. 065 세그먼트 리포트가 있으면 우선 공략 세그먼트 기준으로 여정을 그린다 — "평균 고객" 1장으로 세그먼트 차이를 뭉개지 않는다.

## Phases
1. **브랜드·세그먼트 로드** — brand/profile.md(업종·판매 채널·구매주기) 자동 참조. 065 고객 세그먼트 리포트가 있으면 우선 공략 세그먼트를 여정 주인공으로 확정, 없으면 전체 고객 기준 + [추정] 명시.
2. **여정 단계 확정** — 기본 4단계(인지→탐색→구매→재구매). 업종 특성상 필요하면 탐색을 비교/검증으로 분화하되 6단계 상한.
3. **터치포인트 인벤토리** — 단계별 접점을 채널 단위로 나열 (광고·검색·SNS·상세페이지·결제·배송·CS·재구매 알림 등). 실제 운영 중인 채널만 — 없는 채널은 "공백"으로 표기.
4. **4행 매트릭스 채우기** — 단계×터치포인트마다:
   - **행동**: 고객이 실제 하는 일 (데이터 근거)
   - **감정**: -2~+2 점수 (VOC 인용 근거, 없으면 [추정])
   - **마찰**: 이탈·불만을 만드는 지점 (수치·VOC 인용)
   - **개입 스킬**: 마찰을 줄일 실행 스킬 ID (048 CRO / 074 이메일 / 075 알림톡 / 077 윈백 / 054 리뷰 등)
5. **이탈점 TOP 3 특정** — 마찰 중 이탈 규모×감정 낙폭이 큰 순으로 3개. 각각 근거 1줄 + 개입 스킬 + 기대 효과 방향.
6. ⏸ **개입 우선순위 확정** — TOP 3 중 먼저 실행할 1~2개를 사용자가 선택 → 074 이메일 시퀀스 빌더(관계·재구매 마찰) 또는 048 랜딩페이지 CRO 진단(구매 전환 마찰)으로 핸드오프.

## Anti-Patterns
- **상상 여정** — 데이터·VOC 없이 "고객은 불안해한다"고 채우는 것. 근거 없으면 [추정]·[자료 없음]을 남기는 게 정직한 맵이다.
- **터치포인트 나열로 종료** — 개입 스킬 행이 비면 여정 맵은 벽지다. 마찰 수 = 개입 ID 수.
- **전 구간 동시 개선 제안** — TOP 3 → 1~2개 선택이 원칙. 열 군데 고치자는 맵은 아무것도 못 고친다.
- **업종 무시 템플릿** — B2C 커머스 여정을 교육·B2B에 그대로 적용 금지. 단계·터치포인트는 profile.md 업종에서 도출.
- **1회 작성 후 방치** — 개입 실행(074·048) 후 감정·이탈 수치를 갱신해야 맵이 산다. 재실행 주기를 산출물에 명시.

## 활용
- **CRM 설계 첫 단추**: 065 세그먼트 → 073 여정 맵 → 074 시퀀스로 이어지는 /리텐션캠페인 체인의 두 번째 단계 — "누구에게"(065) 다음 "어디서"(073)를 정한다.
- **광고비 누수 진단**: 인지·탐색 마찰이 크면 광고를 더 태워도 밑 빠진 독 — 048 CRO를 광고 증액보다 먼저 실행할 근거가 된다.
- **재구매 공백 발견**: 재구매 단계 터치포인트가 "공백"으로 나오는 브랜드가 대부분 — 그 칸 하나가 074·077 도입의 사업 근거가 된다.
