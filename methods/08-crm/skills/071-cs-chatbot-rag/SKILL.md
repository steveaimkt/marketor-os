# CS 챗봇 구축 (RAG)

> 벡터 DB 없이 파일 기반 RAG: `_index.md` 키워드 라우팅 → KB 파일 로드 → 근거 있는 답변 + 출처 표기.
> 계승: 지식베이스 레그 에이전트의 KB 필수 6섹션 구조·키워드 라우팅·확신도 3단계(HIGH/MEDIUM/LOW)·에스컬레이션 3레벨·응답 5단계 형식.

## Contract
- 봇은 KB에 근거가 있는 답만 하고, 모든 답변에 출처(`파일명 > 섹션명`)를 표기한다 — 추측성 답변은 구조적으로 차단된다.
- 확신도 LOW(KB에 근거 없음)면 답변 대신 상담원 연결을 안내한다.
- Level 1 주제(법적 분쟁·금전 보상 요구·개인정보·감정 격앙·안전/건강 긴급)는 KB에 있어도 자동응대에서 제외된다.
- 모든 KB 파일은 표준 구조(YAML 헤더 + 필수 6섹션 + 키워드 5~15개)를 지키고, `_index.md`에서 라우팅 가능하다.
- 배포 선언 전 테스트 문답 10문항(정상 6·부분 2·에스컬레이션 2)을 통과해야 한다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) 자동 참조 (업종·상품·응대 톤). 톤 기본값: 해요체, 공감 → 해결 → 안내 순서.
2. **문서 수집·카테고리 설계** — 보유 문서(FAQ·정책·매뉴얼) 입력받고 업종별 TOP 5 카테고리 제안 (예: 배송/반품·교환/결제/회원/상품). 문서가 없으면 사용자와 함께 초안 작성 모드로 전환.
3. **KB 구축** — 파일당 규칙: 파일명 `카테고리_주제.md`, YAML 헤더(category·keywords·last_updated·version), 필수 6섹션(개요/FAQ 3+/처리 절차/정책 기준표/에스컬레이션 조건/관련 문서). 키워드는 구어체 포함 5~15개 (예: "안 옴" → 배송_지연).
4. **_index.md 라우팅 테이블 생성** — 카테고리 맵 + 키워드→파일 매핑. 질문 처리 흐름: 질문 → 인덱스 키워드 매칭 → 파일 로드 → 답변 생성.
5. **응대 정책 정의** — 확신도 3단계(HIGH: 정확한 근거 → 응답 / MEDIUM: 부분 근거 → 응답+고객센터 권유 / LOW: 근거 없음 → 에스컬레이션) + Level 1/2/3 에스컬레이션 기준과 연결. 응답 형식: 공감(1줄)→핵심(2~3줄)→절차(번호)→추가 안내→출처.
6. **테스트 문답 검증 ⏸** — 10문항(정상 6·부분 응대 2·에스컬레이션 2) 실행 → 정답/출처/에스컬레이션 판정 정오표를 사용자가 확인·승인해야 구축 완료 선언.
7. **(선택) Notion 동기화 설계** — CS팀이 비개발자면 Notion=Source of Truth, 로컬=캐시 구조와 동기화 규칙(최종수정일 우선) 안내.
8. **산출 + 체인 제안** — 072 CS 응대 매뉴얼 빌더 (사람 상담원용 매뉴얼을 같은 KB에서 파생).

## Output Format
```markdown
# 🤖 CS 챗봇 구축: {브랜드} ({구축일})
## KB 구성
knowledge_base/
  _index.md          ← 키워드 라우팅 테이블 ({N}개 키워드 → {M}개 파일)
  배송/배송_지연.md   ← 필수 6섹션 ✓ · 키워드 8개
  반품_교환/…

## 응대 정책 요약
| 확신도 | 조건 | 행동 |
|---|---|---|
| HIGH | KB 정확 근거 | 응답 + 출처 |
| MEDIUM | 부분 근거 | 응답 + 고객센터 권유 |
| LOW | 근거 없음 | 상담원 연결 안내 |
자동응대 제외(Level 1): 법적 분쟁 · 금전 보상 · 개인정보 · 감정 격앙 · 안전/건강

## 테스트 결과 (10문항)
| # | 질문 | 기대 | 판정 | 출처 |
|---|---|---|---|---|
| 1 | "배송 언제 와요?" | 정상 응대 | ✅ | 배송_지연.md > 처리 절차 |
## 다음 액션
→ 072 CS 응대 매뉴얼 빌더 (같은 KB로 상담원용 매뉴얼 생성)
```

## Anti-Patterns
- **KB 없이 봇부터** — 매뉴얼이 부실하면 답변도 부실하다 (KB 품질 = CS 품질). 문서 정리가 선행 과제다.
- **출처 없는 답변 허용** — 출처 표기가 빠지는 순간 추측 답변과 구분할 수 없게 된다.
- **민감 주제 자동응대** — "환불 규정이 KB에 있으니 보상 요구도 답한다"는 사고다. Level 1은 무조건 상담원.
- **키워드를 문어체로만 등록** — 고객은 "환불 정책"이 아니라 "돈 돌려줘"라고 쓴다. 구어체·동의어 필수.
- **테스트 생략 배포** — 에스컬레이션 케이스 2문항이 통과 안 된 봇은 사고 대기 상태다.

## 활용
- **1인 쇼핑몰 야간 응대**: 자주 오는 문의 TOP 5 카테고리만 우선 구축해 야간·주말 1차 응대를 맡긴다.
- **CS팀 온보딩 단축**: _index.md와 KB가 신입 상담원 교육 자료를 겸한다 (072로 매뉴얼화).
- **정책 변경 대응**: 정책 바뀔 때 해당 KB 파일 1개 + last_updated만 갱신하면 봇 답변이 즉시 반영된다.
