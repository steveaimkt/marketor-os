# 079 리뷰·후기 요청 자동화

구매 후 D+3 도착 확인 → D+10 리뷰 요청 → D+17 리마인드의 시점 플로우를 설계하고, 대가성 표기 규정을 지킨 인센티브와 함께 ⏸승인 후 발송한다. 수집된 리뷰의 답변·관리는 054 체인에서.

## Contract
1. 요청 시점은 제품 사용주기에 맞춘다 — 기본 D+3(도착 확인)·D+10(리뷰 요청)·D+17(리마인드 1회), 사용 경험 형성이 느린 제품은 D+10을 사용주기만큼 미룬다. 배송 직후 일괄 요청 금지.
2. D+3 메시지는 리뷰를 요구하지 않는다 — 도착 확인·사용 안내·문제 선제 접수만 (불만의 리뷰화 예방이 목적).
3. 인센티브가 있으면 ① 리뷰 게시물에 대가 제공 사실 표기 안내 문구를 포함하고 ② 긍정 리뷰 조건(별점 5점 시 지급 등)을 걸지 않으며 ③ 선정형 이벤트는 기준·인원·발표일을 공개한다.
4. 리뷰 요청은 고객당 최대 2회(요청+리마인드)로 제한되고, 리뷰 작성 확인 시 잔여 발송을 중단한다.
5. ⏸ 실제 발송은 사용자 승인 후에만 실행되고, 발송 내역은 로그로 기록된다.

## Phases
1. **브랜드·데이터 로드** — brand/profile.md(제품·사용주기·톤) 자동 참조. 구매 데이터(주문일·채널·제품) 로드 — 미연결 시 플로우·메시지 설계만 하고 [발송 대기] 표기.
2. **시점 플로우 설계** — 제품 사용주기 반영:
   | 시점 | 메시지 | 목적 | 중단 조건 |
   |---|---|---|---|
   | D+3 | 도착 확인·사용 안내 | 경험 안착 + 불만 선제 접수 | CS 접수 시 요청 보류 |
   | D+10 | 리뷰 요청 (+인센티브) | 리뷰 작성 | 작성 확인 시 종료 |
   | D+17 | 리마인드 1회 | 미작성자 회수 | 작성·수신거부 시 종료 |
   화장품·건강식품 등 효과 체감형은 D+10 → D+{사용주기}로 조정하고 사유 명시.
3. **메시지·인센티브 작성** — 통당 CTA 1개(리뷰 작성 링크). 인센티브 설계: 지급물(적립금·쿠폰)·예산·지급 조건. 대가성 안내 문구를 메시지에 명기: "작성하신 리뷰에는 '적립금을 받고 작성했다'는 표기가 필요해요" 수준으로 고객 행동까지 안내. D+3 메시지에는 리뷰 언급 자체를 넣지 않는다.
4. **compliance-gate 검사** — 대가성 표기 누락·긍정 조건 문구·과대 표현 전수 스캔, 게이트 결과 블록 부착. ⛔ 판정 메시지는 승인 단계로 못 간다.
5. ⏸ **발송 승인** — 플로우+메시지 전문+인센티브 예산을 보여주고 y/n. n이면 발송 없이 설계만 인도.
6. **발송·로그** — 이메일 도구 연결 시 초안 생성(기본), 미연결 시 채널별 복붙 패키지 인도. 발송/인도 내역 로그 1행 기록. (카톡 채널 병행은 075 규격 참조.)
7. **체인 제안** — 054 리뷰 관리 시스템 (수집된 리뷰의 감정 분류·답변 초안·에스컬레이션).

## Output Format
```

판정 요약: 플로우 D+3/D+{10}/D+{17} · 인센티브 {지급물}(건당 {원}) · 게이트 {✅}

### ① 시점 플로우
| 시점 | 메시지 | 목적 | 중단 조건 |
|---|---|---|---|

### ② 메시지 세트 (통당 CTA 1개)
**D+3 도착 확인** > {본문 — 리뷰 언급 없음}

지급물 {적립금 n원} · 조건 {작성 시 — 별점 무관} · 월 예산 {원} · 대가성 표기 안내 ✅

─── 🛡 컴플라이언스 게이트 ─── 판정: ✅ 통과 ───

### ④ 발송 승인 ⏸ → 승인 시: 초안 {n}통 생성 + 발송 로그 기록

```

## Anti-Patterns
- **배송 완료 즉시 리뷰 요청** — 써보기도 전의 요청은 무성의한 리뷰와 무시를 낳는다. D+3은 도착 확인, 요청은 사용 후.
- **긍정 조건 인센티브** — "별점 5점 인증 시 지급"은 규정 위반이자 리뷰 신뢰 붕괴. 지급 조건은 작성 여부만.
- **대가성 표기 생략** — 적립금 주고 표기 안내를 빼면 브랜드가 기만광고 리스크를 진다. 게이트에서 필수 검사.
- **무한 리마인드** — 3회 이상 조르면 리뷰 대신 수신거부가 온다. 요청+리마인드 2회 상한.
- **승인 없이 발송** — 게이트 통과는 문구 검증일 뿐이다. ⏸ y 이전에 나가는 메시지는 0통.

## 활용
- **신제품 초기 리뷰 확보**: 런칭 첫 구매자 대상으로 플로우를 가동해 상세페이지 사회적 증거를 빠르게 축적 — 048 CRO 진단의 "후기 부족" 마찰 해소.
- **포토리뷰 승격 캠페인**: 텍스트 리뷰 작성자에게 포토 추가 시 추가 적립 오퍼 — 인센티브 설계 규정(③) 그대로 재사용.
- **부정 경험 선제 차단**: D+3 응답에서 불만이 접수되면 리뷰 요청을 보류하고 072 CS 매뉴얼 플로우로 전환 — 별점 방어가 아니라 문제 해결이 먼저다.
