# 쇼츠/릴스 대본 팩

> 롱폼 원본 1개 → 독립 메시지 5개 추출 → 채널 문법 적용 숏폼 대본 5개 (첫 3초 훅 별도 설계).
> 계승: 멀티 콘텐츠 생성기 `/spread`의 롱폼→숏폼 변환 규칙 — 핵심 1개 추출·후킹-본문-CTA 3막·릴스/틱톡/쇼츠 채널 차등표.

## Contract
- 대본은 정확히 5개, 각 대본은 원본에서 추출한 독립 메시지 1개만 다룬다 (1대본 = 1메시지).
- 첫 3초 훅은 본문과 별도로 설계하며, 결과·반전·수치 중 하나로 시작한다 — 배경 설명 시작 금지.
- 채널 규격을 지킨다: 쇼츠 30~60초 · 릴스 30~60초 · 틱톡 15~40초 (5초 ≈ 낭독 20자 환산 검산).
- 원본의 수치·주장은 왜곡 없이 보존한다 — 숏폼용 과장 각색 금지.
- 산출물은 compliance-gate 통과 전에는 전달되지 않는다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) + [brand/tone.md](../../../../brand/tone.md) 자동 참조.
2. **원본 분석 + 메시지 5개 추출** — 추출 우선순위: ① Before/After가 가장 극적인 장면 ② 훅의 최강 문장 ③ 실패담·반전 ④ 단독으로 성립하는 팁 ⑤ 결과물 시연 장면. 각 메시지가 원본의 어느 부분에서 왔는지 표기.
3. **채널 문법 적용** — 대상 채널별 차등: 쇼츠(본편 맛보기 톤 · 풀영상 채널 유도 · #Shorts) / 릴스(정보 전달형 · 프로필 링크 유도 · 해시태그 15~20개) / 틱톡(캐주얼 · 루프+팔로우 유도 · #fyp 필수). 채널 미지정 시 쇼츠 기준.
4. **3막 대본 작성** — 각 대본: 훅(0~3초, 결과/반전 먼저) → 본문(메시지 1개 전개, 자막 강조 단어 표기) → CTA(채널별 유도 문구). 화면 지시 1줄 포함.
5. **자체 검증** — 5개 각 1메시지 / 훅 3초 규격 / 길이 환산 검산 / 원본 수치 대조 / 채널별 CTA·해시태그 규칙.
6. **컴플라이언스 게이트** — [gates/compliance-gate.md](../../../../gates/compliance-gate.md) 3 STEP 실행, 게이트 결과 블록 부착. ⛔ 차단이면 수정·재통과 전 전달 금지.
7. **체인 제안** — 027 OSMU 멀티채널 변환기 (같은 원본을 텍스트 채널로도 확산).

## Anti-Patterns
- **롱폼 요약본 5개** — 요약은 숏폼이 아니다. 각 대본은 독립 메시지 1개를 완결시킨다.
- **훅에서 배경 설명** — "오늘은 ~에 대해"로 시작하면 3초 안에 스와이프된다. 결과·반전·수치 먼저.
- **한 대본에 팁 3개** — 메시지가 2개 이상이면 대본을 쪼갠다. 남는 메시지는 다음 대본 소재.
- **채널 무시 복붙** — 틱톡 캐주얼 톤을 쇼츠에 그대로 쓰거나 해시태그 규칙을 섞으면 채널 알고리즘 모두에서 손해.
- **숏폼용 수치 부풀리기** — "2시간→15분"을 "10배 빨라짐"으로 각색하는 순간 게이트 차단 대상이다.

## 활용
- **롱폼 1 → 콘텐츠 6**: 024 스크립트 발행 직후 이 스킬로 숏폼 5개를 뽑아 일주일 내내 본편 유입 트래픽을 만든다.
- **블로그 재활용**: 022 아티클의 소제목 5개를 메시지로 추출하면 텍스트 자산도 숏폼 소스가 된다.
- **훅 실험실**: 같은 메시지 1개에 훅 유형(수치/반전/실패담)만 바꿔 3벌 발행 → 반응 좋은 훅 유형을 040 훅 라이브러리에 축적한다.
