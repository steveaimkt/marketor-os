# 스레드 콘텐츠 팩

> 주제 1개 → 스크롤 정지 훅 → 1편 1포인트 분절 → 내러티브 아크로 이어지는 시리즈 5편 + 참여 유도 리플.
> 설계: 028 링크드인 빌더의 검증 훅 공식(부정 명령형·숫자 대조)을 스레드 문법(짧은 연속 포스트·캐주얼 해요체)으로 변주한 신규 스킬.

## Contract
- 시리즈는 정확히 5편, 각 편은 500자 이내이며 1편은 1포인트만 다룬다 — 링크드인처럼 한 포스트에 ❶❷❸을 몰아넣지 않는다.
- 1편 첫 줄은 스크롤 정지 훅(부정 명령형·조건형·수치 반전 중 1개, 15자 내외)이고, 각 편 끝은 다음 편 예고 또는 여백 질문으로 연결된다.
- 시리즈 전체에 숫자·비용 대조가 1개 이상 포함된다 ("119만원 → 월 1.3만원" 식).
- 문체는 캐주얼 해요체 — 링크드인의 격식 톤을 그대로 가져오지 않는다. 해시태그 기본 미사용.
- 산출물은 compliance-gate 통과 전에는 전달되지 않는다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) + [brand/tone.md](../../../../brand/tone.md) 자동 참조 (계정 톤·금기어). 브랜드 정보 재질문 금지.
2. **훅 변주 설계** — 028 훅 공식을 스레드형으로 압축: 부정 명령형("~하지 마세요")·조건형("~하고 있다면")·수치 반전("2시간 걸리던 게 15분") 3안 작성 → 최강 1안을 1편 첫 줄로.
3. **시리즈 아크 배치** — 5편 고정 역할: ①훅+약속(끝까지 읽으면 얻는 것) → ②문제·공감(구체 상황) → ③해결 핵심(숫자 대조 포함) → ④증거·사례(경험 1개) → ⑤정리+CTA(팔로우/리포스트/댓글 유도 중 목적에 맞는 1개).
4. **편별 작성** — 편당 500자 이내, 2~4문장 단락, 의미 단위 줄바꿈. 각 편이 단독 노출돼도 성립하도록 자기완결 + 연결 문장 1개.
5. **후속 리플 2종** — 시리즈 게시 후 붙일 참여 리플: (a) 질문 리플(대화 유발) (b) 자료·링크 리플(유입, 본문엔 링크 미배치).
6. **자체 검증** — 5편×1포인트 / 편당 500자 / 훅 형식 / 숫자 대조 ≥1 / 해요체 일관 / 연결 문장 존재.
7. **컴플라이언스 게이트** — [gates/compliance-gate.md](../../../../gates/compliance-gate.md) 3 STEP 실행, 게이트 결과 블록 부착 후에만 전달.
8. **체인 제안** — 027 OSMU 멀티채널 변환기 (같은 코어를 링크드인·뉴스레터 버전으로).

## Anti-Patterns
- **링크드인 글 5등분** — 028 완성본을 잘라 붙이면 편마다 훅이 없어 2편부터 이탈한다. 편별로 첫 줄을 다시 쓴다.
- **1편에 결론 전부 공개** — 약속 없이 다 말하면 시리즈를 읽을 이유가 사라진다. 1편은 훅+약속까지만.
- **격식 문체 유지** — "~합니다" 격식 톤은 스레드에서 광고로 읽힌다. 해요체로 낮춘다.
- **본문에 링크 배치** — 링크 포함 포스트는 도달이 깎인다. 링크는 후속 리플로 뺀다.
- **편 간 연결 고리 생략** — 예고·질문 없이 뚝뚝 끊기면 시리즈가 아니라 낱개 5개다.

## 활용
- **브랜드 계정 연재 루틴**: 021 캘린더의 Threads 슬롯(주 2~3회)을 이 스킬로 시리즈화 — 낱개 포스트보다 팔로우 전환이 높다.
- **028과 짝 운영**: 같은 소재를 링크드인(격식·B2B)과 스레드(캐주얼·B2C)로 이원 발행해 채널별 반응 차이를 학습한다.
- **VoC 반박 시리즈**: 006 리뷰 마이닝의 오해·불만 TOP 3를 "①오해 ②실상 ③증거" 아크로 연재해 신뢰를 쌓는다.
