# YouTube 스크립트 (성과 증명형)

> 주제 + 증명 소재 → 훅-시연-데이터-CTA 6섹션 10분 스크립트 (타임코드·화면 지시 포함).
> 계승: 유튜브 스크립트 제작 에이전트 `script-creator`의 TYPE-A 6섹션 구조·TC 계산 공식·구어체 나래이션 규칙 + marketing-os `youtube-content-brief`의 데이터 근거 앵글 도출.

## Contract
- 스크립트는 6섹션 고정 비율을 따른다: HOOK 6% · PAIN 8% · KEY 7% · DEMO 20% · BRIDGE 40% · CTA 20%.
- 첫 30초(HOOK) 안에 영상의 최종 결과물을 먼저 보여준다 — 과정 설명으로 시작 금지.
- 총 글자 수 = 목표 분 × 253자 ±10% (10분 = 약 2,530자). 섹션별 TC는 글자 수로 검산한다.
- 모든 주장 옆에 시연 장면 지시 또는 데이터 근거가 붙는다. 근거 없는 수치는 [추정] 태그.
- 산출물은 compliance-gate 통과 전에는 전달되지 않는다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) + [brand/tone.md](../../../../brand/tone.md) 자동 참조 (톤·포지셔닝·금기어).
2. **증명 포인트 확정** — 이 영상이 증명할 것 1개(핵심 메시지)와 증거 목록(Before/After 수치·시연 결과물·데이터) 정리. 증거가 비면 사용자에게 요청 ⏸ — 증거 없는 성과 증명형은 성립하지 않는다.
3. **앵글 도출** — 경쟁 영상·채널 데이터가 있으면 갭(경쟁은 다루는데 나는 없는 주제)과 강점 교집합으로 앵글 결정. 데이터 미연결 시 타겟 페인포인트 기반 추정([추정] 태그).
4. **6섹션 설계** — HOOK(결과물 시연+무료 공개 예고) → PAIN(구체 수치로 문제 공감) → KEY(해결책 3가지 예고+구독 유도) → DEMO(시스템 설명, 비유 활용) → BRIDGE(단계별 실행 튜토리얼) → CTA(Before/After 대조+자료 배포+다음 행동).
5. **나래이션 작성** — 구어체("~거든요/~잖아요" 혼용) + 구체 수치 + 일상 비유. 감정 곡선: 문제(진지)→해결(밝게)→성과(신나게). 화면 지시 마커 삽입: `<!-- 슬라이드: -->` / `<!-- 화면녹화: -->`.
6. **TC 계산 + 검산** — 섹션별 글자 수 ÷ 253 = 분. SCREEN 구간은 행동 여유 ×1.2. 누적 TC를 섹션 헤더에 표기하고 총량 ±10% 검산.
7. **컴플라이언스 게이트** — [gates/compliance-gate.md](../../../../gates/compliance-gate.md) 3 STEP 실행 (과장 성과·보장성 표현 집중 스캔), 게이트 결과 블록 부착.
8. **체인 제안** — 025 썸네일·제목 A/B (HOOK 소재 전달) → 026 쇼츠/릴스 대본 팩 (이 스크립트가 롱폼 원본).

## Anti-Patterns
- **과정부터 시작하는 오프닝** — "안녕하세요, 오늘은…"은 이탈 1순위. HOOK은 결과물이 먼저다.
- **주장만 있고 증거 없는 섹션** — "정말 편해집니다"에 시연도 수치도 없으면 성과 증명형이 아니다. 근거를 붙이거나 문장을 뺀다.
- **문어체 나래이션** — "~하였습니다" 낭독체는 TTS·실녹음 모두 어색하다. 말하듯 쓴다.
- **비율 무시 BRIDGE 축소** — 시청 지속의 몸통은 40% 튜토리얼. 설명(DEMO)만 길어지면 이탈 곡선이 꺾인다.
- **TC 없는 납품** — 글자 수 검산 없는 스크립트는 촬영 현장에서 반드시 초과한다. 253자/분 공식으로 검산 후 전달.

## 활용
- **주간 영상 루틴**: 021 캘린더의 영상 슬롯 주제를 이 스킬로 대본화 → 025 제목·썸네일 → 026 숏폼 5개까지 한 편에서 7개 콘텐츠 파생.
- **교육·강의 영상**: BRIDGE 비율을 50~60%로 올리고 DEMO에 개념 비유를 확장하면 튜토리얼 강의 대본으로 전용된다.
- **고객 사례 영상**: 고객 Before/After 수치를 증명 포인트로 잡으면 세일즈 자산이 되는 사례 영상 대본이 나온다.
