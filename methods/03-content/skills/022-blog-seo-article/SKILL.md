# 블로그 SEO 아티클

> 키워드 1개 → 검색의도 4유형 판정 → 유형별 표준 아웃라인 → SEO 최적화 본문 + 메타 (게이트 통과본).
> 설계: 001 키워드 수요 리서치와 체인되는 검색의도 분석 구조 — 의도 판정이 틀리면 글 전체가 틀린다.

## Contract
- 본문 구조는 검색의도 4유형(정보형/비교형/거래형/탐색형) 판정 결과의 표준 아웃라인을 따른다 — 판정 근거를 명시한다.
- 키워드는 제목 1회 · H2 중 2개 이상 · 도입 3문장 이내 1회 배치, 본문 밀도 1~2%를 지킨다. 부자연스러운 반복 금지.
- 메타 타이틀 30자 이내(키워드 앞배치) · 메타 디스크립션 70~80자.
- 모든 수치·주장에 출처를 표기하고, 출처 없는 항목은 [추정] 태그를 붙인다.
- 산출물은 compliance-gate 통과 전에는 전달되지 않는다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) + [brand/tone.md](../../../../brand/tone.md) 자동 참조 (톤·금기어·타겟). 브랜드 정보 재질문 금지.
2. **검색의도 판정** — 키워드를 4유형 중 1개로 판정하고 근거 기술. 001 산출물이 있으면 의도 컬럼 계승, 없으면 키워드 형태소로 추정([추정] 태그): "~방법/~이유"=정보형 · "~추천/~비교/vs"=비교형 · "~가격/~구매/최저가"=거래형 · 브랜드명 직접 검색=탐색형.
3. **아웃라인 설계** — 유형별 표준 구조 적용: 정보형(정의→단계별 방법→주의점→FAQ) / 비교형(선정 기준→비교표→상황별 추천→FAQ) / 거래형(핵심 혜택→스펙·증거→후기→구매 안내) / 탐색형(공식 정보 정리→차별점→다음 행동). H2 4~6개, 각 H2에 독자가 얻는 것 1줄.
4. **본문 작성 (2,000~3,000자)** — 도입 3문장 안에 "이 글을 읽으면 얻는 것" + 키워드. 소제목만 훑어도 결론이 보이게. 표·리스트 최소 2개(체류 시간·피처드 스니펫 대응).
5. **메타 + FAQ** — 메타 타이틀·디스크립션 작성, 본문에서 파생한 FAQ 3문항(스키마 마크업용).
6. **자체 검증** — 의도-구조 일치 / 키워드 배치 규칙 / 글자 수 / 출처·[추정] 표기 / H2 4~6개.
7. **컴플라이언스 게이트** — [gates/compliance-gate.md](../../../../gates/compliance-gate.md) 3 STEP 실행, 게이트 결과 블록 부착. ⛔ 차단이면 수정·재통과 전 전달 금지.
8. **체인 제안** — 027 OSMU 멀티채널 변환기 (이 글을 앵커로 링크드인·스레드·뉴스레터 확산).

## Output Format
```markdown

| 독자 상태 | 구매 직전, 선택 기준이 필요함 |
```

## Anti-Patterns
- **의도 무시 만능 구조** — 거래형 키워드에 백과사전식 정보 글을 쓰면 순위도 전환도 없다. 판정이 먼저다.
- **키워드 스터핑** — 밀도 2% 초과 반복은 저품질 신호. 동의어·연관어로 분산한다.
- **출처 없는 수치** — "연구에 따르면"으로 뭉개지 않는다. 출처를 달거나 [추정]을 붙이거나 뺀다.
- **도입부 서사 낭비** — 3문장 안에 얻는 것이 안 보이면 이탈. 브랜드 자랑은 본문 뒤로.
- **금기어를 게이트에 떠넘기기** — 게이트는 최종 방어선. 작성 단계에서 brand/tone.md 금기어를 이미 회피한다.

## 활용
- **키워드 → 글 파이프라인**: 001 TOP 20 표에서 상위 키워드를 주 2건씩 이 스킬로 집필 — 021 캘린더의 Blog 슬롯이 자동으로 채워진다.
- **상세페이지 트래픽 유입**: 거래형 키워드 아티클의 구매 안내 섹션에 자사 상세페이지 링크를 배치해 오가닉 유입 → 전환 경로를 만든다.
- **오래된 글 리라이트**: 순위 하락 글을 검색의도 재판정 → 구조 교체만으로 재상승시킨다 (본문 재작성보다 구조 교정이 먼저).
