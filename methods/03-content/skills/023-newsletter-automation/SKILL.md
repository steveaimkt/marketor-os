# 뉴스레터 제작 자동화

> 주제·자료 → 본문 800자 + 제목 5종 + preheader + 요약 1줄 (브랜드 톤 적용, 게이트 통과본).
> 계승: marketing-os `newsletter-writing`의 글쓰기 6원칙·제목 5패턴·자체 검증 체크리스트.

## Contract
- 본문은 700~900자, 첫 문장에 핵심이 오는 두괄식이다.
- 제목은 5패턴(숫자 호기심·이번 주 정답·반전 금기·데이터 헤드라인·이름 호출) 각 1개씩 정확히 5종이다.
- CTA는 정확히 1개만 포함된다.
- 금기어 0건 — 작성 단계에서 brand/tone.md 금기어를 동의어로 대체한다.
- 산출물은 compliance-gate 통과 전에는 전달되지 않는다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) + [brand/tone.md](../../../../brand/tone.md) 자동 참조 (톤·어휘·금기어·타겟 페르소나·이모지 정책). 브랜드 정보 재질문 금지.
2. **소스 정리** — 자료 3~7개의 핵심을 추출하고, 이번 호 핵심 메시지를 1문장으로 확정.
3. **본문 작성 (~800자)** — 두괄식 · 2인칭("당신의") · 숫자 구체화("더 좋아진" → "21일 후 -38%") · CTA 1개 · 이모지는 주제어 옆 1개 원칙.
4. **제목 5종 + preheader** — 5패턴 각 1개, preheader는 제목과 중복 금지. 예상 오픈율을 "추정" 라벨로 병기.
5. **자체 검증** — 글자 수 700~900 / 두괄식 / CTA 1개 / 금기어 0 / 5패턴 각 1개 / preheader 비중복.
6. **컴플라이언스 게이트** — [gates/compliance-gate.md](../../../../gates/compliance-gate.md) 3 STEP 실행, 산출물 하단에 게이트 결과 블록 부착. ⛔ 차단이면 수정·재통과 전 전달 금지.
7. **체인 제안** — 074 이메일 시퀀스 빌더 (발송은 그쪽의 ⏸ HITL 승인 영역).

|---|---|---|

## 본문 (HTML, {글자 수}자)

→ 074 이메일 시퀀스 빌더 (승인 후 발송·아카이브)

## Anti-Patterns
- **CTA 2개 이상** — 클릭률을 떨어뜨린다. 링크가 여러 개여도 행동 요청은 1개.
- **3인칭 문체** — "고객들은…"이 아니라 "당신의…"로 쓴다.
- **제목 5개가 한 패턴의 변주** — 5패턴 각 1개가 규격. 같은 공식 반복은 재작성.
- **금기어를 게이트에 떠넘기기** — 게이트는 최종 방어선. 작성 단계에서 이미 회피한다.
- **발송까지 실행** — 이 스킬은 작성까지. 발송·구독자 조작은 074의 HITL 승인 절차를 거친다.

## 활용
- **주간 뉴스레터 루틴**: 005 트렌드 스캐닝 산출물을 소스로 받아 매주 같은 구조의 레터를 20분 내 완성한다.
- **신제품 안내 메일**: 상세페이지 핵심 섹션 3개를 소스로 넣고 CTA를 구매 링크 1개로 고정한다.
- **오픈율 실험**: 제목 5종을 074에 넘겨 A/B 발송 → 성과를 다음 호 제목 패턴 선택에 반영한다.
