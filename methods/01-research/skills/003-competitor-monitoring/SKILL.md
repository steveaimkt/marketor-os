# 경쟁사 상시 모니터링

> 경쟁사 사이트·SNS 스냅샷 → 전회 대비 필드 단위 diff → 5분류 태깅 → 주간 변화 리포트.
> 계승: marketing-os `competitor-monitor`의 구조화 추출 스키마·필드 단위 diff·5분류·델타값 보고 원칙. 일간 감시를 **주간 추적 리포트**로 개조.

## Contract
- 전회 스냅샷 대비 **변경된 것만** 보고한다 — 변화가 없으면 "변화 없음" 1줄로 끝낸다 (알림 피로 방지).
- 모든 변경은 5분류(🆕 신제품 / 💰 가격 / 🎯 카피 / 📅 이벤트·프로모션 / 🖼️ 비주얼) 중 정확히 1개로 태깅된다.
- 가격·할인 변경은 서술("내렸다")이 아니라 **델타값**(29,000→16,000, 할인 0%→60%)으로 기록된다.
- 외부 적재(notion)와 스케줄 자동 가동은 ⏸ 사용자 승인 후에만 실행한다 (mutating).
- 직접 수집하지 못한 값은 [추정] 태그를 달고, 수집 실패 대상은 ⚠️ 표시 후 사유를 로그에 남긴다.

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) 자동 참조 (카테고리·직접 경쟁사·주력 채널). 브랜드 정보 재질문 금지.
2. **추적 대상 확정** — 경쟁사 목록(브랜드명·도메인·SNS 핸들) 최대 10개 + 주기(기본 주 1회) 확정. 도메인 유효성 검증: 본문이 비거나 법인 안내·점검 중 플레이스홀더면 실 스토어 URL을 탐색하거나 ⚠️ 표시 후 스킵.
3. **스냅샷 수집** — 핵심 페이지(홈·베스트·이벤트)를 구조화 추출.
   - **데이터 소스**: Firecrawl 연동 시 JSON 스키마 추출(아래 표) / 미연결 시 WebFetch 텍스트 수집(필드 축소 + [추정] 표기) / 그마저 불가하면 사용자 캡처 첨부 수동 모드. 연동 방법은 [data/connections.md](../../../../data/connections.md) 참조.
4. **필드 단위 diff** — 전회 스냅샷과 키 단위 비교 (텍스트 통짜 diff 금지): `sale_price`·`discount_rate` 변동, 베스트 순위 진입·이탈, 신규 제품명, 프로모션 신규·종료 임박(D-2), `hero_banner.headline`·`key_keywords` 집합 변화. **첫 실행이면 기준선 스냅샷만 저장하고 "다음 회차부터 diff 시작"으로 종료.**
5. **5분류 태깅 + 인사이트** — 변경점을 5분류로 묶고 핵심 인사이트 3~5개 추출. 각 인사이트에 델타 근거 명시 (예: "1+1 전환 + 할인 0%→60% — 재고 소진 신호").
6. ⏸ **리포트 적재** — notion 스냅샷 아카이브에 리포트 + 금회 구조체 저장 (승인 후). 미연결 시 로컬 파일 폴백 + 산출물에 명시. 변경 감지 로그 1행 추가.
7. ⏸ **상시 가동 스위치 + 체인 제안** — 스케줄 등록(주 1회)은 명시적 승인 후에만. 변화 폭이 큰 경쟁사는 002 경쟁사 분석 리포트로 정밀 분석 추천.

### 구조화 추출 스키마 (Firecrawl 연동 시)
```json
{
  "hero_banner": {"headline": "string", "subcopy": "string", "cta": "string"},
  "promotions": [{"title": "string", "type": "string", "discount_rate": "string", "end_date": "string"}],
  "bestsellers": [{"rank": "number", "name": "string", "sale_price": "number", "discount_rate": "string", "review_count": "number"}],
  "key_keywords": ["string"]
}
```

## Output Format
```markdown
# 🔍 경쟁사 주간 변화 리포트 ({전회 수집일} → {금회 수집일})
**한 줄 판정**: {가장 중요한 변화 1개 + 대응 필요도} / 변화 없으면 "변화 없음 — 다음 수집 {날짜}"

## 변경점 (5분류)
| 분류 | 경쟁사 | 변경 내용 (델타) | 해석 1줄 |
|---|---|---|---|
| 💰 가격 | B사 | 마스크팩 6종 40,000→16,000 (0%→60%) | 재고 소진성 딜 추정 |
| 🆕 신제품 | A사 | "글루타치온 세트" 베스트 3위 신규 진입 | 톤업 라인 확장 |

## 인사이트 (3~5건)
1. {패턴} — 근거: {델타값} → {권고 액션}

## 수집 로그
- 수집 성공 {n}/{총 n} · ⚠️ 스킵: {대상 — 사유} · 모드: {Firecrawl/WebFetch/수동}
## 다음 액션
→ 002 경쟁사 분석 리포트 (변화 큰 경쟁사 정밀 비교) · ⏸ 주간 자동 가동 등록 여부
```

## Anti-Patterns
- **텍스트 통짜 diff** — 페이지 전문을 비교하면 날짜·배너 로테이션까지 변경으로 잡힌다. 필드 단위로만 비교한다.
- **변화 없는데 리포트 채우기** — 억지 관찰로 분량을 만들지 않는다. "변화 없음" 1줄이 정답이다.
- **서술형 가격 보고** — "할인 시작했다"는 다음 주에 비교 불가. 반드시 이전값→현재값 델타로.
- **승인 없는 적재·스케줄 등록** — notion 쓰기와 자동 가동은 mutating. ⏸ 없이 실행하지 않는다.
- **첫 실행에 diff 조작** — 기준선이 없는 첫 회차에 "변화"를 만들어내지 않는다. 기준선 저장만 하고 끝낸다.

## 활용
- **주간 경쟁 브리핑**: 매주 월요일 리포트를 팀 채널에 공유 — 가격·프로모션 대응을 주간 회의 안건으로 직결한다.
- **행사 시즌 감시 강화**: 대형 행사 전후로 주기를 임시 단축해 경쟁 딜 구성을 추적, 057 프로모션 캘린더 대응의 입력으로 쓴다.
- **분기 리뷰 소재**: 스냅샷 아카이브 누적분으로 경쟁사의 분기 단위 포지셔닝 이동을 잡아 002 정밀 분석으로 넘긴다.
