# 경쟁사 분석 리포트

> 경쟁 브랜드 2~5개 → 포지셔닝·가격·채널 3축 비교표 → "우리가 비집고 들어갈 빈틈" 3가지.
> 계승: `/시장조사` 경쟁 비교 파트의 스펙 비교표·가격대 분포·출처 병기 규칙과 "공통 강조 3개 / 차별화 포인트 / 빈틈" 3단 요약 프레임.

## Contract
- 모든 경쟁사는 동일한 3축(포지셔닝·가격·채널) + 메시지 축으로 나란히 비교된다 — 축이 비는 칸은 "미확인"으로 남기고 지어내지 않는다.
- 비교표의 모든 수치·주장에는 수집 출처와 수집 시점이 병기된다. 직접 확인하지 못한 값은 [추정] 태그를 단다.
- 기회 갭은 자사가 실행 가능한 것만 3가지로 좁히고, 각 갭은 비교표의 근거 행을 참조한다.
- 경쟁사가 이기는 축도 그대로 기록한다 (자사 우위로 왜곡 금지).
- 같은 입력을 다시 돌리면 같은 구조의 산출물이 나온다 (비교 축·요약 프레임 고정).

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) 자동 참조 (자사 포지셔닝·가격대·주력 채널). 브랜드 정보 재질문 금지.
2. **경쟁사·축 확정** — 경쟁 브랜드 2~5개 확정. 기본 3축(포지셔닝·가격·채널) + 메시지 축, 사용자 요청 축 추가 가능. 5개 초과 시 상위 노출·직접 경쟁 순으로 압축.
3. **공개 정보 수집** — 소스: 자사몰·스토어 상세페이지, 검색 결과, SNS 계정, 사용자 첨부 캡처. 웹 조회 불가 시 추정 모드로 전환하고 모든 값에 [추정] 태그.
4. **3축 비교표 작성** — 동일 축·동일 형식으로 자사 포함 나란히. 가격은 대표 상품 기준 + 옵션 조건 명시, 채널은 운영 여부·활성도(최근 30일 게시 유무)로 기록.
5. **강약점 크로스** — 006 리뷰 마이닝 산출물이 있으면 리뷰 기반 강약점을 축에 병합 (없으면 스킵하고 명시).
6. **3단 요약 + 기회 갭** — ① 경쟁사들이 공통으로 강조하는 것 3개 ② 경쟁사별 차별화 포인트 ③ 우리가 비집고 들어갈 빈틈 3가지 (각각 근거 행 참조 + 실행 난이도 상/중/하).
7. **산출 + 체인 제안** — 009 가격 조사 & 포지셔닝 맵 (가격 축 심화) · 003 경쟁사 상시 모니터링 (변화 추적 전환).

### 지원 소스 (입력으로 받을 수 있는 것)
- 경쟁사 상세페이지 캡처 (영역별 5~6장 — 메인 비주얼/옵션/스펙표/특징/리뷰)
- 스토어 검색 결과 캡처·URL (상위 노출 순위 근거)
- SNS 계정 URL (채널 활성도 판정용)
- 뉴스·보도자료 텍스트 (핵심 인용 + 출처 형식으로 정리)

## Anti-Patterns
- **축이 어긋난 비교** — A사는 가격 얘기, B사는 디자인 얘기를 쓰는 것. 모든 경쟁사를 같은 축으로 채운다.
- **출처 없는 수치** — 가격·리뷰수를 출처·시점 없이 적으면 한 달 뒤 쓸모없는 표가 된다.
- **갭 남발** — 기회 10개 나열은 분석 포기다. 실행 가능한 3개로 압축한다.
- **자사 편향** — 모든 축에서 자사가 이기는 표는 신뢰를 잃는다. 지는 축을 먼저 확인한다.
- **리포트를 결론으로 착각** — 이 비교표는 시작점이다. 핵심 수치는 원본 소스와 대조하라고 산출물에 명시한다.

## 활용
- **신제품 기획 전**: 빈틈 3가지를 012 USP 정의의 입력으로 넘겨 차별화 포지션을 잡는다.
- **분기 전략 리뷰**: 같은 경쟁사 세트로 분기마다 재실행해 축별 변화를 비교한다 (상시 추적이 필요해지면 003으로 전환).
- **광고 메시지 점검**: "공통 강조 3개"와 겹치는 자사 카피를 찾아내 차별화 메시지로 교체한다.
