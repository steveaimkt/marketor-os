# 트렌드 스캐닝

> 시드 키워드 → 연관어 확장 → 3채널(뉴스·커뮤니티·검색) 분리 수집 → 관련성 채점 → 급상승 추출 → 패턴→시사점→액션 합성.
> 계승: marketing-os `trend-scanner`의 연관어 확장·3채널 분리 검색·관련성 점수(0~5)·인사이트 합성 3단·출처 퍼널 해석. 일간 알림을 **카테고리 시그널 추출**로 개조.

## Contract
- 모든 시그널에는 출처 채널 + 매체 도메인 + 근거 링크(또는 발췌)가 붙는다 — 출처 없는 시그널은 표에 못 올라간다.
- 관련성 점수(0~5)가 3 이상인 결과만 보고한다. 제외어 포함 결과는 드롭한다.
- 급상승 판정에는 기준이 명시된다: 스캔 기간 전반부 대비 후반부 언급·검색 증가율 (실측 불가 시 [추정] 태그 + 근거 서술).
- 시사점은 반드시 패턴 → 시사점(왜 중요한가) → 액션(실행 단위 + 근거 데이터 연결) 3단으로 쓴다.
- 같은 시드로 다시 돌리면 같은 구조의 산출물이 나온다 (채널 구성·점수 기준·합성 프레임 고정).

## Phases
1. **브랜드 컨텍스트 로드** — [brand/profile.md](../../../../brand/profile.md) 자동 참조 (카테고리·타깃·제외어). 브랜드 정보 재질문 금지. 시드 미입력 시 profile.md 핵심 카테고리를 시드로 사용.
2. **연관어 확장** — 시드 1~3개 → 연관 키워드 5~8개씩 확장 (예: 토너패드 → 모공패드·각질패드·진정패드…). 확장어도 제외어 필터 통과 필수.
3. **3채널 분리 수집** — 채널별로 나눠 실행하고 출처를 채널 단위로 기록:
   - 뉴스: 최근 30일 카테고리 기사 (WebSearch/뉴스 검색)
   - 커뮤니티·SNS: 후기·해시태그·커뮤니티 글
   - 검색: 네이버 데이터랩·연관 검색어 (검색 수요 흐름)
   - **데이터 소스**: NaverSearch·데이터랩 MCP 또는 Firecrawl 연동 시 실측 / 미연결 시 WebSearch 결과 기반 추정 모드 (수치에 [추정] 표기, 네이버 채널은 검색 결과 중 naver 도메인만 라벨). 연동 방법은 [data/connections.md](../../../../data/connections.md) 참조.
4. **관련성 채점** — 결과별 0~5점: 브랜드 카테고리 적합(0~2) + 시드와의 관련도(0~2) + 최신성(0~1). 3점 이상만 통과.
5. **급상승 추출** — 기간 전반부 대비 후반부 증가율로 급상승 키워드 선별 (예: +180%). 채널 2곳 이상에서 잡힌 시그널은 ★ 교차 확인 표시.
6. **인사이트 합성** — 통과 시그널을 3~5개 패턴으로 클러스터링 → 패턴별 시사점 → 액션 3건. 출처 분포를 퍼널로 해석: 뉴스=인지 / 비교·랭킹=고려 / 후기·커뮤니티=전환 신호.
7. **산출 + 체인 제안** — 021 콘텐츠 캘린더 빌더 (시그널 → 월간 소재) · 011 신제품 컨셉 브레인스토밍 (시그널 → 시장 갭) 추천으로 마무리.

## Anti-Patterns
- **출처 없는 "요즘 뜬다"** — 링크·발췌 없는 시그널은 감상이다. 표에서 제외한다.
- **점수 미달 끼워넣기** — 관련성 2점짜리를 "재미있어서" 올리지 않는다. 3점 컷은 노이즈 방어선이다.
- **키워드 나열로 종료** — 수집은 절반이다. 패턴→시사점→액션 합성이 이 스킬의 핵심 가치다.
- **단일 채널 일반화** — 커뮤니티 한 곳에서만 잡힌 시그널을 시장 트렌드로 승격하지 않는다. 교차 확인 표시로 구분한다.
- **증가율 조작** — 실측 못한 증가율에 그럴듯한 수치를 지어내지 않는다. [추정] + 근거 서술이 원칙이다.

## 활용
- **월초 콘텐츠 기획**: 시그널 TOP 10을 021에 넘겨 월간 캘린더의 "시의성 소재" 슬롯을 채운다.
- **신제품 기획 입력**: 전환 퍼널(후기) 시그널 중 자사 미대응 니즈를 011 컨셉 브레인스토밍의 시장 갭 입력으로 쓴다.
- **주간 정기 스캔**: 같은 시드로 매주 돌려 증가율 추이를 비교 — 2주 연속 상승 시그널만 실행 대상으로 승격한다.
